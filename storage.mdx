Gcore Object Storage is an S3-compatible service for storing media files, logs, backups, AI/ML datasets, and other data types.

## Storage types

Gcore Object Storage offers three storage types:

- **S3 Fast** — high-performance S3-compatible storage for AI/ML workloads, IoT telemetry, and latency-sensitive applications
- **S3 Standard** — general-purpose S3-compatible storage for backups and content delivery
- **SFTP** — legacy file-based storage accessed via SFTP protocol, with the same pricing as S3 Standard

When [creating](/storage/create-an-s3-or-sftp-storage) a storage, the location name indicates the storage type.

## Documentation

New to Object Storage? Start with **Create Object Storage** to provision a storage instance and generate access credentials.

  * **Create Object Storage** — create an S3 or SFTP storage, generate access keys, and configure authentication
  * **Billing** — pricing tiers and billing rules for S3 and SFTP storage
  * **Use Object Storage with CDN** — assign storage as a CDN resource origin and configure content delivery
  * **Manage Object Storage** — manage buckets, control access, and use AWS CLI or S3cmd to set lifecycle policies and generate presigned URLs
  * **Manage SFTP storage** — connect via FileZilla, manage SSH keys in the [Gcore Customer Portal](https://portal.gcore.com)
  * **Request files from Object Storage** — request scheme for objects stored in S3 and SFTP storage
  * **View statistics** — used space, number of requests, and traffic reports
  * **Troubleshooting** — resolve 4xx storage errors
