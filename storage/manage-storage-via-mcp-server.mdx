Gcore MCP Server enables AI assistants to manage Object Storage through natural language commands. This guide covers Storage-specific tools and example conversations.

For general setup instructions, see [Gcore MCP Server overview](/account-settings/integrations/gcore-mcp-server-overview).

## Configure for Storage

Set the `GCORE_TOOLS` environment variable to expose Storage tools:

```bash
GCORE_TOOLS="storage.*"
```

This exposes approximately 40 Storage tools covering S3-compatible object storage and SFTP storage.

## Available Storage tools

| Resource | Operations | Description |
|----------|------------|-------------|
| `object_storages` | list, create, get, delete | S3-compatible storage buckets |
| `sftp_storages` | list, create, get, delete | SFTP storage instances |
| `locations` | list | Available storage locations |
| `ssh_keys` | list, create, delete | SSH keys for SFTP access |
| `statistics` | get | Storage usage statistics |

**Prompt:**
```

```

**Prompt:**

### Delete storage

**Prompt:**
```
Delete the storage bucket "old-backup"
```

<Warning>
Deleting a storage bucket removes all objects permanently. This action cannot be undone.
</Warning>

```
3. Then I'll configure my CDN to use this as origin
```

```
What storage locations are available for migrating data from Luxembourg to a US region?
```

### Storage cost analysis

**Prompt:**

```
Show storage usage and estimated costs for all buckets
```

## S3 compatibility

Gcore Object Storage is S3-compatible. After creating a bucket via MCP, use any S3 client:

```bash
# AWS CLI example
aws s3 ls s3://app-uploads --endpoint-url https://s3.lu.gcore.com

# Upload a file
aws s3 cp file.txt s3://app-uploads/ --endpoint-url https://s3.lu.gcore.com
```

## Tool reference

### storage.object_storages.list

Lists all storage buckets.

**Parameters:** None required

**Returns:** Array of buckets with name, location, size, and object count

### storage.object_storages.create

Creates a new S3-compatible storage bucket.

**Parameters:**
- `name` (required): Bucket name
- `location` (required): Storage location

### storage.locations.list

Lists available storage locations.

**Parameters:** None required

**Returns:** Array of locations with region and S3 endpoint
