Object Storage and SFTP Storage can serve as origins for Gcore CDN resources, allowing content to be delivered from storage through the CDN.

## Use as an origin

Both storage types connect as CDN origins through the same resource creation wizard, with origin configuration varying by storage type.

### Use Object Storage

1. Navigate to the [CDN](https://portal.gcore.com/cdn/resources/list) section and click **Create CDN resource**.

<Frame>![CDN resources list with the Create CDN resource button](/images/docs/storage/use-storage-as-the-origin-for-your-cdn-resource/create-cdn-resource-10.png)</Frame>

2. In the **Origin** section, **Specify content origin** is selected by default.

3. Choose a connection type based on bucket visibility:

   - **Public bucket** (anonymous access allowed): In the **URL** tab, enter the bucket hostname in the **Origin source** field: `<bucket-name>.<location>.cloud.gcore.lu`.
   - **Private bucket** (authentication required): Select the **S3 storage** tab and enter the **Access Key ID**, **Secret access key**, **AWS region**, and **Bucket name**. A separate [private bucket guide](/cdn/cdn-resource-options/general/use-a-private-bucket-as-an-origin) covers the full setup.

<Frame>![CDN resource creation form with URL tab and Origin source field](/images/docs/storage/use-storage-as-the-origin-for-your-cdn-resource/image3527.png)</Frame>

4. (Optional) Specify a custom domain — the [setup guide](/cdn/cdn-resource-options/general/create-and-set-a-custom-domain-for-the-content-delivery-via-cdn) covers required DNS changes.

### Use SFTP Storage

1. Navigate to CDN resources and click **Create CDN resource**.

<Frame>![CDN resources list with the Create CDN resource button](/images/docs/storage/use-storage-as-the-origin-for-your-cdn-resource/create-cdn-resource-10.png)</Frame>

2. In the **Origin** section, select the **URL** tab and enter the origin source in the **Origin source** field using the format `<storage-name>.<hostname>`.

The storage name and hostname are available in **Object Storage** > **Object Storages** — open the three-dot actions menu for the storage and select **Details**.

<Frame>![SFTP storage Details panel showing the storage name and hostname](/images/docs/storage/use-storage-as-the-origin-for-your-cdn-resource/use-sftp-as-origin-40.png)</Frame>

For example, if the storage name is `12345-test` and the hostname is `ams.origin.gcdn.co`, enter `12345-test.ams.origin.gcdn.co` in the **Origin source** field.

3. (Optional) Specify a custom domain — the custom domain setup guide covers required DNS changes.

## Request content from CDN

URL patterns differ depending on whether the origin is Object Storage or SFTP Storage.

### Object Storage is the origin

The addressing scheme `<bucket-name>.<location>.cloud.gcore.lu` hides the storage hostname from CDN requests, keeping the origin private.

Use the following schema to request files through CDN:

`http(s)://<custom-domain>/<folder>/<file>`

If files are in the bucket root rather than a subfolder, omit `<folder>` from the URL.

Given a bucket `mybucket` at `s-ed1.cloud.gcore.lu` and a CDN resource with custom domain `cdn.example.com`, request `picture.png` using:

`http://cdn.example.com/picture.png`

<Note>
By default, buckets and all stored files are private. To make files public, configure [ACL&nbsp;rules](/storage/manage-object-storage/configure-aws-sli-s3cmd-and-aws-javascript-sdk/configure-access-control-on-s3-storage-with-aws-cli-and-s3cmd) using AWS CLI or S3cmd, or follow the [Amazon ACL](https://docs.aws.amazon.com/AmazonS3/latest/dev/acl-overview.html#canned-acl) documentation.
</Note>

### SFTP Storage is the origin

Upload files to the `htdocs` directory in the SFTP storage. When requesting content through CDN, omit the directory from the URL.

Path to a file in the storage:

`sftp://12345-test@ams.origin.gcdn.co:2200/export/home/12345-test/htdocs/picture.png`

If the CDN custom domain is `cdn.example.com`, the request URL is:

`http://cdn.example.com/picture.png`
