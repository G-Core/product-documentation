Object Storage and SFTP storage serve files over HTTP/HTTPS, but the URL schema differs between the two types.

## Request content directly from the object storage

Use the following schema to request uploaded files:

```sh
http(s)://<bucket-name>.<hostname>
```

The hostname is available on the [Storages](https://portal.gcore.com/storage/storages/list) tab in the **Details** section.

<Frame>![Storage list with the Details option open in the actions dropdown](/images/docs/storage/request-content-directly-from-the-storage/storage-list-details-menu.png)</Frame>

For example, consider a bucket `mybucket` with a file `picture.png` in the Luxembourg location. The Luxembourg location hostname is `s-ed1.cloud.gcore.lu`, so the URL to request the file is:

```sh
http://mybucket.s-ed1.cloud.gcore.lu/picture.png
```

Object Storage supports both HTTP and HTTPS protocols.

## Request content directly from the SFTP storage

Upload files to the `htdocs` directory, but don't include the directory name when requesting content.

Use the following schema to form the URLs:

```sh
https://<storage-name>.<hostname>/<file-name>
```

The storage name and hostname are available on the Storages tab in the **Details** section.

For example, an SFTP storage named `12345-test` at `ams.origin.gcdn.co` with a file uploaded at:

```sh
sftp://12345-test@ams.origin.gcdn.co:2200/export/home/12345-test/htdocs/picture.png
```

is accessible at:

```sh
https://12345-test.ams.origin.gcdn.co/picture.png
```

SFTP storage serves content over HTTPS.
