## FileZilla overview

FileZilla is an open-source SFTP client that supports both password and SSH key authentication for connecting to Gcore SFTP storage.

Download and install FileZilla from the [FileZilla website](https://filezilla-project.org), then follow the steps below to configure the connection.

## Site Manager setup

Open the FileZilla Site Manager by clicking the icon in the top-left corner of the toolbar:

<Frame>![FileZilla toolbar with arrow pointing to the Site Manager icon](/images/docs/storage/manage-sftp-storage/connect-to-your-storage-with-filezilla/13822230046353.png)</Frame>

In Site Manager, click **New Site**:

<Frame>![FileZilla Site Manager dialog with the New Site button highlighted](/images/docs/storage/manage-sftp-storage/connect-to-your-storage-with-filezilla/13814505201553.png)</Frame>

Enter a name for the site, then open the **General** tab on the right.

## Connection configuration

Both authentication methods use the same base configuration — only the **Logon Type** field and the **Key file** step differ:

| Field | Password | SSH key |
|-------|----------|---------|
| **Logon Type** | **Ask for password** | **Key file** |
| **Key file** | Not required | Path to the private key file on the local machine |

Configure the connection as follows:

1. In the **Protocol** field, select **SFTP - SSH File Transfer Protocol**.

2. In the **Host** field, enter the hostname from the **Details** panel in the [Gcore Customer Portal](https://portal.gcore.com). To open it: go to **Object Storages**, click **...** next to the storage name, then select **Details**.

3. In the **Port** field, enter `2200`.

4. Set **Logon Type** to **Ask for password** (password) or **Key file** (SSH key), as shown in the table above.

5. In the **User** field, enter the **Object Storage\User name** shown in the Details panel.

6. For SSH key authentication only: in the **Key file** field, specify the path to the private key file. To create and add an SSH key, see [SSH key setup](/storage/manage-sftp-storage/create-and-add-an-ssh-key-to-your-storage).

7. Click **OK** to save, then click **Connect** to open the connection.
