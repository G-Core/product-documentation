Storage usage metrics are available in the **Statistics** section of the [Gcore Customer Portal](https://portal.gcore.com). Gcore uses the following metrics for storage billing:

- Used space
- Requests
- Traffic
- Number of objects

The page supports filtering by time period (last 24 hours, last 48 hours, last week, last month, last year, or a custom date range), storage type (S3 or SFTP), location, and storage name.

<Frame>
  ![Statistics page showing storage usage graphs with metric tabs and filter options](/images/docs/storage/check-storages-usage-reports/Screenshot_70.png)
</Frame>

Data appears in the portal within a couple of hours after storage creation. The graphs display aggregated values — used space is measured every hour and shown as max and average usage for the selected period. Requests for longer periods — a year of data — may take more time to process.

For SFTP storage, system files — SSH keys, empty directories, and a `robots.txt` file — account for about 350 kB of used space. If a storage was created but no files have been uploaded, statistics will reflect only this system-file space. Gcore rounds used space to the nearest whole GB for invoicing.
