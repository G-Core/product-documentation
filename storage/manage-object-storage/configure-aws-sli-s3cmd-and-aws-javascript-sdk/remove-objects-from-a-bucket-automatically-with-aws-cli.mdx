## Lifecycle policy overview

A lifecycle policy is an XML or JSON file containing rules that automate object removal based on the object's age and other parameters, following the [lifecycle configuration](https://docs.aws.amazon.com/AmazonS3/latest/userguide/how-to-set-lifecycle-configuration-intro.html) standard. Gcore supports lifecycle policies where an expiration time can be set; objects are automatically deleted from the bucket once that time is reached.

## Lifecycle policy logic

Object removal runs around midnight UTC. Three states determine when an object is removed:

1. **Lifecycle policy set before upload.** If the policy is set for 1 day and the object is uploaded on January 2, it is removed on January 4.

2. **Lifecycle policy set after upload.** If an object is uploaded on January 1 (at any time) and a 1-day expiration policy is then applied, the object is deleted on January 3.

3. **Lifecycle policy removed.** Deleting the lifecycle policy from the bucket stops automatic removal — objects already in the bucket are kept.

## Gcore lifecycle configuration elements

A Gcore lifecycle configuration file looks like this:

```xml
<LifecycleConfiguration>
  <Rule>
    <ID>...</ID>
    <Prefix/>
    <Status>Enabled</Status>
    <Expiration>
      <Days>...</Days>
    </Expiration>
  </Rule>
</LifecycleConfiguration>
```

| Element | Description |
|---------|-------------|
| `<Rule>` | Wraps all settings for one rule. |
| `<ID>` | Unique rule identifier — up to 255 characters, letters, digits, and underscores. |
| `<Prefix/>` | Leave empty to apply the rule to the entire bucket; specify a folder name to scope the rule to that folder only. |
| `<Status>` | `Enabled` activates the rule; `Disabled` suspends it without deleting it. |
| `<Expiration><Days>` | Number of days after which objects are automatically deleted. |

## Manage a lifecycle policy with AWS CLI

AWS CLI provides three commands for managing lifecycle policies: `put-bucket-lifecycle` to apply a policy, `get-bucket-lifecycle-configuration` to verify it, and `delete-bucket-lifecycle` to remove it.

### Add a lifecycle policy

The following steps apply a lifecycle policy to an Object Storage bucket using a JSON configuration file.

1. Create a `lifecycle.json` file with the following content:

```json
{
  "Rules": [
    {
      "ID": "one_day",
      "Prefix": "",
      "Status": "Enabled",
      "Expiration": {
        "Days": 1
      }
    }
  ]
}
```

where:

- `one_day` — rule ID,
- `""` (empty prefix) — the rule applies to the entire bucket,
- `"Enabled"` — the rule is active,
- `1` — objects are deleted after 1 day.

<Note>
To apply the rule to a specific folder, set `"Prefix"` to the folder name — for example, `"Prefix": "deleteme/"`.
</Note>

2. Run the following command from the directory containing `lifecycle.json`, replacing the placeholder values:

```sh
aws s3api put-bucket-lifecycle --bucket my-bucket --lifecycle-configuration file://lifecycle.json --endpoint-url=https://<storage-hostname>
```

where:

- `my-bucket` — the name of the bucket,
- `<storage-hostname>` — the storage endpoint hostname, found in the **Details** section of the storage in the [Gcore Customer Portal](https://portal.gcore.com); see [S3 URLs](/storage/manage-object-storage/s3-service-urls-and-default-region-names) for all available hostnames.

3. Verify the policy was applied:

```sh
aws s3api get-bucket-lifecycle-configuration --bucket my-bucket --endpoint-url=https://<storage-hostname>
```

The response contains the applied policy as JSON.

### Delete a lifecycle policy

Run the following command to remove the lifecycle policy from the bucket:

```sh
aws s3api delete-bucket-lifecycle --bucket my-bucket --endpoint-url=https://<storage-hostname>
```

where `my-bucket` and `<storage-hostname>` are the same values used when applying the policy.
