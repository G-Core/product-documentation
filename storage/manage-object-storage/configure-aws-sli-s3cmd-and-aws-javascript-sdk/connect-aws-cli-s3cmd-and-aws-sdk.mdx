[Gcore Object Storage](https://gcore.com/storage) is compatible with S3-based tools including AWS CLI and S3cmd. The sections below cover installing and configuring each tool to connect to Gcore Object Storage.

## AWS CLI

AWS CLI, or the AWS Command Line Interface, is the software for managing AWS services. It integrates with Gcore Object Storage for bucket management using commands from [Amazon's official documentation](https://docs.aws.amazon.com/cli/latest/reference).

### Install AWS CLI

1. Follow the instructions in the [Amazon article](https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html) to install the latest version of the AWS CLI.

2. To verify that the installation was successful, run the `aws --version` command in the terminal.

```sh
aws --version
```

The output shows the installed version and operating system. For example:

```sh
aws-cli/2.7.24 Python/3.8.8 Windows/10 exe/AMD64 prompt/off
```

If the terminal cannot find the `aws` command, refer to the [Troubleshooting AWS CLI errors](https://docs.aws.amazon.com/cli/latest/userguide/cli-chap-troubleshooting.html) guide.

### Configure your storage for use with AWS CLI

1. Run the `aws configure` command to launch the configuration wizard.

2. Provide the following information:

  * **Access Key**: Enter the access key from the storage Details dialog.
  * **Secret Key**: Enter the secret key from the storage Details dialog.
  * **Default region name**: Enter your [storage region](/storage/manage-object-storage/s3-service-urls-and-default-region-names), for example, `s-ed1`.
  * Leave all other parameters at their default values.

3. To verify the configuration, run the following command:

```sh
aws s3 ls --endpoint-url=https://s-ed1.cloud.gcore.lu
```

Replace `s-ed1.cloud.gcore.lu` with the actual storage endpoint from the [S3 service URLs and default region names](/storage/manage-object-storage/s3-service-urls-and-default-region-names) article.

If the credentials are correct, the command exits without errors and lists any existing buckets. An empty output with exit code 0 is expected for a new storage with no buckets.

## S3cmd

S3cmd is a free command line tool for uploading, retrieving, and managing data in Amazon S3 and other cloud storage services. For the full command reference, see the [official S3cmd documentation](https://s3tools.org/usage).

### Install S3cmd

1. Go to the [Download](https://s3tools.org/download) section in the S3cmd documentation.

2. Download the package for your operating system.

3. Install the latest version of S3cmd.

### Configure your storage for use with S3cmd

#### Configure with the wizard

1. Run the following command to launch the configuration wizard:

```sh
s3cmd --configure
```

2. Specify the following information:

  * **Access Key**: Enter the access key from the storage Details dialog.
  * **Secret Key**: Enter the secret key from the storage Details dialog.
  * **S3 Endpoint**: Enter the [storage endpoint](/storage/manage-object-storage/s3-service-urls-and-default-region-names), for example, `s-ed1.cloud.gcore.lu`.
  * **DNS-style bucket+hostname**: Enter the storage endpoint again, for example, `s-ed1.cloud.gcore.lu`. Optionally append the port: `s-ed1.cloud.gcore.lu:80`.
  * Leave all other parameters at their default values.

S3cmd will attempt to connect to the storage. If the credentials are correct, the output shows:

```sh
Success. Your access key and secret key worked fine :-)
```

S3cmd saves the configuration to `~/.s3cfg` in the following format:

```toml
[default]
access_key = 123ABC456DEF...
secret_key = EXAMPLE...
host_base = s-ed1.cloud.gcore.lu
host_bucket = s-ed1.cloud.gcore.lu
```

#### Configure without the wizard

Use this method for scripted or automated environments where interactive input is not available. The command writes the configuration directly to `~/.s3cfg`:

```sh
s3cmd --access_key 12*****6DEF --secret_key EX***** --host s-ed1.cloud.gcore.lu --host-bucket s-ed1.cloud.gcore.lu
```

Where:

  * `12*****6DEF` is the access key from the storage Details dialog.
  * `EX*****` is the secret key from the storage Details dialog.
  * `s-ed1.cloud.gcore.lu` is the [storage endpoint](/storage/manage-object-storage/s3-service-urls-and-default-region-names).

## AWS JavaScript SDK

<Note>The examples below use AWS SDK for JavaScript v2 (`aws-sdk`). AWS has released SDK v3 (`@aws-sdk/client-s3`) as the current recommended version. The v2 SDK is in maintenance mode but still functional. For new projects, see the [AWS SDK for JavaScript v3 migration guide](https://docs.aws.amazon.com/sdk-for-javascript/v3/developer-guide/migrating-to-v3.html).</Note>

### Connect AWS JavaScript SDK

To connect the interface with your storage, add the AWS SDK to your HTML page:

```html
<html>
    <head>
        <script src="https://sdk.amazonaws.com/js/aws-sdk-2.742.0.min.js"></script>
        <script src="./js/index2.js"></script>
    </head>
    <body>
        <h1>List of files</h1>
        <ul id="list">
        </ul>
    </body>
</html>
```

### Manage storage with AWS JavaScript SDK

#### Configure storage and create an S3 bucket

Open the configuration file (`./js/index2.js`) and specify the storage credentials and bucket name:

```js
var s3BucketName = "test";
var host = "https://s-ed1.cloud.gcore.lu";
var access_key = "1234";
var secret_key = "5678";

AWS.config.accessKeyId = access_key;
AWS.config.secretAccessKey = secret_key;
AWS.config.endpoint = host;

var s3 = new AWS.S3({
    sslEnabled: true
});
```

Where:

  * `test` is the bucket name;
  * `https://s-ed1.cloud.gcore.lu` is the [storage endpoint](/storage/manage-object-storage/s3-service-urls-and-default-region-names);
  * `1234` is the access key from the Gcore Customer Portal.
  * `5678` is the secret key from the Gcore Customer Portal.

#### Set CORS policy on a bucket

Below is a wildcard CORS policy that allows cross-origin GET, HEAD, PUT, POST, and DELETE requests from all sources.

<Note>This configuration is permissive and may be insecure for production environments.</Note>

```xml
<CORSConfiguration>
<CORSRule>
    <ID>Allow everything</ID>
    <AllowedOrigin>*</AllowedOrigin>
    <AllowedMethod>GET</AllowedMethod>
    <AllowedMethod>HEAD</AllowedMethod>
    <AllowedMethod>PUT</AllowedMethod>
    <AllowedMethod>POST</AllowedMethod>
    <AllowedMethod>DELETE</AllowedMethod>
    <AllowedHeader>*</AllowedHeader>
    <MaxAgeSeconds>30</MaxAgeSeconds>
</CORSRule>
</CORSConfiguration>
```

After creating the policy file, apply it to the bucket:

```sh
s3cmd setcors cors.xml s3://<bucket_name>
```

#### Add objects to a bucket

The example adds two objects (`test_file1` and `test_file2`) to the `s3test` bucket.

```js
var params1 = {
   Bucket: "s3test", Key: "test_file1",
   Body: "test"
};
var params2 = {
   Bucket: "s3test", Key: "test_file2",
   Body: "test"
};
var request = s3.putObject(params1);
request.send(function (err, data) {
   if (err) console.log("Error:", err.code, err.message);
   else console.log(data);
});
var request = s3.putObject(params2);
request.send(function (err, data) {
   if (err) console.log("Error:", err.code, err.message);
   else console.log(data);
});
```

#### Get a list of objects

The example retrieves a list of objects in the `test_2` bucket.

```js
params = {
    Bucket: "test_2"
};
s3.listObjects(params, function(err, data) {
    if (err) return;
    data.Contents.map(function(info) {
        var ul = document.getElementById("list");
        var li = document.createElement("li");
        li.innerText = info.Key + " " + info.LastModified;
        ul.append(li);
    });
});
```
