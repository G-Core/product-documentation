Gcore Object Storage organizes files into buckets. This article covers how to create buckets, manage files using the built-in file manager, control public access, and configure lifecycle policies through the customer portal.

## Create a bucket

1. Navigate to the [Object Storage](https://portal.gcore.com/storage/storages/list) section. Click the relevant storage name.


<Frame>![Open the Buckets interface](/images/docs/storage/manage-object-storage/manage-buckets-via-the-control-panel/manage-buckets-cp-10.png)</Frame>



Alternatively, open the three-dot menu on the storage row and select **Buckets**.

2. Click **Add new bucket**.


<Frame>![Add new bucket](/images/docs/storage/manage-object-storage/manage-buckets-via-the-control-panel/manage-buckets-cp-20.png)</Frame>



3. A dialog will appear. Enter a name for the bucket that meets the following criteria:

  * Between 3 to 63 characters in length.
  * Lowercase only.
  * No underscores, trailing dashes, consecutive dots, or a mix of dots and dashes; they conflict with DNS notation rules.
  * The bucket name must be unique within the entire Gcore Object Storage system, not just your account. If a name is already used, you'll receive a `This bucket name already exists. Please use a different name` error message.



Click **Create**.

<Note>Avoid using sensitive information in the bucket name. The bucket name appears in object URLs and is publicly visible.</Note>

After creating the bucket, it is ready for the following operations:

  * [File management](/storage/manage-object-storage/manage-buckets-via-the-control-panel#file-manager) — uploading, downloading, and organizing files
  * [Public HTTP access](/storage/manage-object-storage/manage-buckets-via-the-control-panel#managing-access) — making files accessible via a direct URL
  * [Lifecycle policy](/storage/manage-object-storage/manage-buckets-via-the-control-panel#adding-lifecycle-policy-for-s3-in-luxembourg-only) — automatic deletion of objects after a set period (S3 Luxembourg storages only)



## File manager

### Access (Override CORS)

To access the file manager, the portal origin `https://portal.gcore.com` must be added to the bucket's CORS policy. This is a one-time setup per bucket.

Click the bucket name to open the access settings dialog, then click **Override CORS**.

<Note>CORS must be configured individually for each bucket.</Note>


<Frame>![Override CORS](/images/docs/storage/manage-object-storage/manage-buckets-via-the-control-panel/manage-buckets-cp-30.gif)</Frame>



CORS can also be configured via the [API](/api-reference/storage/storage/create-s3-bucket-cors), which allows adding multiple allowed origins beyond `https://portal.gcore.com`.

### Authorization

To authorize, click on the bucket name, enter the Access key and Secret key, then click **Auth**.


<Frame>![Authorize to file manager](/images/docs/storage/manage-object-storage/manage-buckets-via-the-control-panel/manage-buckets-cp-40.gif)</Frame>



The file manager is also accessible from the bucket three-dot menu by selecting **File manager**.


<Accordion title="Access key and Secret key features">
<Note>Entered keys are saved for the duration of the session and do not need to be re-entered. Keys are requested again at the start of each new session.</Note>

To avoid re-entering keys manually, save them using the browser's credential manager.


<Frame>![Manage passwords browser functionality](/images/docs/storage/manage-object-storage/manage-buckets-via-the-control-panel/manage-buckets-cp-50.png)</Frame>



The Access key and Secret key are generated during storage creation. Gcore does not store these values. If the keys are lost, click **Generate new keys** in the [Object Storages](https://portal.gcore.com/storage/storages/list) list using the three-dot menu.


<Frame>![Object storage three-dot menu with the Generate new keys option](/images/docs/storage/manage-object-storage/manage-buckets-via-the-control-panel/manage-buckets-cp-60.png)</Frame>


</Accordion>


### Adding folders

After [authorizing in the file manager](/storage/manage-object-storage/manage-buckets-via-the-control-panel#authorization), click **Add folder**. Enter a name in the field and click **Create**.


<Frame>![Creation of folders](/images/docs/storage/manage-object-storage/manage-buckets-via-the-control-panel/manage-buckets-cp-70.gif)</Frame>



The interface shows the last modification date for each folder. Click a folder name to open it.

### Uploading files

To upload files, [authorize access to your bucket](/storage/manage-object-storage/manage-buckets-via-the-control-panel#authorization) and click **Select and upload file(s)** either in the bucket root or within a specific folder. Then, follow the standard upload process.


<Frame>![Uploading of files](/images/docs/storage/manage-object-storage/manage-buckets-via-the-control-panel/manage-buckets-cp-80.png)</Frame>



### Deleting folders and files

To delete folders or files, [authorize access to your bucket](/storage/manage-object-storage/manage-buckets-via-the-control-panel#authorization) and click **Delete** next to the desired object. Alternatively, select the checkboxes next to the object names and click **Delete selected**.


<Frame>![Deletion of files](/images/docs/storage/manage-object-storage/manage-buckets-via-the-control-panel/manage-buckets-cp-90.png)</Frame>



<Note>Deleting a folder removes all files nested within it.</Note>

### Copying URL or object storage URL

To copy the links of certain files, [authorize access to your bucket](/storage/manage-object-storage/manage-buckets-via-the-control-panel#authorization), select objects and click the relevant buttons: **Copy S3 URL** or **Copy URL**.


<Frame>![Copy S3 URL and Copy URL](/images/docs/storage/manage-object-storage/manage-buckets-via-the-control-panel/manage-buckets-cp-100.png)</Frame>



For instance, the links for the file `image 3972.png` will look like this:

  * `s3://s-ed1.cloud.gcore.lu/000-sample-for-articles/test/image%3972.png`
  * `https://s-ed1.cloud.gcore.lu/000-sample-for-articles/test/image%3972.png`



<Note>If public HTTP access is not enabled for the bucket, accessing the file URL returns a 403 error. See [Managing access](/storage/manage-object-storage/manage-buckets-via-the-control-panel#managing-access).</Note>

## Managing access

To use [Gcore Object Storage as the origin for a CDN resource](/storage/use-storage-as-the-origin-for-your-cdn-resource) or to make files directly accessible via an Object Storage URL, enable public HTTP access. Open the bucket interface, click the three dots to the right, select **Public access to all files**, and confirm by clicking **Apply**.


<Frame>![Confirmation dialog for enabling public HTTP access to all files in the bucket](/images/docs/storage/manage-object-storage/manage-buckets-via-the-control-panel/http-access-s3-bucket.png)</Frame>



To revert the bucket to private access, go to the bucket interface, click the three dots to the right, and click **Default access to all files**.


<Frame>![Confirmation dialog for reverting the bucket to default private access](/images/docs/storage/manage-object-storage/manage-buckets-via-the-control-panel/default-access-s3-bucket.png)</Frame>



## Adding lifecycle policy (for S3 in Luxembourg only)

To add a lifecycle policy, navigate to the bucket interface, click the three dots to the right, and select **Lifecycle Management**. Set how long files should be retained before deletion and click **Save changes**.


<Frame>![Set lifecycle policy](/images/docs/storage/manage-object-storage/manage-buckets-via-the-control-panel/manage-buckets-cp-120.gif)</Frame>



To remove an existing policy, click **Cancel policy**.

For storages outside the S3 Luxembourg location, lifecycle policies can be configured using the AWS CLI tool, as outlined in the [separate guide](/storage/manage-object-storage/configure-aws-sli-s3cmd-and-aws-javascript-sdk/remove-objects-from-a-bucket-automatically-with-aws-cli).
