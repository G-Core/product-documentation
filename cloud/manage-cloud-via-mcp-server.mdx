Gcore MCP Server enables AI assistants to manage Cloud infrastructure through natural language commands. This guide covers Cloud-specific tools and example conversations.

For general setup instructions, see [Gcore MCP Server overview](/account-settings/integrations/gcore-mcp-server-overview).

## Configure for Cloud

Set the `GCORE_TOOLS` environment variable to expose Cloud tools:

```bash
# All cloud tools (approximately 400 tools)
GCORE_TOOLS="cloud.*"

# Or use predefined toolsets for specific tasks
GCORE_TOOLS="management,instances,storage,networking"
```

For focused workflows, use specific toolsets:
- `instances` - Virtual machine operations
- `storage` - Volumes and file shares
- `networking` - Networks, floating IPs, load balancers
- `security` - Security groups, SSH keys

## Available Cloud tools

### Compute

| Resource | Operations | Description |
|----------|------------|-------------|
| `instances` | list, create, get, delete, start, stop, resize | Virtual machines |
| `baremetal` | list, create, get, delete, power_on, power_off | Bare metal servers |
| `gpu_baremetal` | clusters.list, clusters.create, clusters.delete | GPU bare metal clusters |
| `gpu_virtual` | clusters.list, clusters.create, clusters.delete | GPU virtual clusters |

### Storage

| Resource | Operations | Description |
|----------|------------|-------------|
| `volumes` | list, create, get, delete, attach, detach | Block storage |
| `volume_snapshots` | list, create, get, delete | Volume snapshots |
| `file_shares` | list, create, get, delete | Shared file systems |

### Networking

| Resource | Operations | Description |
|----------|------------|-------------|
| `networks` | list, create, get, delete | Virtual networks |
| `floating_ips` | list, create, get, delete, assign, unassign | Public IP addresses |
| `load_balancers` | list, create, get, delete, update | Load balancers |
| `security_groups` | list, create, get, delete, add_rule | Firewall rules |

### Management

| Resource | Operations | Description |
|----------|------------|-------------|
| `projects` | list, get | Cloud projects |
| `regions` | list, get | Available regions |
| `quotas` | get | Resource quotas |
| `ssh_keys` | list, create, get, delete | SSH public keys |

## Example conversations

### List projects and regions

**Prompt:**
```
List all my cloud projects
```

**Response:**
```
Found 3 projects:

1. production (ID: 780356)
2. staging (ID: 780412)
3. development (ID: 780489)
```

**Prompt:**
```
Show available regions
```

### Create a virtual machine

**Prompt:**
```
Create a virtual machine in project 780356 with 4 vCPUs, 8 GB RAM, 
Ubuntu 22.04, and 80 GB SSD in Luxembourg region
```

**Response:**
```
Created virtual machine:
- Name: vm-1
- ID: 12345
- Flavor: g1-standard-4-8
- Image: Ubuntu 22.04 LTS
- Volume: 80 GB SSD
- Region: Luxembourg
- Status: spawning

The instance will be ready in approximately 2 minutes.
```

### Manage instances

**Prompt:**
```
List all running instances in project 780356
```

**Prompt:**
```
Stop instance 12345
```

**Prompt:**
```
Resize instance 12345 to 8 vCPUs and 16 GB RAM
```

### Storage operations

**Prompt:**
```
Create a 100 GB SSD volume named "data-volume" in project 780356, Luxembourg region
```

**Prompt:**
```
Attach volume data-volume to instance 12345
```

**Prompt:**
```
Create a snapshot of volume data-volume
```

### Networking

**Prompt:**
```
List all floating IPs in project 780356
```

**Prompt:**
```
Create a floating IP and assign it to instance 12345
```

**Prompt:**
```
Create a load balancer for instances 12345 and 12346 on port 80
```

### Security groups

**Prompt:**
```
Create a security group that allows SSH (port 22) and HTTP (port 80) from anywhere
```

**Prompt:**
```
Add a rule to security group 5678 allowing HTTPS (port 443)
```

## Common workflows

### Deploy a web server

**Combined prompt:**
```
In project 780356, Luxembourg region:
1. Create a VM with 2 vCPUs, 4 GB RAM, Ubuntu 22.04
2. Create a floating IP and assign it
3. Create a security group allowing SSH, HTTP, and HTTPS
4. Apply the security group to the VM
```

### Set up a database server

**Prompt:**
```
Create a database server setup:
- VM with 4 vCPUs, 16 GB RAM
- 500 GB SSD data volume
- Private network only (no floating IP)
- Security group allowing PostgreSQL port 5432 from internal network only
```

### Clean up unused resources

**Prompt:**
```
List all unattached volumes in project 780356
```

**Prompt:**
```
Find floating IPs that are not assigned to any instance
```

## Environment variables

For Cloud operations, you can set default project and region:

```bash
GCORE_CLOUD_PROJECT_ID="780356"
GCORE_CLOUD_REGION_ID="76"
```

This eliminates the need to specify project and region in every request.
