Cloud resource usage and associated costs are available on the following pages: 

  * **Cost Report**: costs for Cloud resources based on the Pay-As-You-Go pricing model. This report doesn't include costs associated with prepaid resources (active commit features). 
  * **Reservation Cost Report**: monthly resource usage and costs for both consumed and pre-paid resources.

Reports are also available via [API](https://api.gcore.com/docs/cloud#tag/Cost-Reports/operation/CostReportResourcesHandler.post).

<Info>
**Info**

Statistics for the last hour might contain incomplete data. For accuracy, review cost information collected more than an hour ago.
</Info> 

## Cost Report

The Cost Report provides data about each Cloud resource, allowing more accurate forecasting of pay-as-you-go expenses. 

To open the Cost Report page, go to **Cloud** > **Cloud Management** > **Cost Report**.

<Info>
**Info**

This report doesn't contain information about prepaid resources. For accounts with resource reservations, check the [Reservation Cost Report](/cloud/getting-started/view-statistics-on-expenses#reservation-cost-report) for accurate cost data.
</Info>

The cost data can be filtered by: 

  * **Project**: costs for all projects or selected ones. 

  * **Region**: costs for products created in all regions or selected ones. 

  * **Resource type**: costs for selected resource types. 

  * **Tags**: resources filtered by assigned tags.

  * **Consumption period**: costs for a week, two weeks, or a custom period. The maximum selectable period is one month.

### GPU Cloud resources

GPU Cloud resources appear in the Cost Report with the following resource types:

  * **Virtual GPU Cluster**: GPU-enabled virtual machines
  * **Bare Metal GPU Cluster**: dedicated GPU servers
  * **Inference**: AI inference endpoints

To view GPU-related costs, use the **Resource type** filter and select the relevant GPU resource types. The same filtering, export, and analysis features apply to GPU resources as to other Cloud resources.

### Deleted projects

The report also contains statistics for deleted projects and regions. Deleted projects are displayed in the Projects dropdown with a **(Deleted)** label next to their names, allowing inclusion in cost analysis.


<Frame>![Projects dropdown with deleted projects](/images/docs/cloud/getting-started/view-statistics-on-expenses/projects-dropdown.png)</Frame>



Cost statistics is displayed in a graph and table format. In the graph view, the **Total** tab displays the cumulative costs associated with each Cloud resource. 


<Frame>![Cost report graph with resource usage](/images/docs/cloud/getting-started/view-statistics-on-expenses/cost-report-graph.png)</Frame>



More granular statistics are available on the following tabs: 

  * **Total $** : cumulative costs for all resources
  * **Hours** : resource usage measured in hours 
  * **Gigabyte-hours** : storage and memory usage over time
  * **Gigabyte-minutes** : granular storage usage by resources 
  * **Gigabyte-seconds** : most granular storage usage measurement
  * **Millions** : usage measured in millions of units (e.g., API requests)
  * **Gigabytes** : total gigabytes of used resources 
  * **Flavors, hours** : usage hours broken down by VM flavors



Under the graph, there is a table with more detailed information about resources consumed within a selected period. The table features the following resource properties: 

  * Resource name 
  * Resource type 
  * Region 
  * Project 
  * Tags
  * Usage 
  * First seen date 
  * Last seen date 
  * Cost


<Frame>![A table with resources overview](/images/docs/cloud/getting-started/view-statistics-on-expenses/cost-report-search-table.png)</Frame>

### Download Cost Report data

To export the billing report for the selected period, click **Export CSV** in the top-right corner of the screen. Two file types are available: 

  * **Detailed report**: a detailed breakdown of each resource in use. 

  * **Totals**: the total cost of resources in each location.

### Cost optimization tips

To effectively manage cloud spending using Cost Report data:

- **Monitor regularly**: Review cost reports weekly to identify unexpected spending spikes before they accumulate.
- **Use filters strategically**: Filter by project or resource type to isolate high-cost areas and identify optimization opportunities.
- **Track deleted resources**: Include deleted projects in the analysis to understand historical spending patterns and avoid recreating costly configurations.
- **Export for analysis**: Download detailed CSV reports for deeper analysis in spreadsheet tools or integration with financial systems.
- **Compare periods**: Use custom date ranges to compare spending across different time periods and identify trends.

## Reservation Cost Report

This report presents information about prepaid and [reserved](/cloud/getting-started/resource-reservation/about-resource-reservation) resources, active commits, and costs for pay-as-you-go services. 

To open the Reservation Cost Report page, go to **Cloud** > **Cloud Management** > **Reservation Cost Report**.

Unlike the Cost Report, the Reservation Cost Report doesn't show statistics for individual resources—total data is displayed for each month. 

Cost data can be filtered by: 

  * **Region**: costs for products created in all regions or selected ones. 
  * **Resource type**: costs for selected resource types. 
  * **Consumption period**: costs for a particular month.

Cost data is available both as a chart and as a table with more detailed information. In the chart view, the **Total** tab displays cumulative costs associated with reserved Cloud resources. 


<Frame>![Reservation cost report graph](/images/docs/cloud/getting-started/view-statistics-on-expenses/reservation-cost-report.png)</Frame>



More granular information is available on the following tabs: 

  * **Total $** : cumulative costs for reserved resources
  * **Hours** : resource usage in hours
  * **Gigabyte-hours** : storage and memory usage over time
  * **Gigabyte-minutes** : granular storage usage
  * **Gigabyte-seconds** : most granular storage measurement
  * **Millions** : usage in millions of units
  * **Gigabytes** : total gigabytes used
  * **Flavors, hours** : usage by VM flavors
  * **Commit usage** : reserved resource consumption



Under the chart, there is a table with a detailed report of resources consumed during the selected timeframe. 

  * Resource type 
  * Region 
  * Commit: the number of reserved resources (active commitments) 
  * Usage 
  * Commit cost: fixed monthly costs for resources according to the plan. 
  * Overcommit cost: costs for overuse of resources within the plan. 
  * Total cost



### Download Reservation Cost Report data

To export the billing report for the selected period, click **Export totals CSV** in the top-right corner of the screen.


<Frame>![Export reservation cost report button](/images/docs/cloud/getting-started/view-statistics-on-expenses/reservation-cost-report-export-csv.png)</Frame>



The following table explains what information is included in the **Totals** file CSV report file to reserved resources. 

**Column** | **Description**  
---|---  
Type | The reserved Cloud resource type.  
Feature | If a resource has multiple configurations (flavors), such as [different volume types](/cloud/virtual-instances/volumes/about-volumes#available-volume-types), this column specifies the configuration.  
Region_id | ID of the region where the resource was reserved. Available via [API](https://api.gcore.com/docs/cloud#tag/Regions).  
Region_name | Geographical location of the data center where the resource was reserved.  
Period | The timeframe for the displayed statistics.  
Units | The measurement unit. Represented as usage time (gbminutes, minutes, milliseconds) or consumed resources (bytes, GBS).  
Value | Total consumed value.  
Commit value | Total amount of resources that have been reserved.  
Commit cost | The cost for a reserved amount of cloud resources.  
Overcommit cost | Additional charges incurred after the resource usage exceeded the committed (reserved) amount.  
Error | Any errors associated with the reserved resource during the specified period.  
Currency | The type of currency in which the cost is calculated.
