All operations below are performed from the **Access** tab in Project Settings. Only users with the Client Administrator or Project Administrator role can make these changes.

## Edit user roles

1. In the [Gcore Customer Portal](https://portal.gcore.com), navigate to **Cloud** > **Project Settings** and open the **Access** tab.

2. Click **Edit** next to the user.


<Frame>![Access tab showing the Edit button next to a project user](/images/docs/cloud/getting-started/projects/users/manage-user-access-to-your-project/edit-users.png)</Frame>



3. In the **Role** column, select a new role from the dropdown: Observer, Project Administrator, User, or Internal Network Only User.


<Frame>![Role dropdown open showing available project roles](/images/docs/cloud/getting-started/projects/users/manage-user-access-to-your-project/edit-user-actions.png)</Frame>



4. Click **Save**.

5. In the **Reassign user role** dialog, click **Assign** to confirm.

## Deny access to a project

Denying access removes the user's role without removing the user from the project.

1. In the **Access** tab, click **Edit** next to the user, then click **Delete role**.


<Frame>![Access tab with the Delete role button highlighted](/images/docs/cloud/getting-started/projects/users/manage-user-access-to-your-project/delete-user-actions.png)</Frame>



2. Confirm the deletion. The user's **Role** column changes to **No access**.


<Frame>![Access tab showing No access in the Role column](/images/docs/cloud/getting-started/projects/users/manage-user-access-to-your-project/user-no-access.png)</Frame>



## Restore user access to a project

To restore access for a user whose role shows as No access, assign a new role.

1. In the **Access** tab, click **Edit** next to the user.


<Frame>![Access tab showing the Edit button next to a user with No access role](/images/docs/cloud/getting-started/projects/users/manage-user-access-to-your-project/user-no-access-edit.png)</Frame>



2. Select the required role and click **Save**.

3. In the **Reassign user role** dialog, click **Assign** to confirm.


<Frame>![Reassign user role confirmation dialog with Assign button](/images/docs/cloud/getting-started/projects/users/manage-user-access-to-your-project/reassign-role.png)</Frame>



The user regains access to the project.
