Tags are key-value pairs that you can assign to Cloud resources. They help you organize, identify, and manage resources across your infrastructure.

## Use cases

Tags provide a flexible way to categorize resources based on your organizational needs:

| Use case | Example tags |
|----------|--------------|
| **Group by service** | `service:authentication`, `service:payment`, `service:frontend` |
| **Identify environment** | `env:production`, `env:staging`, `env:development` |
| **Track ownership** | `team:backend`, `owner:john.doe` |
| **Cost allocation** | `cost-center:marketing`, `project:website-redesign` |

When you add a tag like `env:production` to a Load Balancer, you can quickly identify that resource belongs to your production environment. This labeling helps when managing multiple resources across different projects or teams.

## Tag restrictions

Tags must follow these formatting rules:

| Parameter | Requirement |
|-----------|-------------|
| Key length | 3-255 characters |
| Value length | 3-255 characters |
| Forbidden characters | `=` in keys |
| Whitespace | Leading and trailing spaces are trimmed |

<Warning>
Tag keys cannot contain the `=` character. This restriction exists because Gcore uses internal encoding that relies on this character as a separator.
</Warning>

## Add tags in the Customer Portal

You can add tags when creating a new resource or update them on existing resources.

### Add tags during resource creation

1. Navigate to the resource creation page (for example, **Cloud** > **Virtual Instances** > **Create Instance**).

2. In the **Additional options** section, turn on the **Add tags** toggle.

3. Enter a **Key** and **Value** for each tag.

4. Click **Add tag** to add more tags if needed.

5. Complete the resource creation.

### Add tags to existing resources

The process varies slightly depending on the resource type. For Load Balancers:

1. Navigate to **Cloud** > **Networking** > **Load Balancers**.

2. Click the Load Balancer name to open its settings.

3. Navigate to the **Tags** tab.

4. Add or modify tags as needed.

5. Click **Save**.

For other resources, look for the **Tags** section in the resource settings or details page.

## Add tags via API

When creating or updating resources via the [Gcore API](/api-reference/overview), include tags in the `metadata` field:

```json
{
  "name": "my-instance",
  "metadata": {
    "env": "production",
    "team": "backend"
  }
}
```

## Add tags via Terraform

When using the [Gcore Terraform provider](/cloud/manage-cloud-via-terraform), specify tags in the `metadata_map` attribute:

```hcl
resource "gcore_instance" "example" {
  name        = "my-instance"
  flavor_id   = "g1-standard-2-4"
  
  metadata_map = {
    env  = "production"
    team = "backend"
  }
}
```
