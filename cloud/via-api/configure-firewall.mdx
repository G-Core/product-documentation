A security group acts as a firewall for virtual machines, controlling which inbound and outbound traffic is allowed. This guide creates a security group, adds rules for common services, and attaches it to a VM in 4 API calls.

<Info>
A permanent [API token](/developer-tools/rest-api/authentication) is required, along with a project ID and region ID. Retrieve them from the [Gcore Customer Portal](https://portal.gcore.com) or by calling <code>GET&nbsp;/cloud/v1/projects</code> and <code>GET&nbsp;/cloud/v1/regions</code>.
</Info>

```bash
export GCORE_API_KEY="{YOUR_API_KEY}"
export PROJECT_ID="{YOUR_PROJECT_ID}"
export REGION_ID="{YOUR_REGION_ID}"
```

## Step 1. Create a security group

A new security group starts empty for inbound traffic and includes a full set of default egress rules covering all protocols. Add inbound rules in the next step.

| Parameter | Required | Description |
|-----------|----------|-------------|
| `name` | Yes | Used later to attach the group to a VM. |

<Tabs>
  <Tab title="curl">
    ```bash
    curl -X POST "https://api.gcore.com/cloud/v2/security_groups/$PROJECT_ID/$REGION_ID" \
      -H "Authorization: APIKey $GCORE_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"name": "web-firewall"}'
    ```
  </Tab>
  <Tab title="Python SDK">
    ```python
    import os
    from gcore import Gcore

    client = Gcore(api_key=os.environ["GCORE_API_KEY"])
    r = client.cloud.security_groups.create(
        project_id=int(os.environ["PROJECT_ID"]),
        region_id=int(os.environ["REGION_ID"]),
        name="web-firewall",
    )
    task = client.cloud.tasks.poll(r.tasks[0], polling_interval_seconds=3,
                                   polling_timeout_seconds=30)
    sg_id = task.created_resources.security_groups[0]
    ```
  </Tab>
</Tabs>

```json
{
  "tasks": ["d4a4b10e-..."]  // → save as SG_TASK_ID
}
```

```bash
export SG_ID="{SG_ID}"
```

<Info>
Use <code>POST&nbsp;/cloud/v2/security_groups/...</code> — the v1 endpoint is deprecated.
</Info>

## Step 2. Add inbound rules

Each rule opens a specific port or protocol for incoming traffic. Protocol names must be lowercase.

| Parameter | Required | Description |
|-----------|----------|-------------|
| `direction` | Yes | `"ingress"` for inbound, `"egress"` for outbound. |
| `protocol` | No | `"tcp"`, `"udp"`, `"icmp"`, or omit for all protocols. Lowercase only. |
| `port_range_min` / `port_range_max` | No | Port range (1–65535). Omit for protocols without ports such as ICMP. |
| `ethertype` | No | `"IPv4"` or `"IPv6"`. Defaults to `"IPv4"`. |
| `remote_ip_prefix` | No | CIDR to restrict traffic source. `"0.0.0.0/0"` allows all IPs. |

<Tabs>
  <Tab title="curl">
    ```bash
    # HTTP (port 80)
    curl -X POST \
      "https://api.gcore.com/cloud/v2/security_groups/$PROJECT_ID/$REGION_ID/$SG_ID/rules" \
      -H "Authorization: APIKey $GCORE_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "direction": "ingress",
        "protocol": "tcp",
        "port_range_min": 80,
        "port_range_max": 80,
        "ethertype": "IPv4",
        "remote_ip_prefix": "0.0.0.0/0"
      }'

    # Repeat for HTTPS (443), SSH (22), and ICMP (omit port fields, set protocol: "icmp")
    ```
  </Tab>
  <Tab title="Python SDK">
    ```python
    rules = [
        {"protocol": "tcp",  "port_range_min": 80,   "port_range_max": 80,   "desc": "HTTP"},
        {"protocol": "tcp",  "port_range_min": 443,  "port_range_max": 443,  "desc": "HTTPS"},
        {"protocol": "tcp",  "port_range_min": 22,   "port_range_max": 22,   "desc": "SSH"},
        {"protocol": "icmp", "port_range_min": None, "port_range_max": None, "desc": "ICMP"},
    ]

    for rule in rules:
        kwargs = {
            "group_id": sg_id,
            "project_id": int(os.environ["PROJECT_ID"]),
            "region_id":  int(os.environ["REGION_ID"]),
            "direction": "ingress",
            "ethertype":  "IPv4",
            "remote_ip_prefix": "0.0.0.0/0",
            "protocol": rule["protocol"],
        }
        if rule["port_range_min"]:
            kwargs["port_range_min"] = rule["port_range_min"]
            kwargs["port_range_max"] = rule["port_range_max"]

        r = client.cloud.security_groups.rules.create(**kwargs)
        client.cloud.tasks.poll(r.tasks[0], polling_interval_seconds=3,
                                polling_timeout_seconds=30)
        print(f"Added {rule['desc']} rule")
    ```
  </Tab>
</Tabs>

<Warning>
Outbound traffic on port 25 (SMTP) is blocked at the infrastructure level and cannot be opened via security group rules. Contact [Gcore support](https://gcore.com/contact-us) to unblock it with a business justification.
</Warning>

## Step 3. Verify the security group

Retrieve the security group to confirm all rules were applied.

<Tabs>
  <Tab title="curl">
    ```bash
    curl "https://api.gcore.com/cloud/v1/securitygroups/$PROJECT_ID/$REGION_ID/$SG_ID" \
      -H "Authorization: APIKey $GCORE_API_KEY"
    ```
  </Tab>
  <Tab title="Python SDK">
    ```python
    sg = client.cloud.security_groups.get(
        group_id=sg_id,
        project_id=int(os.environ["PROJECT_ID"]),
        region_id=int(os.environ["REGION_ID"]),
    )
    for rule in sg.security_group_rules:
        print(rule.direction, rule.protocol, rule.port_range_min)
    ```
  </Tab>
</Tabs>

```json
{
  "id": "1578bb84-...",
  "name": "web-firewall",
  "security_group_rules": [
    {"direction": "ingress", "protocol": "tcp",  "port_range_min": 80},
    {"direction": "ingress", "protocol": "tcp",  "port_range_min": 443},
    {"direction": "ingress", "protocol": "tcp",  "port_range_min": 22},
    {"direction": "ingress", "protocol": "icmp"},
    "...39 default egress rules..."
  ]
}
```

<Info>
A new security group is automatically populated with 39 default egress rules covering all standard protocols. These allow all outbound traffic except port 25 and do not need to be added manually.
</Info>

## Step 4. Attach the security group to a VM

There are two ways to attach a security group: at VM creation or to a running instance.

### Attach at VM creation

Pass the security group UUID in the `security_groups` field:

<Tabs>
  <Tab title="curl">
    ```bash
    curl -X POST "https://api.gcore.com/cloud/v2/instances/$PROJECT_ID/$REGION_ID" \
      -H "Authorization: APIKey $GCORE_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "name": "my-vm",
        "flavor": "g1-standard-2-4",
        "ssh_key_name": "my-key",
        "interfaces": [{"type": "external"}],
        "volumes": [{"source": "image", "image_id": "{IMAGE_ID}", "size": 20, "boot_index": 0}],
        "security_groups": [{"id": "'"$SG_ID"'"}]
      }'
    ```
  </Tab>
  <Tab title="Python SDK">
    ```python
    r = client.cloud.instances.create(
        project_id=int(os.environ["PROJECT_ID"]),
        region_id=int(os.environ["REGION_ID"]),
        name="my-vm",
        flavor="g1-standard-2-4",
        ssh_key_name="my-key",
        interfaces=[{"type": "external"}],
        volumes=[{"source": "image", "image_id": os.environ["IMAGE_ID"],
                  "size": 20, "boot_index": 0}],
        security_groups=[{"id": sg_id}],
    )
    ```
  </Tab>
</Tabs>

### Attach to a running instance

To add a security group to a VM that is already running, use the group name (not the UUID):

<Tabs>
  <Tab title="curl">
    ```bash
    curl -X POST \
      "https://api.gcore.com/cloud/v1/instances/$PROJECT_ID/$REGION_ID/$INSTANCE_ID/addsecuritygroup" \
      -H "Authorization: APIKey $GCORE_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"name": "web-firewall"}'
    ```
  </Tab>
  <Tab title="Python SDK">
    ```python
    client.cloud.instances.assign_security_group(
        instance_id=os.environ["INSTANCE_ID"],
        project_id=int(os.environ["PROJECT_ID"]),
        region_id=int(os.environ["REGION_ID"]),
        name="web-firewall",   # name string, not UUID
    )
    ```
  </Tab>
</Tabs>

<Info>
The `security_groups` field at instance creation takes the UUID (`{"id": "..."}`), while `assign_security_group` takes the group name string. Both refer to the same security group.
</Info>

## Clean up

Detach the security group from all instances before deleting it, otherwise the request returns an error.

```bash
curl -X DELETE \
  "https://api.gcore.com/cloud/v1/securitygroups/$PROJECT_ID/$REGION_ID/$SG_ID" \
  -H "Authorization: APIKey $GCORE_API_KEY"
```
