A Linux VM with a public IP address, accessible via SSH, takes 6 API calls and about 30 seconds to build.

<Info>
A permanent [API token](/account-settings/api-tokens) is required, along with a project ID and region ID. Retrieve them from the [Gcore Customer Portal](https://portal.gcore.com) or by calling `GET /cloud/v1/projects` and `GET /cloud/v1/regions`.
</Info>

Set these as environment variables for use throughout the guide:

```bash
export GCORE_API_KEY="{YOUR_API_KEY}"
export PROJECT_ID="{YOUR_PROJECT_ID}"
export REGION_ID="{YOUR_REGION_ID}"
```

## Step 1. Add an SSH key

An SSH key must be registered in the project before instance creation — the instance references it by name, not by ID.

| Parameter | Required | Description |
|-----------|----------|-------------|
| `name` | Yes | An identifier used in Step 4 as `ssh_key_name`. Choose any unique name. |
| `public_key` | No | An RSA or Ed25519 public key. Omit to generate a new key pair. |

<Tabs>
  <Tab title="curl">
    ```bash
    curl -X POST "https://api.gcore.com/cloud/v1/ssh_keys/$PROJECT_ID" \
      -H "Authorization: APIKey $GCORE_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "name": "my-key",
        "public_key": "ssh-ed25519 AAAA... user@host"
      }'
    ```
  </Tab>
  <Tab title="Python SDK">
    ```python
    import os
    from gcore import Gcore

    client = Gcore(api_key=os.environ["GCORE_API_KEY"])
    ssh_key = client.cloud.ssh_keys.create(
        project_id=int(os.environ["PROJECT_ID"]),
        name="my-key",
        public_key="ssh-ed25519 AAAA... user@host",
    )
    ```
  </Tab>
</Tabs>

```json
{
  "id": "94e8584f-322a-4a7c-948b-a171def5eb0e",
  "name": "my-key",
  "state": "ACTIVE"
}
```

<Tip>
When `public_key` is omitted, the response includes a `private_key` field — save it immediately, as it will not be shown again.
</Tip>


## Step 2. Choose an OS image

The image ID is required for the boot volume in Step 4.

| Parameter | Required | Description |
|-----------|----------|-------------|
| `visibility=public` | No | Filters to Gcore-managed images. Omit to include custom uploaded images. |

<Tabs>
  <Tab title="curl">
    ```bash
    curl "https://api.gcore.com/cloud/v1/images/$PROJECT_ID/$REGION_ID?visibility=public" \
      -H "Authorization: APIKey $GCORE_API_KEY"
    ```
  </Tab>
  <Tab title="Python SDK">
    ```python
    images = client.cloud.instances.images.list(
        project_id=int(os.environ["PROJECT_ID"]),
        region_id=int(os.environ["REGION_ID"]),
        visibility="public",
    )
    for img in images.results:
        print(img.id, img.name)
    ```
  </Tab>
</Tabs>

```json
{
  "results": [
    {
      "id": "e02998d0-a7bf-4f90-930d-549ff559019a",  // → save as IMAGE_ID
      "name": "ubuntu-22.04-x64",
      "os_distro": "ubuntu",
      "os_version": "22.04",
      "architecture": "x86_64",
      "min_disk": 5
    }
  ]
}
```

Save the `id` of the chosen image:

```bash
export IMAGE_ID="{IMAGE_ID}"
```

The `min_disk` value is the minimum boot volume size in GB — the volume created in Step 4 must be at least this size.

## Step 3. Choose a flavor

The flavor determines the vCPU and RAM allocation. Flavor IDs are strings, not UUIDs.

<Tabs>
  <Tab title="curl">
    ```bash
    curl "https://api.gcore.com/cloud/v1/flavors/$PROJECT_ID/$REGION_ID" \
      -H "Authorization: APIKey $GCORE_API_KEY"
    ```
  </Tab>
  <Tab title="Python SDK">
    ```python
    flavors = client.cloud.flavors.list(
        project_id=int(os.environ["PROJECT_ID"]),
        region_id=int(os.environ["REGION_ID"]),
    )
    for f in flavors.results:
        print(f.flavor_id, f"{f.vcpus} vCPU / {f.ram // 1024} GB RAM")
    ```
  </Tab>
</Tabs>

```json
{
  "results": [
    {"flavor_id": "g1-standard-1-2", "vcpus": 1, "ram": 2048},
    {"flavor_id": "g1-standard-2-4", "vcpus": 2, "ram": 4096},  // → save as FLAVOR_ID
    {"flavor_id": "g1-standard-4-8", "vcpus": 4, "ram": 8192}
  ]
}
```

```bash
export FLAVOR_ID="g1-standard-2-4"
```

## Step 4. Create the instance

The instance is created with a public network interface — `type: "external"` assigns a public IP automatically, without a separate floating IP.

| Parameter | Required | Description |
|-----------|----------|-------------|
| `name` | Yes | Display name for the instance. |
| `flavor` | Yes | The `flavor_id` string from Step 3. |
| `ssh_key_name` | Yes | The `name` value from Step 1, not the UUID. |
| `interfaces[].type` | Yes | `"external"` for a public IP. Use `"subnet"` for a private network interface. |
| `volumes[].source` | Yes | `"image"` to boot from an image. |
| `volumes[].size` | Yes | Boot volume size in GB. Must be ≥ `min_disk` from the image. |
| `volumes[].boot_index` | Yes | Set to `0` for the primary boot volume. |

<Tabs>
  <Tab title="curl">
    ```bash
    curl -X POST "https://api.gcore.com/cloud/v2/instances/$PROJECT_ID/$REGION_ID" \
      -H "Authorization: APIKey $GCORE_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "name": "my-vm",
        "flavor": "'"$FLAVOR_ID"'",
        "ssh_key_name": "my-key",
        "interfaces": [{"type": "external"}],
        "volumes": [
          {
            "source": "image",
            "image_id": "'"$IMAGE_ID"'",
            "size": 20,
            "boot_index": 0
          }
        ]
      }'
    ```
  </Tab>
  <Tab title="Python SDK">
    ```python
    task_id_list = client.cloud.instances.create(
        project_id=int(os.environ["PROJECT_ID"]),
        region_id=int(os.environ["REGION_ID"]),
        name="my-vm",
        flavor=os.environ["FLAVOR_ID"],
        ssh_key_name="my-key",
        interfaces=[{"type": "external"}],
        volumes=[{
            "source": "image",
            "image_id": os.environ["IMAGE_ID"],
            "size": 20,
            "boot_index": 0,
        }],
    )
    task_id = task_id_list.tasks[0]
    ```
  </Tab>
</Tabs>

```json
{
  "tasks": ["b63f0fff-eda7-4247-9550-74186a8b47fb"]  // → save as TASK_ID
}
```

```bash
export TASK_ID="{TASK_ID}"
```


## Step 5. Wait for the instance to be ready

Instance creation is asynchronous. Poll `GET /cloud/v1/tasks/{task_id}` every 5 seconds until `state` is `FINISHED`.

<Tabs>
  <Tab title="curl">
    ```bash
    curl "https://api.gcore.com/cloud/v1/tasks/$TASK_ID" \
      -H "Authorization: APIKey $GCORE_API_KEY"
    ```
  </Tab>
  <Tab title="Python SDK">
    ```python
    import time

    while True:
        task = client.cloud.tasks.get(task_id)
        if task.state == "FINISHED":
            instance_id = task.created_resources["instances"][0]
            break
        if task.state == "ERROR":
            raise RuntimeError(task.error)
        time.sleep(5)
    ```
  </Tab>
</Tabs>

While running:

```json
{
  "state": "RUNNING",
  "task_type": "create_vm"
}
```

When complete:

```json
{
  "state": "FINISHED",
  "created_resources": {
    "instances": ["7b9e5975-d5f7-4ab8-af0b-a0ef1deef57e"],  // → save as INSTANCE_ID
    "volumes": ["219157e9-7d01-4eb4-b8a2-4362e779734a"]
  }
}
```

```bash
export INSTANCE_ID="{INSTANCE_ID}"
```

## Step 6. Get the public IP address

With the instance in `ACTIVE` state, retrieve its public IP from the `addresses` field.

<Tabs>
  <Tab title="curl">
    ```bash
    curl "https://api.gcore.com/cloud/v1/instances/$PROJECT_ID/$REGION_ID/$INSTANCE_ID" \
      -H "Authorization: APIKey $GCORE_API_KEY"
    ```
  </Tab>
  <Tab title="Python SDK">
    ```python
    instance = client.cloud.instances.get(
        instance_id=os.environ["INSTANCE_ID"],
        project_id=int(os.environ["PROJECT_ID"]),
        region_id=int(os.environ["REGION_ID"]),
    )
    for network_name, ifaces in instance.addresses.items():
        for iface in ifaces:
            print(f"{network_name}: {iface['addr']}")
    ```
  </Tab>
</Tabs>

```json
{
  "status": "ACTIVE",
  "addresses": {
    "pub_net": [
      {
        "addr": "31.184.246.112",  // → use as the SSH target
        "type": "fixed"
      }
    ]
  },
  "security_groups": [{"name": "default"}]
}
```

<Info>
The keys in `addresses` are network names, not a fixed string. The name depends on the region — typically `"pub_net"` for public interfaces. Iterate over all keys when parsing programmatically.
</Info>

<Info>
The `default` security group is applied automatically. It allows inbound SSH (port 22) and ICMP, so the instance is reachable immediately.
</Info>

## Step 7. Connect via SSH

Use the public IP from the previous step. Replace `ubuntu` with the default user for the chosen OS image:

```bash
ssh -i ~/.ssh/my-key ubuntu@31.184.246.112
```

Default SSH users by OS:

| OS | Default user |
|----|-------------|
| Ubuntu | `ubuntu` |
| Debian | `debian` |
| CentOS | `centos` |
| Rocky Linux | `rocky` |
| Fedora | `fedora` |

The instance is now running. Add [block storage](/cloud/via-api/deploy-vm-with-block-storage) for persistent storage that survives instance deletion, or restrict inbound traffic with [firewall rules](/cloud/via-api/configure-firewall).

## Clean up

To delete the instance and its boot volume, pass the volume ID in the `volumes` query parameter. This removes both in a single call.

<Tabs>
  <Tab title="curl">
    ```bash
    curl -X DELETE \
      "https://api.gcore.com/cloud/v1/instances/$PROJECT_ID/$REGION_ID/$INSTANCE_ID?volumes=$BOOT_VOLUME_ID" \
      -H "Authorization: APIKey $GCORE_API_KEY"
    ```
  </Tab>
  <Tab title="Python SDK">
    ```python
    client.cloud.instances.delete(
        instance_id=os.environ["INSTANCE_ID"],
        project_id=int(os.environ["PROJECT_ID"]),
        region_id=int(os.environ["REGION_ID"]),
        volumes=os.environ["BOOT_VOLUME_ID"],
    )
    ```
  </Tab>
</Tabs>

```json
{
  "tasks": ["ef3b0d34-..."]
}
```

Poll `GET /cloud/v1/tasks/{task_id}` until `state` is `FINISHED`. The boot volume ID is available in the instance details under `volumes[0].id`.

<Info>
If a floating IP was assigned to the instance, add `delete_floatings=true` to the query string to release it at the same time: `?volumes={id}&delete_floatings=true`.
</Info>
