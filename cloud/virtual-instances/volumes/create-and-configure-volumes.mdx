<MethodSection id="portal" label="Customer Portal">

You can view, create, and manage volumes in the Gcore Customer Portal on the **Volumes** page. Here's how to navigate the page:


<Frame>![Volumes page with numbered UI elements](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/volumes-page-annotated.png)</Frame>



1\. **Volumes** : View all volumes created in a project.

2\. **Instance** : Check if a volume is attached to a Virtual Machine.

3\. **IOPS limit/burst** : Check the maximum number of read and write operations a volume can handle in a second and when a burst traffic occurs. 

4\. **MBps limit/burst** : Check the bandwidth limit for a volume and a max bandwidth when a burst traffic occurs.

5\. **Without attachments** : View all volumes that aren't attached to a Virtual Machine.

In the following sections, you'll find information on how to create and manage volumes. For a general overview of volumes and available volume types, check out our [dedicated guide](/cloud/virtual-instances/volumes/about-volumes). 

## Create a volume

You can create volumes in several ways: [when creating a Virtual Machine](/cloud/virtual-instances/create-an-instance), from the **Volumes** page, or from a snapshot. The latter two approaches are described in the following sections. 

<Info>
**Info**

You can only use a volume in the same region where a Virtual Machine is created.
</Info>

<Tabs>
  <Tab title="From the Volumes page">
    1\. In the Gcore Customer Portal, navigate to **Cloud** > **Storage**.

    2\. Click **Create volume**.

    <Frame>![Volumes page with numbered storage and create volume tabs](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/create-volume.png)</Frame>

    3\. A new **Add volume** page will open. Here you can customize volume settings.

    <Frame>![Dialog with options for configuring new volume](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/new-volume-volumes-page.png)</Frame>

    4\. Specify how many GB of disk space you need. If it exceeds your current limit, [send a request for quota increase](/cloud/getting-started/request-a-quota-increase#how-to-send-a-request).

    5\. Select the [volume type](/cloud/virtual-instances/volumes/about-volumes#available-volume-types). Faster volume types are designed for latency-sensitive transactional operations and storing critical data. Slower volumes are more suitable for archived or non-critical data and for less frequent operations.

    6\. (Optional) Attach the volume to a Virtual Machine. A volume can only be used in the same region where a VM is created.

    7\. Create a name for the volume. You can use Latin letters, numbers, underscores, spaces, and dots. The name should be between 3 and 63 characters long.

    <Tip>
    **Tip**

    When you add volumes via the Gcore Customer Portal or API, you don't have to keep volume names consistent in the operating system (like sda, sdb, or sdc). We recommend using Universally Unique Identifiers (UUID) instead of names for OS operations.
    </Tip>

    8\. (Optional) To add metadata to a volume, specify tags as key-value pairs.

    9\. Click **Save**. 

    Your volume will appear on the **Volumes** page.
  </Tab>
  <Tab title="From a volume snapshot">
    If you need to quickly duplicate or back up information on your Virtual Machine or provision new resources from pre-existing data, you can create a volume from a snapshot of an existing volume. 

    To create a volume:

    1\. In the Gcore Customer Portal, navigate to **Cloud** > **Snapshots**.

    2\. Find the snapshot you want to create volume from and click on the three-dot icon to open snapshot settings. 

    <Frame>![Snapshots tab with expanded settings menu](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/new-volume-snapshot.png)</Frame>

    3\. Click **Create volume**.

    4\. Select the Virtual Machine to which you want to attach a new volume.

    5\. Click **Create**.

    <Frame>![Create volume dialog with options to select Virtual Machine and add volume name](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/create-volume-from-snapshot.png)</Frame>

    The volume will appear on the **Volumes** page.
  </Tab>
</Tabs>

## Attach volumes to an existing Virtual Machine

If a volume is attached to a Virtual Machine, you'll see the VM name on the **Volumes** page in the **Instance** column. Otherwise, you'll see the "Attach to instance" link. You can attach more than one volume to a Virtual Machine. However, volumes can only be used in the same region where the VM is created.

You can attach volumes to a Virtual Machine either from the **Volumes** page or from the VM settings.

<Tabs>
  <Tab title="From the Volumes page">
    1\. In the Gcore Customer Portal, navigate to **Cloud** > **Storage**.

    2\. Find the volume that you want to attach and click **Attach to instance**. 

    <Frame>![Volumes page with annotated attach to Virtual Machine link](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/attach-volume.png)</Frame>

    3\. Select the Virtual Machine from the **Choose instance to attach to** dropdown.

    4\. (Optional) Add an attachment tag.

    <Frame>![Attach volume dialog with options to select Virtual Machine and add a tag](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/attach-volume-to-instance.png)</Frame>

    5\. Click **Attach volume**.

    You've successfully attached a volume to the Virtual Machine. 
  </Tab>
  <Tab title="From Virtual Machine settings">
    1\. In the Gcore Customer Portal, navigate to **Cloud** > **Virtual instances**.

    2\. Find the Virtual Machine you want to update and click its name to open it.

    <Frame>![Virtual Machines tab with two created instances](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/cloud-instances.png)</Frame>

    3\. Open the **Volumes** tab.

    4\. Click **Add volume**.

    <Frame>![Volumes tab with Add volume button highlighted](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/attach-volume-instance-settings.png)</Frame>

    5\. Click **Use existing** to attach an existing volume to an Virtual Machine. 

    You can also create a new volume here. Click **Create new** and configure the settings as described under the [Create a volume](/cloud/virtual-instances/volumes/create-and-configure-volumes#create-a-volume) section. 

    6\. Select the volume that you want to attach to the Virtual Machine.

    7\. (Optional). Add an attachment tag.


    <Frame>![Add volume menu with options to add Virtual Machine and tags](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/add-volume-to-instance.png)</Frame>

    8\. Click **Add volume**.

    You've successfully attached a volume to the Virtual Machine. 
  </Tab>
</Tabs>

## Manage volumes

The options for managing volumes differ depending on the volume you want to modify: boot or regular. 

If you have a large number of volumes, you can also search for volumes by name or UUID values to find the one you need. 

### Rename a volume

You can rename both boot and regular volumes:

1\. In the Gcore Customer Portal, navigate to **Cloud** > **Storage**.

2\. Click the three-dot icon next to the volume you want to rename.


<Frame>![Volumes tab with expanded settings menu](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/three-dot-menu-regular.png)</Frame>



3\. Click **Rename**.

4\. Enter a new name.

5\. Select **Save** to apply the changes. 


<Frame>![Rename volume dialog](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/rename-volume.png)</Frame>



Your volume has been renamed. 

### Detach a volume from a Virtual Machine

You can only detach regular volumes. The boot volume cannot be detached from an Virtual Machine as long as the VM exists. If you want to detach a boot volume, you need to delete the VM first.

To detach a volume:

1\. In the Gcore Customer Portal, navigate to **Cloud** > **Storage**.

2\. Click the three-dot icon next to the volume you want to detach.


<Frame>![Volumes tab with expanded settings menu](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/three-dot-menu-attached.png)</Frame>



3\. Choose the Virtual Machine from which you want to detach a volume.

4\. Select **Confirm**.


<Frame>![Dialog asking to select a VM to detach the volume from](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/detach-volume-dialog.png)</Frame>



You've detached a volume from the Virtual Machine. 

<Warning>
**Warning**

Detaching a volume will not cancel your payment unless you delete the volume. You can also add a detached volume to any Virtual Machine at any time.
</Warning>

### Resize a volume

You can resize both boot and regular volumes:

1\. In the Gcore Customer Portal, navigate to **Cloud** > **Storage**.

2\. Click the three-dot icon next to the volume you want to resize.


<Frame>![Volumes tab with expanded settings menu](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/three-dot-menu-regular.png)</Frame>



3\. In the **Resize volume** dialog that opens, you can see your current volume size in GB. Enter the new volume size. 

4\. Click **Resize volume** to apply the changes.


<Frame>![Dialog asking for a new volume size](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/resize-volume.png)</Frame>



You've changed your volume size. 

<Warning>
**Warning**

If you have snapshots for this volume, after you increase the size, you won't be able to [revert from the last snapshot](/cloud/virtual-instances/volumes/create-and-configure-volumes#revert-volume-from-snapshot). You can only create a new volume from this volume's snapshot.
</Warning>

### Change volume type

This option is available for regular volumes of High IOPS and Standard types. You can't change the type of boot volume. However, you can create a new volume from a volume snapshot with the required type.

<Info>
**Info**

Before proceeding with the following instructions, make sure that a volume is detached from a Virtual Machine. Otherwise, the **Retype** button won't appear in your settings list.
</Info>

To retype a volume:

1\. In the Gcore Customer Portal, navigate to **Cloud** > **Storage**.

2\. Click the three-dot icon next to the volume you want to retype.


<Frame>![Volumes tab with expanded settings menu](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/three-dot-menu-regular.png)</Frame>



3\. Click **Retype**.

4\. Select the desired volume type and then click **Retype** to apply the updates.


<Frame>![Dialog asking to select a new volume type](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/retype-volume.png)</Frame>



You've successfully changed the volume type. 

If you have multiple volumes attached to one Virtual Machine, you can use the `rsync` and `cp` utilities to migrate data between volumes within an operating system.

### Take a volume snapshot

You can create a volume snapshot and use it later for backup, further recovery, and deploying new Virtual Machines from snapshots of boot volumes. 

To take a snapshot:

1\. In the Gcore Customer Portal, navigate to **Cloud** > **Storage**.

2\. Click the three-dot icon next to the needed volume.


<Frame>![Volumes tab with expanded settings menu](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/three-dot-menu-regular.png)</Frame>



3\. Click **Take snapshot**.

4\. Enter snapshot name.

5\. Click **Take snapshot**.


<Frame>![Dialog asking to add a snapshot name](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/take-snapshot.png)</Frame>



The snapshot will appear on the **Snapshots** page, along with the other created snapshots.

### Revert volume to the latest snapshot

In case of an accidental data loss or data corruption, you might need to restore a volume to its previous state. If you [made a snapshot of that volume](/cloud/virtual-instances/volumes/create-and-configure-volumes#take-volume-snapshot), you can roll your volume back to the state when that snapshot was taken. 

<Info>
**Info**

Before proceeding with the following instructions, make sure that the volume is [detached from an instance](/cloud/virtual-instances/volumes/create-and-configure-volumes#detach-a-volume-from-an-instance) and that you have the volume's snapshot. Otherwise, you'll get an error when trying to revert the volume to its previous state.
</Info>

To revert a volume:

1\. In the Gcore Customer Portal, navigate to **Cloud** > **Storage**.

2\. Click the three-dot icon next to the volume you want to revert.


<Frame>![Volumes tab with expanded settings menu](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/three-dot-menu-regular.png)</Frame>



3\. Click **Revert to the latest snapshot**.

4\. Click **Revert**.


<Frame>![Dialog asking to confirm volume reverting](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/revert-volume.png)</Frame>

 <Info>
**Info**

After resizing a volume, reversion is unavailable because the latest snapshot is taken for the old size. You can only create a new volume. For the new size, create a new snapshot.
</Info>

### Create an image from a bootable volume

You can create an image from a boot volume and use that image to create a new Virtual Machine. 

1\. In the Gcore Customer Portal, navigate to **Cloud** > **Storage**.

2\. Click the three-dot icon next to the volume from which you want to create an image.


<Frame>![Volumes tab with expanded settings menu](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/three-dot-menu-boot.png)</Frame>



3\. Select **Create image**.

4\. Specify the image name.

5\. Select architecture.

6\. Click **Create image**.


<Frame>![Dialog to select image and and architecture](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/create-image.png)</Frame>



Within a few minutes, the image will be created and available on the **Images** page.

### Delete volume

You can delete a regular volume that's not attached to a Virtual Machine. Boot volumes can only be deleted together with the VM.

To delete a volume: 

1\. In the Gcore Customer Portal, navigate to **Cloud** > **Storage**.

2\. Click the three-dot icon next to the volume you want to remove.


<Frame>![Volumes tab with expanded settings menu](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/three-dot-menu-regular.png)</Frame>



3\. Click **Delete**.

4\. Confirm your action by typing "Delete" in the text field.

5\. Click **Delete volume.**


<Frame>![Dialog asking to confirm volume deletion](/images/docs/cloud/virtual-instances/volumes/create-and-configure-volumes/delete-volume.png)</Frame>



The volume has been removed from the Gcore Customer Portal. 

<Tip>
**Tip**

You can also use the **Actions** dropdown to delete multiple volumes at once. For this option to be active, you must have at least one regular volume that is not attached to any Virtual Machine.
</Tip>

<MethodSection id="api" label="REST API">

Volumes in Gcore Cloud are block storage devices that attach to Virtual Machines over the network and provide persistent storage that survives VM restarts. The REST API covers the full lifecycle: creating a blank volume, attaching and detaching it from a VM, resizing it, and deleting it.

<Info>
A permanent [API token](/account-settings/api-tokens) is required, along with a project ID and region ID. Retrieve them from the Customer Portal or by calling <code>GET&nbsp;/cloud/v1/projects</code> and <code>GET&nbsp;/cloud/v1/regions</code>.
</Info>

Open a bash terminal and set these as environment variables before running the examples:

```bash
export GCORE_API_KEY="{YOUR_API_KEY}"
export PROJECT_ID="{YOUR_PROJECT_ID}"
export REGION_ID="{YOUR_REGION_ID}"
```

## Create a volume

A volume must be in the same region as the Virtual Machine it will attach to. The `source: "new-volume"` field creates blank storage; use `"snapshot"` to restore from an existing [snapshot](/cloud/virtual-instances/snapshots/take-a-snapshot-of-your-file-system).

| Parameter | Required | Description |
|-----------|----------|-------------|
| `name` | Yes | Display name. Latin letters, numbers, underscores, spaces, dots. 3-63 characters. |
| `source` | Yes | `"new-volume"` for blank storage, `"snapshot"` to create from a snapshot. |
| `size` | Yes | Volume size in GiB. Minimum 1. |
| `type_name` | No | `"standard"` (default), `"ssd_hiiops"`, `"cold"`, or `"ultra"`. Available types vary by region. |

<Tabs>
  <Tab title="curl">
    ```bash
    curl -X POST "https://api.gcore.com/cloud/v1/volumes/$PROJECT_ID/$REGION_ID" \
      -H "Authorization: APIKey $GCORE_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "name": "data-volume",
        "source": "new-volume",
        "size": 10,
        "type_name": "standard"
      }'
    ```
  </Tab>
  <Tab title="Python SDK">
    ```python
    import os
    from gcore import Gcore

    client = Gcore(api_key=os.environ["GCORE_API_KEY"])
    volume = client.cloud.volumes.create_and_poll(
        project_id=int(os.environ["PROJECT_ID"]),
        region_id=int(os.environ["REGION_ID"]),
        name="data-volume",
        source="new-volume",
        size=10,
        type_name="standard",
    )
    volume_id = volume.id
    ```
  </Tab>
</Tabs>

The API returns:

```json
{
  "tasks": ["b63f0fff-5d22-4c23-8e09-f7a3e1c1a999"]
}
```

For curl, poll <code>GET&nbsp;/cloud/v1/tasks/{task_id}</code> every 5 seconds until `state` is `FINISHED`, then read the volume ID from `created_resources.volumes[0]`:

```json
{
  "state": "FINISHED",
  "created_resources": {
    "volumes": ["8f91a111-88e5-402d-a1e3-83988d9b1bb9"]
  }
}
```

```bash
export VOLUME_ID="{VOLUME_ID}"
```

## Attach to a Virtual Machine

A volume can only be attached to a VM in the same region. One VM can have multiple volumes attached simultaneously.

| Parameter | Required | Description |
|-----------|----------|-------------|
| `instance_id` | Yes | UUID of the VM to attach to. |

<Tabs>
  <Tab title="curl">
    ```bash
    curl -X POST \
      "https://api.gcore.com/cloud/v2/volumes/$PROJECT_ID/$REGION_ID/$VOLUME_ID/attach" \
      -H "Authorization: APIKey $GCORE_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"instance_id": "'"$INSTANCE_ID"'"}'
    ```
  </Tab>
  <Tab title="Python SDK">
    ```python
    client.cloud.volumes.attach_to_instance_and_poll(
        volume_id=volume_id,
        project_id=int(os.environ["PROJECT_ID"]),
        region_id=int(os.environ["REGION_ID"]),
        instance_id=os.environ["INSTANCE_ID"],
    )
    ```
  </Tab>
</Tabs>

The API returns:

```json
{
  "tasks": ["c74f1abc-..."]
}
```

For curl, poll <code>GET&nbsp;/cloud/v1/tasks/{task_id}</code> every 5 seconds until `state` is `FINISHED`. The volume `status` field changes to `in-use` once attached.

## Detach from a Virtual Machine

Only regular (non-boot) volumes can be detached, because boot volumes cannot be removed from a running VM. Detaching does not delete the volume - it remains available for reattachment.

<Tabs>
  <Tab title="curl">
    ```bash
    curl -X POST \
      "https://api.gcore.com/cloud/v2/volumes/$PROJECT_ID/$REGION_ID/$VOLUME_ID/detach" \
      -H "Authorization: APIKey $GCORE_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"instance_id": "'"$INSTANCE_ID"'"}'
    ```
  </Tab>
  <Tab title="Python SDK">
    ```python
    client.cloud.volumes.detach_from_instance_and_poll(
        volume_id=volume_id,
        project_id=int(os.environ["PROJECT_ID"]),
        region_id=int(os.environ["REGION_ID"]),
        instance_id=os.environ["INSTANCE_ID"],
    )
    ```
  </Tab>
</Tabs>

The API returns:

```json
{
  "tasks": ["d85e2cde-..."]
}
```

For curl, poll <code>GET&nbsp;/cloud/v1/tasks/{task_id}</code> every 5 seconds until `state` is `FINISHED`. The volume `status` returns to `available`.

## Resize a volume

Volume size can only be increased, not decreased. Resizing does not require detaching - it can be done while the volume is attached.

| Parameter | Required | Description |
|-----------|----------|-------------|
| `size` | Yes | New size in GiB. Must be greater than the current size. |

<Tabs>
  <Tab title="curl">
    ```bash
    curl -X POST \
      "https://api.gcore.com/cloud/v1/volumes/$PROJECT_ID/$REGION_ID/$VOLUME_ID/extend" \
      -H "Authorization: APIKey $GCORE_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"size": 20}'
    ```
  </Tab>
  <Tab title="Python SDK">
    ```python
    volume = client.cloud.volumes.resize_and_poll(
        volume_id=volume_id,
        project_id=int(os.environ["PROJECT_ID"]),
        region_id=int(os.environ["REGION_ID"]),
        size=20,
    )
    print(f"New size: {volume.size} GiB")
    ```
  </Tab>
</Tabs>

The API returns:

```json
{
  "tasks": ["e91f3abc-..."]
}
```

For curl, poll <code>GET&nbsp;/cloud/v1/tasks/{task_id}</code> every 5 seconds until `state` is `FINISHED`. After resizing, extend the filesystem inside the OS to use the new space (for example, `resize2fs` for ext4 or `xfs_growfs` for XFS).

<Warning>
If the volume has snapshots, resizing it prevents reverting to those snapshots. Only new snapshots taken after the resize can be used for reversion.
</Warning>

## Delete a volume

Only detached volumes can be deleted. Boot volumes are deleted automatically when the VM is deleted (pass `?volumes={volume_id}` to the instance delete call to include the boot volume).

<Tabs>
  <Tab title="curl">
    ```bash
    curl -X DELETE \
      "https://api.gcore.com/cloud/v1/volumes/$PROJECT_ID/$REGION_ID/$VOLUME_ID" \
      -H "Authorization: APIKey $GCORE_API_KEY"
    ```
  </Tab>
  <Tab title="Python SDK">
    ```python
    client.cloud.volumes.delete_and_poll(
        volume_id=volume_id,
        project_id=int(os.environ["PROJECT_ID"]),
        region_id=int(os.environ["REGION_ID"]),
    )
    ```
  </Tab>
</Tabs>

The API returns:

```json
{
  "tasks": ["f02g4bcd-..."]
}
```

For curl, poll <code>GET&nbsp;/cloud/v1/tasks/{task_id}</code> every 5 seconds until `state` is `FINISHED`. Billing for the volume stops immediately after deletion.
