SSH key pairs secure access to Linux Virtual Machines. A public key is stored in the [Gcore Customer Portal](https://portal.gcore.com), and the matching private key remains on the local machine.

## Generate SSH keys

Keys can be generated in the Customer Portal or with CLI tools and SSH clients:

- [In the Gcore Customer Portal](/cloud/virtual-instances/connect/connect-to-your-instance-via-ssh#generate-ssh-keys-in-the-customer-portal)
- [Via CLI tools and SSH clients](/cloud/virtual-instances/connect/connect-to-your-instance-via-ssh#generate-ssh-keys-via-cli-tools-or-ssh-clients)

<Warning>
Never share a private SSH key or password with third parties. Unauthorized access to a Virtual Machine can expose sensitive data stored on the instance.
</Warning>

### Generate SSH keys in the customer portal

Keys can be created during [Virtual Machine creation](/cloud/virtual-instances/create-an-instance) or separately on the **SSH Keys** page.

After generation, the public key appears in the **SSH Keys** list. The private key file downloads to local storage.

<Info>
When naming a public key, use only Latin characters, underscores, spaces, and dots. The name must be between 3 and 63 characters.
</Info>

<Tabs>
  <Tab title="Generate an SSH key when creating a VM">
    When [creating a Virtual Machine](/cloud/virtual-instances/create-an-instance), in **Step 6. Set up authentication**, configure the **SSH key** field.

    <Frame>
      <img src="/images/docs/cloud/virtual-instances/connect/connect-to-your-instance-via-ssh/create-instance-ssh-keys.png" alt="SSH key field with Select ssh key dropdown and links to add or generate a key" width="70%"/>
    </Frame>

    To attach or create a key:

    1. In the **SSH key** section, select an existing key from the **Select ssh key** dropdown, or use one of the links below the dropdown:
       - **add a new SSH Key** — paste a public key generated outside the portal.
       - **generate a SSH Key** — create a new key pair in the portal.

    2. If **generate a SSH Key** is selected, enter a name in the **Autogenerate SSH Key** dialog and click **Create SSH Key**. The browser downloads the private key file.

    <Frame>
      <img src="/images/docs/cloud/virtual-instances/connect/connect-to-your-instance-via-ssh/autogenerate-ssh-key.png" alt="Autogenerate SSH Key dialog with Name field and Create SSH Key button" width="70%"/>
    </Frame>
  </Tab>
  <Tab title="Generate an SSH key in the SSH Keys section">
    A key pair can be generated on the **SSH Keys** page before Virtual Machine creation and selected during instance setup.

    To generate the keys:

    1. In the Customer Portal, navigate to **Cloud** > **SSH Keys**.

    2. Click **Autogenerate SSH Key**.

    <Frame>
      <img src="/images/docs/cloud/virtual-instances/connect/connect-to-your-instance-via-ssh/cloud-ssh-keys-annotated.png" alt="SSH Keys page with Autogenerate SSH Key and Add SSH Key buttons" width="70%"/>
    </Frame>

    3. Enter a name and click **Create SSH Key**. The public key is added to the **SSH Keys** list, and the private key file downloads to local storage.

    <Frame>
      <img src="/images/docs/cloud/virtual-instances/connect/connect-to-your-instance-via-ssh/autogenerate-ssh-key.png" alt="Autogenerate SSH Key dialog with Name field and Create SSH Key button" width="70%"/>
    </Frame>
  </Tab>
</Tabs>

## Generate SSH keys via CLI tools or SSH clients

Local tools can generate key pairs on the administrator machine. Paste the resulting public key into the Customer Portal.

<Tabs>
  <Tab title="Terminal or Command Prompt">
    Follow these instructions to generate SSH keys on Linux, macOS, or Windows 10/11 devices:

    1. Open Terminal (Linux, macOS) or Command Prompt (Windows).

    2. Run the following command to generate a key pair:

    ```sh
    ssh-keygen -t rsa -b 2048
    ```

    3. When prompted for a file path, specify a custom location (for example, `~/.ssh/my_key`) or accept the default. The default directory is `~/.ssh` on Linux and macOS and `C:\Users\username\.ssh` on Windows.

    4. When prompted for a passphrase, enter one for additional security or leave the field empty to create a key without a passphrase.

    5. Confirm the passphrase, or leave the field empty again. The private key and the `.pub` public key file appear in the chosen directory.

    6. Add the public key to the Customer Portal as described in [Add a public SSH key to the customer portal](/cloud/virtual-instances/connect/connect-to-your-instance-via-ssh#add-a-public-ssh-key-to-the-customer-portal).
  </Tab>
  <Tab title="Windows Subsystem for Linux (WSL)">
    Linux distributions on Windows 11 can generate SSH keys for Virtual Machine access. WSL must be installed first.

    <Info>
      To install WSL, follow the instructions from the official Microsoft guide: [How to install Linux on Windows with WSL](https://learn.microsoft.com/en-us/windows/wsl/install).
    </Info>

    To generate SSH keys via WSL:

    1. Open Windows Command Prompt or PowerShell.

    2. Launch the default Linux distribution by running `wsl.exe`. For other ways to start a distribution, see [Ways to run multiple Linux distributions with WSL](https://learn.microsoft.com/en-us/windows/wsl/install#ways-to-run-multiple-linux-distributions-with-wsl).

    3. Run the following command to generate a key pair:

    ```text
    ssh-keygen -t rsa -b 2048
    ```

    4. When prompted for a file path, specify a custom location or accept the default `~/.ssh` directory inside the Linux environment.

    5. When prompted for a passphrase, enter one or leave the field empty to create a key without a passphrase.

    6. Confirm the passphrase, or leave the field empty again. The private key and the `.pub` public key file appear in the chosen directory.

    7. Add the public key to the Customer Portal as described in [Add a public SSH key to the customer portal](/cloud/virtual-instances/connect/connect-to-your-instance-via-ssh#add-a-public-ssh-key-to-the-customer-portal).
  </Tab>
  <Tab title="PuTTYgen">
    Follow these instructions to generate SSH keys on Windows 10/11 devices:

    1. Download and install the [PuTTY package](https://putty.org/).

    2. Launch the PuTTYgen app.

    3. In **Type of key to generate**, select **RSA**.

    4. In **Number of bits in a generated key**, set the value to `2048`.

    <Frame>
      <img src="/images/docs/cloud/virtual-instances/connect/connect-to-your-instance-via-ssh/puttygen-connect.png" alt="PuTTYgen with RSA selected and 2048 bits configured" width="70%"/>
    </Frame>

    5. Click **Generate**. Move the cursor in the **Key** field until the key appears.

    <Frame>
      <img src="/images/docs/cloud/virtual-instances/connect/connect-to-your-instance-via-ssh/puttygen-generate-key.png" alt="PuTTYgen with a generated public key displayed" width="70%"/>
    </Frame>

    6. (Optional) Enter a passphrase in **Key passphrase** and retype it in **Confirm passphrase** to protect the private key.

    7. Click **Save private key** to download the key to local storage.

    <Info>
      Keys generated in PuTTYgen or the Customer Portal must be converted to `.ppk` format before connecting with PuTTY. See [Key conversion](/cloud/ssh-keys/convert-an-ssh-key-from-pem-to-ppk).
    </Info>
  </Tab>
</Tabs>

## Add a public SSH key to the customer portal

If the key was generated outside the Customer Portal, add the public key manually:

1. In the Customer Portal, navigate to **Cloud** > **SSH Keys**.

2. Click **Add SSH Key**.

<Frame>
  <img src="/images/docs/cloud/virtual-instances/connect/connect-to-your-instance-via-ssh/cloud-ssh-keys-add.png" alt="SSH Keys page with Add SSH Key button" width="70%"/>
</Frame>

3. In the **SSH key content** field, paste the public key.

4. Enter a name and click **Add SSH Key**.

<Frame>
  <img src="/images/docs/cloud/virtual-instances/connect/connect-to-your-instance-via-ssh/add-ssh-key-annotated.png" alt="Add SSH Key dialog with SSH key content and Name fields" width="70%"/>
</Frame>

5. The key appears in the **SSH Keys** list and can be selected during Virtual Machine creation.

<Info>
After a public key is added, it is shared automatically with all users in the same project. To restrict visibility, see [Stop sharing](/cloud/virtual-instances/connect/connect-to-your-instance-via-ssh#stop-sharing-ssh-keys).
</Info>

## Manage SSH keys in the customer portal

Public SSH keys can be deleted, shared with other project users, or restricted to the owner only.

### Delete SSH keys

Only the user who created or uploaded an SSH key can delete it. To delete a key:

1. In the Customer Portal, navigate to **Cloud** > **SSH Keys**.

<Frame>
  <img src="/images/docs/cloud/virtual-instances/connect/connect-to-your-instance-via-ssh/cloud-ssh-keys.png" alt="SSH Keys list in the Customer Portal" width="70%"/>
</Frame>

2. Find the key to remove and click the menu icon in that row.

3. Click **Delete**.

4. In the confirmation dialog, click **Delete SSH key**.

<Frame>
  <img src="/images/docs/cloud/virtual-instances/connect/connect-to-your-instance-via-ssh/delete-ssh-keys.png" alt="SSH key row menu with Stop sharing and Delete options" width="70%"/>
</Frame>

## Stop sharing SSH keys

After a key is added or generated in the Customer Portal, the public key is shared automatically with all users in the same project.

To stop sharing and restrict visibility to the owner:

1. In the Customer Portal, navigate to **Cloud** > **SSH Keys**.

<Frame>
  <img src="/images/docs/cloud/virtual-instances/connect/connect-to-your-instance-via-ssh/cloud-ssh-keys.png" alt="SSH Keys list in the Customer Portal" width="70%"/>
</Frame>

2. Find the key to restrict and click the menu icon in that row.

3. Click **Stop sharing**.

<Frame>
  <img src="/images/docs/cloud/virtual-instances/connect/connect-to-your-instance-via-ssh/stop-sharing-ssh-keys.png" alt="SSH key row menu with Stop sharing and Delete options" width="70%"/>
</Frame>

To share the key again, open the same menu and click **Share**.
