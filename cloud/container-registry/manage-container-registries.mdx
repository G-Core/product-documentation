<MethodSection id="portal" label="Customer Portal">

All created registries appear on the **Container Registry** page in the [Gcore Customer Portal](https://portal.gcore.com):


<Frame>![Container registry page in the Customer Portal](/images/docs/cloud/container-registry/manage-container-registries/container-registry-page.png)</Frame>



The storage limit, users, and images of a registry can be managed from this page. A search bar is available to filter registries by name.

## Change storage limit

The storage limit can be increased to host more images or reduced to optimize storage usage.

To change the storage limit:

1\. In the [Gcore Customer Portal](https://portal.gcore.com), navigate to **Cloud** > **Container Registry**.

2\. Find the registry and click the three-dot icon next to it.

3\. Click **Change storage limit**.


<Frame>![Container registry settings with change limit button highlighted](/images/docs/cloud/container-registry/manage-container-registries/container-registry-change-limit.png)</Frame>



4\. Enter a new limit and click **Save** to apply the changes.


<Frame>![A confirmation dialog to change storage limit](/images/docs/cloud/container-registry/manage-container-registries/set-new-storage-limit.png)</Frame>



The storage limit has been changed. 

## Manage images in the container registry

Open the **Images** tab to view the following information about the images in the registry:

  * Image name 

  * Artifacts associated with the image 

  * The number of times the image was pulled from the registry 

  * The date and time when the image was last modified



<Frame>![Images tab in the registry overview](/images/docs/cloud/container-registry/manage-container-registries/images-tab-cropped.png)</Frame>



Available actions on images: view and manage related artifacts, pull images from the registry, or delete them. Use the search bar to find a specific image.

**View and manage image artifacts**

To view image artifacts, click the image name to open it. The **Artifacts** page shows the artifact name, its tags, size, and the last associated push and pull operations. A search bar is available to filter artifacts.


<Frame>![Images tab with the list of image artifacts](/images/docs/cloud/container-registry/manage-container-registries/image-artifacts.png)</Frame>



To delete an artifact or its tag: 

1\. Find the needed artifact and click the three-dot icon next to it. 

2\. Select the required action from the dropdown and proceed with the instructions. 


<Frame>![Image settings with the highlighted delete and delete tags buttons](/images/docs/cloud/container-registry/manage-container-registries/delete-image-artifact.png)</Frame>



To apply actions to multiple artifacts at once, select the checkboxes next to the relevant artifacts and select the action from the **Group actions** dropdown.

**Delete an image**

1\. On the **Images** tab, find the image you want to remove.


<Frame>![Image settings with the highlighted delete button](/images/docs/cloud/container-registry/manage-container-registries/delete-image.png)</Frame>



2\. Click the three-dot icon and select **Delete**. 

3\. Confirm by typing "Delete" in the text field.

4\. Click **Delete Image**.


<Frame>![Image deletion dialog](/images/docs/cloud/container-registry/manage-container-registries/confirm-image-deletion.png)</Frame>



The image has been removed from the registry.

**Delete multiple images with group actions**

To delete multiple images at once, select the checkboxes next to the target images and select the action from the **Group actions** dropdown.


<Frame>![Group actions dropdown on the Images tab](/images/docs/cloud/container-registry/manage-container-registries/image-group-actions.png)</Frame>



## Manage users in the container registry

Open the **Users** tab to view the following information about created users in the registry:

  * Username 

  * User permissions 

  * The date and time when the user was created 

  * (Optional) **Expires at** — displayed when a credential expiration date was set during user creation



<Frame>![Users tab in the registry overview](/images/docs/cloud/container-registry/manage-container-registries/users-tab.png)</Frame>



User permissions can be changed, the password regenerated, or the user deleted from the registry.

<Tip>
For details on creating a user, see [Create a Container Registry](/cloud/container-registry/create-container-registry).
</Tip>

**Change user permissions**

The following permissions are available:

  * **Pull images** : restricts the user to pull-only access, preventing unauthorized or accidental changes to the registry.

  * **Push and pull images** : grants full access — the user can both pull and push images and image versions.




To change user permissions: 

1\. On the **Users** tab, find the user and click the three-dot icon next to it.

2\. Select **Change permissions**.


<Frame>![Users settings with the highlighted change permissions button](/images/docs/cloud/container-registry/manage-container-registries/change-user-permissions.png)</Frame>



3\. Update the permissions as needed and click **Save** to apply the changes.


<Frame>![Change permissions dialog](/images/docs/cloud/container-registry/manage-container-registries/select-new-user-permissions.png)</Frame>



**Reset user password**

A user password is generated automatically during creation. If it has expired or is no longer known, regenerate it:

1\. On the **Users** tab, find the user and click the three-dot icon next to it.

2\. Select **Reset password**.


<Frame>![Users settings with the highlighted reset password button](/images/docs/cloud/container-registry/manage-container-registries/reset-user-password.png)</Frame>



3\. Confirm password reset.


<Frame>![Reset password confirmation dialog](/images/docs/cloud/container-registry/manage-container-registries/confirm-password-reset.png)</Frame>



4\. Copy and save the new password — it cannot be retrieved after this step.

**Delete a user**

1\. On the **Users** tab, find the user and click the three-dot icon next to it.

2\. Select **Delete**.


<Frame>![Users settings with the highlighted delete button](/images/docs/cloud/container-registry/manage-container-registries/delete-user.png)</Frame>



3\. Confirm by typing "Delete" in the text field.

4\. Click **Delete user**.


<Frame>![Delete user confirmation dialog](/images/docs/cloud/container-registry/manage-container-registries/confirm-user-deletion.png)</Frame>



**Manage multiple users with group actions**

To delete multiple users or change their permissions simultaneously, select the checkboxes next to the target users and select the action from the **Group actions** dropdown.


<Frame>![Users settings with the highlighted group actions dropdown](/images/docs/cloud/container-registry/manage-container-registries/group-actions-users.png)</Frame>



## Delete a registry

<Warning>
Deleting a Container Registry permanently removes all data stored in it. This action cannot be undone.
</Warning>

To delete a registry:

1\. In the [Gcore Customer Portal](https://portal.gcore.com), navigate to **Cloud** > **Container Registry**.

2\. Find the registry and click the three-dot icon next to it.

3\. Select **Delete**.


<Frame>![Container registry page with the highlighed delete button](/images/docs/cloud/container-registry/manage-container-registries/delete-registry.png)</Frame>



4\. Type "Delete" to confirm.

The registry has been removed.

<MethodSection id="api" label="REST API">

Use the Gcore API to inspect and maintain a container registry: resize its storage, browse repositories and artifacts, manage registry users, and remove resources that are no longer needed.

<Info>
An [API token](/account-settings/api-tokens) is required, along with a [project ID](/api-reference/cloud/projects/list-projects) and [region ID](/api-reference/cloud/regions/list-regions).
</Info>

Set the following environment variables before running the examples:

```bash
export GCORE_API_KEY="{YOUR_API_KEY}"
export GCORE_CLOUD_PROJECT_ID="{YOUR_PROJECT_ID}"
export GCORE_CLOUD_REGION_ID="{YOUR_REGION_ID}"
export REGISTRY_ID="{YOUR_REGISTRY_ID}"
```

<Info>
**REGISTRY\_ID** is the numeric ID of an existing container registry. To create a registry or find its ID, see [Create a container registry](/cloud/container-registry/create-container-registry).
</Info>

## Resize storage limit

This call updates the maximum storage capacity of the registry in GiB.

| Parameter | Description |
|-----------|-------------|
| `storage_limit` | New storage limit in GiB |

<Tabs>
  <Tab title="Python SDK">
```python
registry = client.cloud.registries.resize(registry_id=registry_id, storage_limit=20)
print(f"Storage limit: {registry.storage_limit} GiB")
```
  </Tab>
  <Tab title="Go SDK">
```go
registry, err := client.Cloud.Registries.Resize(ctx, registryID, cloud.RegistryResizeParams{
    StorageLimit: gcore.Int(20),
})
if err != nil {
    log.Fatal(err)
}
fmt.Printf("Storage limit: %d GiB\n", registry.StorageLimit)
```
  </Tab>
  <Tab title="curl">
```bash
curl -s -X PATCH \
  "https://api.gcore.com/cloud/v1/registries/$GCORE_CLOUD_PROJECT_ID/$GCORE_CLOUD_REGION_ID/$REGISTRY_ID/resize" \
  -H "Authorization: APIKey $GCORE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"storage_limit": 20}'
```

Response:

```json
{
  "id": 607,
  "name": "my-registry",
  "storage_limit": 20,
  "storage_used": 0,
  "url": "registry.luxembourg-2.cloud.gcore.dev/1000503-1186668-76-my-registry/",
  "repo_count": 0,
  "created_at": "2026-05-30T21:35:34Z",
  "updated_at": "2026-05-30T22:10:00Z"
}
```
  </Tab>
</Tabs>

The [resize](/api-reference/cloud#tag/Registry/operation/RegistryResizeHandler.patch) endpoint returns the full registry object, including the updated `storage_limit`.

## List repositories

This call returns all repositories (images) stored in the registry, along with their artifact count and pull statistics.

<Tabs>
  <Tab title="Python SDK">
```python
repos = client.cloud.registries.repositories.list(registry_id=registry_id)
print(f"Total repositories: {repos.count}")
for r in repos.results:
    print(f"  {r.name}  artifacts={r.artifact_count}  pulls={r.pull_count}")
```
  </Tab>
  <Tab title="Go SDK">
```go
repos, err := client.Cloud.Registries.Repositories.List(ctx, registryID,
    cloud.RegistryRepositoryListParams{})
if err != nil {
    log.Fatal(err)
}
fmt.Printf("Total repositories: %d\n", repos.Count)
for _, r := range repos.Results {
    fmt.Printf("  %s  artifacts=%d  pulls=%d\n", r.Name, r.ArtifactCount, r.PullCount)
}
```
  </Tab>
  <Tab title="curl">
```bash
curl -s \
  "https://api.gcore.com/cloud/v1/registries/$GCORE_CLOUD_PROJECT_ID/$GCORE_CLOUD_REGION_ID/$REGISTRY_ID/repositories" \
  -H "Authorization: APIKey $GCORE_API_KEY"
```

Response:

```json
{
  "count": 1,
  "results": [
    {
      "id": 42,
      "name": "nginx",
      "artifact_count": 2,
      "pull_count": 5,
      "registry_id": 607,
      "created_at": "2026-05-30T22:00:00Z",
      "updated_at": "2026-05-30T22:05:00Z"
    }
  ]
}
```
  </Tab>
</Tabs>

Each [repository](/api-reference/cloud#tag/Registry/operation/RegistryRepositoriesHandler.get) object includes the registry ID, artifact count, pull count, and creation timestamp.

## List artifacts in a repository

This call returns all artifacts (image layers) stored in a repository, including their tags and size.

<Tabs>
  <Tab title="Python SDK">
```python
REPOSITORY_NAME = "nginx"  # repository name from List repositories

artifacts = client.cloud.registries.artifacts.list(
    registry_id=registry_id,
    repository_name=REPOSITORY_NAME,
)
print(f"Total artifacts: {artifacts.count}")
for a in artifacts.results:
    tags = [t.name for t in (a.tags or [])]
    print(f"  digest={a.digest[:20]}...  size={a.size} bytes  tags={tags}")
```
  </Tab>
  <Tab title="Go SDK">
```go
const repoName = "nginx" // repository name from List repositories

artifacts, err := client.Cloud.Registries.Artifacts.List(ctx, repoName,
    cloud.RegistryArtifactListParams{RegistryID: registryID})
if err != nil {
    log.Fatal(err)
}
fmt.Printf("Total artifacts: %d\n", artifacts.Count)
for _, a := range artifacts.Results {
    var tags []string
    for _, t := range a.Tags {
        tags = append(tags, t.Name)
    }
    fmt.Printf("  digest=%s...  size=%d bytes  tags=%v\n",
        a.Digest[:20], a.Size, tags)
}
```
  </Tab>
  <Tab title="curl">
```bash

curl -s \

  -H "Authorization: APIKey $GCORE_API_KEY"
```

Response:

## Delete a tag

This call removes a single tag from an artifact while keeping the artifact itself intact. Use the `digest` from List artifacts and the tag `name` from the `tags` array.

<Tabs>
  <Tab title="Python SDK">
```python
REPOSITORY_NAME = "nginx"
DIGEST   = "sha256:a3f4e1b2c9..."  # artifact digest from List artifacts
TAG_NAME = "latest"               # tag name from List artifacts

client.cloud.registries.tags.delete(
    tag_name=TAG_NAME,
    registry_id=registry_id,
    repository_name=REPOSITORY_NAME,
    digest=DIGEST,
)
print("Tag deleted.")
```
  </Tab>
  <Tab title="Go SDK">
```go
const repoName = "nginx"
const digest  = "sha256:a3f4e1b2c9..." // artifact digest from List artifacts
const tagName = "latest"               // tag name from List artifacts

err = client.Cloud.Registries.Tags.Delete(ctx, tagName,
    cloud.RegistryTagDeleteParams{
        RegistryID:     registryID,
        RepositoryName: repoName,
        Digest:         digest,
    })
if err != nil {
    log.Fatal(err)
}
fmt.Println("Tag deleted.")
```
  </Tab>
  <Tab title="curl">
```bash
REPOSITORY_NAME="nginx"
DIGEST="sha256:a3f4e1b2c9..."   # artifact digest from List artifacts
TAG_NAME="latest"               # tag name from List artifacts

curl -s -X DELETE \
  "https://api.gcore.com/cloud/v1/registries/$GCORE_CLOUD_PROJECT_ID/$GCORE_CLOUD_REGION_ID/$REGISTRY_ID/repositories/$REPOSITORY_NAME/artifacts/$DIGEST/tags/$TAG_NAME" \
  -H "Authorization: APIKey $GCORE_API_KEY"
```
  </Tab>
</Tabs>

The [delete tag](/api-reference/cloud#tag/Registry/operation/RegistryTagHandler.delete) operation returns HTTP 204. To remove all tags together with the artifact, use the Delete an artifact operation instead.

## Change user permissions

This call updates whether a user can push images in addition to pulling them. Pass `read_only: true` to restrict the user to pull-only access.

| Parameter | Description |
|-----------|-------------|
| `duration` | Credential lifetime in days; `-1` for indefinite access |
| `read_only` | `true` = pull only; `false` = push and pull |

<Tabs>
  <Tab title="Python SDK">
```python
USER_ID = int(os.environ["USER_ID"])

updated = client.cloud.registries.users.update(
    user_id=USER_ID,
    registry_id=registry_id,
    duration=-1,
    read_only=True,  # restrict to pull-only
)
print(f"read_only: {updated.read_only}")
```
  </Tab>
  <Tab title="Go SDK">
```go
updated, err := client.Cloud.Registries.Users.Update(ctx, userID,
    cloud.RegistryUserUpdateParams{
        RegistryID: registryID,
        Duration:   -1,
        ReadOnly:   gcore.Bool(true), // restrict to pull-only
    })
if err != nil {
    log.Fatal(err)
}
fmt.Printf("read_only: %v\n", updated.ReadOnly)
```
  </Tab>
  <Tab title="curl">
```bash
USER_ID=730

curl -s -X PATCH \
  "https://api.gcore.com/cloud/v1/registries/$GCORE_CLOUD_PROJECT_ID/$GCORE_CLOUD_REGION_ID/$REGISTRY_ID/users/$USER_ID" \
  -H "Authorization: APIKey $GCORE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"duration": -1, "read_only": true}'
```

Response:

```json
{
  "id": 730,
  "name": "r_1000503-1186668-76-my-registry+myuser",
  "read_only": true,
  "duration": -1,
  "expires_at": "1969-12-31T23:59:59Z",
  "created_at": "2026-05-30T21:39:41Z"
}
```
  </Tab>
</Tabs>

The [update user](/api-reference/cloud#tag/Registry/operation/RegistryUserHandler.patch) endpoint also accepts a `duration` change in the same request, so permissions and expiry can be updated together.

## Regenerate user password

This call generates a new password for a registry user. The new `secret` is returned only in this response — it cannot be retrieved later.

<Tabs>
  <Tab title="Python SDK">
```python
refreshed = client.cloud.registries.users.refresh_secret(
    registry_id=registry_id,
    user_id=USER_ID,
)
print(f"New password: {refreshed.secret}")  # shown once — save it now
```
  </Tab>
  <Tab title="Go SDK">
```go
refreshed, err := client.Cloud.Registries.Users.RefreshSecret(ctx, userID,
    cloud.RegistryUserRefreshSecretParams{RegistryID: registryID})
if err != nil {
    log.Fatal(err)
}
fmt.Printf("New password: %s\n", refreshed.Secret) // shown once — save it now
```
  </Tab>
  <Tab title="curl">
```bash
USER_ID=730

curl -s -X POST \
  "https://api.gcore.com/cloud/v1/registries/$GCORE_CLOUD_PROJECT_ID/$GCORE_CLOUD_REGION_ID/$REGISTRY_ID/users/$USER_ID/refresh_secret" \
  -H "Authorization: APIKey $GCORE_API_KEY"
```

Response:

```json
{
  "id": 730,
  "name": "r_1000503-1186668-76-my-registry+myuser",
  "secret": "T53rmPo1UeuiSiTarDCa2MKj2K6tuiQB",
  "duration": -1,
  "expires_at": "1969-12-31T23:59:59Z",

```
  </Tab>
</Tabs>

The `secret` returned by the [refresh secret](/api-reference/cloud#tag/Registry/operation/RegistryRefreshSecretHandler.post) endpoint is shown only once — save it before closing the response. Run `docker login` again with the new password to start a fresh session.
