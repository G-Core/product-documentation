Data stored in a Kubernetes container's file system is temporary and not intended for long-term retention. When a container is replaced or terminated, all information within it is deleted and cannot be restored.

To prevent data loss, connect the Managed Kubernetes cluster to the NFS Container Storage Interface (CSI) driver. This driver connects [Kubernetes](/cloud/kubernetes/about-gcore-kubernetes) and [Gcore File Shares](/cloud/file-shares/about-file-shares), allowing Kubernetes clusters to dynamically provision, attach, and manage NFS volumes as [persistent volumes](/cloud/kubernetes/storage/create-a-pvc-and-bind-it-to-a-pod).

## Provision clusters with persistent NFS volumes

Kubernetes integration with the NFS driver is enabled by default, and the driver is installed in each cluster. 

<Info>
To provision persistent NFS volumes, ensure that a Managed Kubernetes cluster and File Share are created in the same project and connected to the same network and subnetwork.
</Info>

### Step 1. Create a Managed Kubernetes cluster

Create a cluster by following the instructions from our guide: [Create a Managed Kubernetes cluster](/cloud/kubernetes/clusters/create-a-kubernetes-cluster). 

If you already have a Kubernetes cluster created, proceed to the next step.

### Step 2. Create a file share

Create a File Share in the same project and region as the Kubernetes cluster:

1\. In the Gcore Customer Portal, navigate to **Storage** > **File Shares**.

2\. Click **Create File Share**.


<Frame>![File Shares page with a highlighted button to create file share](/images/docs/cloud/kubernetes/storage/add-file-share.png)</Frame>



3\. Configure File Share settings:

  * **Basic settings**: Enter the name of the File Share and specify its size and protocol.

  * **Network settings**: Select the private network and subnetwork for file sharing. Ensure they match the network configured in the Kubernetes cluster.

  * **Access**: Select **Custom access rules** to define access by IP address. Click **Add rule** and specify the IP addresses of machines that should have access, along with their access modes. Alternatively, select **Accessible for the selected subnet** to grant access to all addresses on the network.

  * **Additional options**: Optionally add metadata tags to the File Share.




The File Share should be set up within a few minutes.

### Step 3. Validate persistent NFS volume configuration

Run the following command to retrieve information about the storage classes configured in the cluster:

```sh
kubectl get storageclass
```

If the configuration is correct, the `gcore-nfs-*` storage class should be listed among other storage classes.

### Resize a file share

<Info>
To change the size of an attached file share, first unmount it, then remount it inside the Kubernetes cluster after resizing. Resizing a file share while it is still mounted will cause applications to stop working correctly.
</Info>
