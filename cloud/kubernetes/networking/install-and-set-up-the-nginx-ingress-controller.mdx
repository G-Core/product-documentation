Installing the nginx Ingress Controller automatically creates a [Load Balancer](https://gcore.com/cloud/load-balancers). This Load Balancer acts as an entry point for incoming traffic. When traffic arrives, the Load Balancer forwards it to the Ingress Controller. The Ingress Controller routes traffic to the appropriate services in the Managed Kubernetes cluster based on the rules defined in Ingress manifests.

## Step 1. Install Helm

Using [Helm](https://helm.sh/) package manager simplifies the process of deploying, managing, and scaling applications on Kubernetes.

The following sections cover Helm installation on the most common operating systems. For additional options, refer to the official [installing Helm](https://helm.sh/docs/intro/install/) guide.

<Tabs>
    <Tab title="On Linux">
        1\. In the Terminal, run the official Helm installation script:

        ```sh
        curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash
        ```

        This command downloads and installs the latest version of Helm and places the binary in `/usr/local/bin/helm` (or another location depending on the system configuration).

        2\. Verify the installation by checking the Helm version: `helm version`. The installed version is displayed.
    </Tab>
    <Tab title="On macOS">
        If you have [Homebrew](https://brew.sh/) installed, run `brew install helm`.

        Verify the installation by running `helm version`. This command returns the installed Helm version.

        If Homebrew is not installed, follow the installation steps for Linux.
    </Tab>
    <Tab title="On Windows">
        Download the latest Helm release from the [Helm GitHub releases](https://github.com/helm/helm/releases) page and extract the ZIP file to a local directory, for example `C:\Program Files\Helm`.

        Add Helm binary to the `PATH`:

        1\. Open the **Start** menu and search for **Environment Variables**.

        2\. Click **Edit** the system environment variables.

        3\. In the **System properties** window, click **Environment variables**.

        4\. In the **System variables** section, click **New**.

        5\. Provide a variable name and set the full path to the directory where `helm.exe` is located (e.g., `C:\Program Files\Helm`).

        6\. Click **OK** to save the changes.

        7\. Verify the installation. In the Terminal, run `helm version`. The latest release version downloaded from the GitHub repository in step 1 is displayed.
    </Tab>
</Tabs>

## Step 2. Create a namespace

Run the following kubectl command to create a namespace:

```sh
kubectl create namespace ingress-nginx
 ```

## Step 3. Add Helm repository for the nginx controller

Run the following command to add the helm repository for the nginx Ingress Controller:

```sh
helm repo add ingress-nginx https://kubernetes.github.io/ingress-nginx
 ```

## Step 4. Update helm repository

Update data in the helm repository:

```sh
helm repo update
 ```

## Step 5. Install nginx Ingress controller

Run the following command:

```sh
helm install ingress-nginx ingress-nginx/ingress-nginx \
\--namespace ingress-nginx \
\--set controller.ingressClassResource.name=nginx
 ```

After the nginx Ingress Controller is installed, a Load Balancer is automatically created.

## Step 6. Get an IP of a load balancer

To get the IP address of the created Load Balancer, execute the command below:

```sh
watch kubectl -n ingress-nginx get svc
 ```

Wait until the `<pending>` state of the `EXTERNAL-IP` column changes to an IP address.

The Load Balancer appears in the Gcore Customer Portal on the **Load Balancers** page and routes all incoming traffic to the nginx Ingress Controller.
