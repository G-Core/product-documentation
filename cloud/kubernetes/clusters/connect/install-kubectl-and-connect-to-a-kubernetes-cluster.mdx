kubectl is a command-line tool for controlling Kubernetes clusters. With kubectl, resources can be created and configured, pods can be managed, and other cluster operations can be performed.

kubectl works by sending API requests to the cluster. The cluster processes the requests and applies the specified changes.

For the full list of kubectl operations, refer to the [kubectl official documentation](https://kubernetes.io/docs/reference/generated/kubectl/kubectl-commands).

There is also a graphical alternative called Lens — a desktop application for managing clusters without the command line. Download it at [k8slens.dev](https://k8slens.dev). This guide covers kubectl only.

## Install kubectl and connect from Windows

1\. Download kubectl for Windows. The example below downloads v1.32.0. For the latest version, find the updated command in the [Kubernetes official documentation](https://kubernetes.io/docs/tasks/tools/install-kubectl-windows/#install-kubectl-binary-with-curl-on-windows):

```sh
curl.exe -LO https://dl.k8s.io/release/v1.32.0/bin/windows/amd64/kubectl.exe
```

2\. Add the directory containing **kubectl.exe** to the **PATH** environment variable so the system can find it. First, copy the download path from the command line:

<Frame>![image_28.png](/images/docs/cloud/kubernetes/clusters/connect/install-kubectl-and-connect-to-a-kubernetes-cluster/image_28.png)</Frame>

Then navigate to **Windows settings** > **System** > **About** > **Advanced system settings** (or right-click **This PC** and select **Properties** > **Advanced system settings**):

<Frame>![image_61.png](/images/docs/cloud/kubernetes/clusters/connect/install-kubectl-and-connect-to-a-kubernetes-cluster/image_61.png)</Frame>

In the dialog, click **Environment Variables**, double-click the **Path** system variable, paste the kubectl.exe path into the bottom empty field, and save the changes:

<Frame>![image_62.png](/images/docs/cloud/kubernetes/clusters/connect/install-kubectl-and-connect-to-a-kubernetes-cluster/image_62.png)</Frame>

3\. Create the **.kube** directory in `C:\Users\[username]` to store the cluster configuration file:

<Frame>![image_46.png](/images/docs/cloud/kubernetes/clusters/connect/install-kubectl-and-connect-to-a-kubernetes-cluster/image_46.png)</Frame>

4\. Download the configuration file from the cluster page in the Gcore Customer Portal:

<Frame>![Screenshot_2023-02-27_at_18.50_1.jpg](/images/docs/cloud/kubernetes/clusters/connect/install-kubectl-and-connect-to-a-kubernetes-cluster/13323281147921.png)</Frame>

5\. Rename the downloaded file to **config** (no extension) and place it in the **.kube** folder. kubectl can now access it:

<Frame>![image_45.png](/images/docs/cloud/kubernetes/clusters/connect/install-kubectl-and-connect-to-a-kubernetes-cluster/image_45.png)</Frame>

6\. Verify the kubectl configuration:

```sh
kubectl get pod
```

The output will be either `No resources found in default namespace` (when the cluster has no pods) or a list of running pods. Either result confirms kubectl is correctly configured.

## Install kubectl and connect from Ubuntu or Linux

1\. Download the latest stable kubectl binary:

```sh
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
```

2\. Make the kubectl binary executable:

```sh
chmod +x ./kubectl
```

3\. Move the binary to a directory in the **PATH** variable:

```sh
sudo mv ./kubectl /usr/local/bin/kubectl
```

4\. Create the **.kube** directory in the home directory:

```sh
mkdir ~/.kube
```

5\. Download the configuration file from the cluster page in the Gcore Customer Portal:

<Frame>![Screenshot_2023-02-27_at_18.50_1.jpg](/images/docs/cloud/kubernetes/clusters/connect/install-kubectl-and-connect-to-a-kubernetes-cluster/13323281147921.png)</Frame>

6\. Rename the downloaded file to **config** (no extension) and place it in the **.kube** folder:

<Frame>![image_47.png](/images/docs/cloud/kubernetes/clusters/connect/install-kubectl-and-connect-to-a-kubernetes-cluster/image_47.png)</Frame>

7\. Verify kubectl configuration:

```sh
kubectl get pod
```

The output will be either `No resources found in default namespace` or a list of pods, confirming kubectl is correctly configured.

## Install kubectl and connect from macOS

1\. Download the latest stable kubectl binary. For Intel Macs, use `amd64`; for Apple Silicon (M1/M2/M3), use `arm64`:

```sh
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/darwin/amd64/kubectl"
```

2\. Make the binary executable:

```sh
chmod +x ./kubectl
```

3\. Move the binary to a directory in the **PATH** variable. Check which directories are in PATH:

```sh
echo $PATH
```

Select a directory from the output and move kubectl there. For example:

```sh
sudo mv ./kubectl /usr/local/bin/kubectl
```

4\. Create the **.kube** directory in the home directory:

```sh
mkdir ~/.kube
```

5\. Download the configuration file from the cluster page in the Gcore Customer Portal:

<Frame>![Screenshot_2023-02-27_at_18.50_1.jpg](/images/docs/cloud/kubernetes/clusters/connect/install-kubectl-and-connect-to-a-kubernetes-cluster/13323281147921.png)</Frame>

6\. Rename the downloaded file to **config** (no extension) and place it in the **.kube** folder.

7\. Verify kubectl configuration:

```sh
kubectl get pod
```

The output will be either `No resources found in default namespace` or a list of pods, confirming kubectl is correctly configured.

## Renewal of kubectl certificates

Gcore renews kubectl access certificates every two years. Notifications are sent before the renewal date:

- **In the Gcore Customer Portal**: a notification appears with the expiry date and a reminder to download the new kubeconfig from the cluster page.
- **Via email**: two weeks before the renewal date, an email is sent with the renewal date. After the certificate is updated, a confirmation email is sent with a reminder to download the new certificate.
