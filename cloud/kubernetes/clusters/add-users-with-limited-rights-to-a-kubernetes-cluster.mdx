To follow this guide, install kubectl first. For instructions, see [Install kubectl and connect to a Kubernetes cluster](/cloud/kubernetes/clusters/connect/install-kubectl-and-connect-to-a-kubernetes-cluster).

## Basics you need to know

**What are service accounts?** Service accounts allow pods to read and use Kubernetes API objects, and can also be used to create a kubeconfig file that grants namespace-scoped access to Kubernetes objects for any user or service. For more information, refer to the Kubernetes documentation: [Managing Service Accounts](https://kubernetes.io/docs/reference/access-authn-authz/service-accounts-admin).

**What is a namespace?** A namespace is a method of organizing and isolating groups of resources within a single cluster. This enables multiple teams, projects, or customers to share a Kubernetes cluster. For more information, refer to the Kubernetes documentation: [Namespaces](https://kubernetes.io/docs/concepts/overview/working-with-objects/namespaces).

## Add users with limited rights

1\. Create a namespace:

```sh
kubectl create ns test-namespace
```

2\. Create a service account and a RoleBinding. Replace `test-namespace`, `test-serviceaccount`, and `test-serviceaccount-rolebinding` with the intended names:

```sh
cat <<EOF | kubectl apply -f -
apiVersion: v1
kind: ServiceAccount
metadata:
  name: test-serviceaccount
  namespace: test-namespace
---
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: test-serviceaccount-rolebinding
  namespace: test-namespace
subjects:
- kind: ServiceAccount
  name: test-serviceaccount
  namespace: test-namespace
roleRef:
  kind: ClusterRole
  name: edit
  apiGroup: rbac.authorization.k8s.io
EOF
```

3\. Create a long-lived token Secret for the service account. Save the following manifest to a file or apply it directly:

```sh
cat <<EOF | kubectl apply -f -
apiVersion: v1
kind: Secret
metadata:
  name: test-serviceaccount-token
  namespace: test-namespace
  annotations:
    kubernetes.io/service-account.name: test-serviceaccount
type: kubernetes.io/service-account-token
EOF
```

Retrieve the decoded token:

```sh
kubectl -n test-namespace get secret test-serviceaccount-token -o jsonpath="{.data.token}" | base64 -d
```
