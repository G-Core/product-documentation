Each pool has minimum and maximum node counts that determine how far autoscaling can scale the pool. The limits can be changed at any time to reflect a different expected workload.

1\. In the [Gcore Customer Portal](https://portal.gcore.com), navigate to **Cloud** > **Managed Kubernetes**, then click the cluster name.

<Frame>![Managed Kubernetes clusters list showing cluster names and status](/images/docs/cloud/kubernetes/clusters/autoscaling/change-autoscaling-limits/change-autoscaling-clusters-list.png)</Frame>

2\. Open the **Pools** section, click **⋯** next to the pool, and select **Edit Pool Settings**.

<Frame>![Pools section with the Edit Pool Settings menu open for pool-1](/images/docs/cloud/kubernetes/clusters/autoscaling/change-autoscaling-limits/change-autoscaling-pool-menu.png)</Frame>

3\. Enter the **Minimum nodes** and **Maximum nodes** values, then click **Save**.

<Frame>![Edit Pool Settings form showing Minimum nodes and Maximum nodes fields](/images/docs/cloud/kubernetes/clusters/autoscaling/change-autoscaling-limits/change-autoscaling-pool-form.png)</Frame>

After saving, the new limits take effect:

- If the specified count is higher than the current number of nodes, the system deploys new ones.
- If the specified count is lower, the system removes redundant ones.
