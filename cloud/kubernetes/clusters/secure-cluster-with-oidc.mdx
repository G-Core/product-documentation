Centralized access management for a Kubernetes cluster can be configured using [OpenID Connect (OIDC)](https://openid.net/developers/how-connect-works/) authentication.

Integrating Kubernetes with an OIDC provider enables a single sign-on (SSO) experience: users can access multiple Kubernetes clusters without re-entering credentials.

OIDC relies on industry-standard security mechanisms, which reduces the risk of unauthorized access or data breaches.

## Enable OIDC authentication for Managed Kubernetes

OIDC authentication can be enabled [during cluster creation](/cloud/kubernetes/clusters/create-a-kubernetes-cluster) or in the settings of an existing cluster. The configuration process is the same in both cases.

To enable OIDC authentication:

1\. If creating a new cluster, scroll down to the **OIDC authentication** section. For an existing cluster, open the cluster overview and click the **Advanced settings** tab.

2\. Click **Enable OIDC authentication** toggle and configure the following settings:

  - **Issuer URL**: Enter the URL of the OIDC provider. For example, `https://login.microsoftonline.com`.

  - **Client ID**: Enter a unique identifier to assign to the cluster.

<Frame>![OpenID Connect authentication settings for a Kubernetes container](/images/docs/cloud/kubernetes/clusters/secure-cluster-with-oidc/oids-issuer-client-id.png)</Frame>

  - **Groups claim** (optional): Specify the name of the claim in the ID token that stores a group name. Without user groups, each user is treated independently.

  - **Set required claims** toggle: Enter any additional claims that must be present in the ID token. If a token is missing any of these claims, authentication fails.

  - **Signing algorithms**: Choose which cryptographic algorithms the API server should accept when verifying ID tokens. By default, tokens must be signed using the RS256 algorithm.

  - **Username claim**: Specify the name of the claim in the ID token that contains the username.

<Frame>![OpenID Connect authentication settings for a Kubernetes container](/images/docs/cloud/kubernetes/clusters/secure-cluster-with-oidc/oids-claims.png)</Frame>

3\. Save the changes.

OIDC authentication is now configured for the cluster.

## Disable OIDC authentication for Kubernetes

<Warning>
Disabling OIDC authentication removes the current configuration. If authentication is re-enabled later, all settings must be reconfigured from scratch.
</Warning>

To disable OIDC authentication:

1\. In the Gcore Customer Portal, navigate to **Cloud** > **Managed Kubernetes**.

2\. Click the cluster name to open it.

3\. Open the **Advanced settings** tab.

4\. Click to expand the **OIDC authentication** section.

5\. Disable the toggle.

<Frame>![Disabled OpenID Connect authentication toggle](/images/docs/cloud/kubernetes/clusters/secure-cluster-with-oidc/disable-oidc.png)</Frame>

6\. Click **Save changes**.

OIDC authentication is now disabled for the cluster.

## Update OIDC authentication settings

<Info>
To reset all OIDC settings to defaults, click **Restore Default**. This undoes all configured OIDC settings.
</Info>

1\. In the Gcore Customer Portal, navigate to **Cloud** > **Managed Kubernetes**.

2\. Click the cluster name to open it.

3\. Open the **Advanced settings** tab.

4\. Click to expand the **OIDC authentication** section.

5\. Adjust the settings as needed.

6\. Click **Save changes**.

The OIDC settings are now updated.
