This article outlines basic steps for setting up a mail server using popular components such as Postfix and Dovecot. Mail server setup may vary depending on the selected software and operating system, as well as the environment's specific settings. This guide focuses on Ubuntu and Debian operating systems.

<Warning>
Outbound traffic on port 25 (SMTP) is blocked by default on Gcore Cloud VMs to prevent spam. Ports 465 and 587 are open. Before setting up a mail server that requires port 25, [contact the Gcore support team](https://gcore.com/contact-us) to request unblocking with a business justification. The use of email services must comply with [Gcore terms of use](https://gcore.com/legal); accounts violating these terms will be blocked.
</Warning>

## Steps

1. Create a Virtual Machine.

Install a Cloud VM using the [VM creation guide](/cloud/virtual-instances/create-an-instance). The minimum available configuration (1vCPU and 2 GB RAM) is sufficient to install and run a mail server.

2. Check internet access.

Check the **Network settings** tab in the machine settings. The interface must be set to the public to access the internet.


<Frame>![Network settings tab in the machine settings](/images/docs/cloud/use-cases/1.png)</Frame>



If the private network method was initially selected, a [floating IP address](/cloud/networking/ip-address/create-and-configure-a-floating-ip-address) is required to access the internet.

<Info>
Ensure port 25 is unblocked before proceeding with mail server installation. See the warning at the top of this guide for details.
</Info>

3. Connect to the Gcore Cloud VM via [SSH](/cloud/virtual-instances/connect/connect-to-your-instance-via-ssh) or the [VNC console](/cloud/virtual-instances/connect/connect-to-your-instance-via-control-panel) in the Customer Portal.

4. Install Postfix.

[Postfix](https://www.postfix.org/) is one of the popular Mail Transfer Agents (MTAs) responsible for sending and delivering mail messages. To install Postfix, run the following commands:

```sh
sudo apt-get update
sudo apt-get install postfix
```

During Postfix installation, a prompt appears to select a configuration type. Select the `Internet Site` option, enter the fully qualified hostname of the server, and click `<Ok>`.


<Frame>![Internet Site option](/images/docs/cloud/use-cases/3.png)</Frame>

5. Configure Postfix.

Navigate to the Postfix configuration file:

```sh
sudo nano /etc/postfix/main.cf
```

In this configuration file, edit the following parameters:

```sh
myhostname = your_server_domain.com
mydestination = your_server_domain.com, localhost.localdomain, localhost
mynetworks = 127.0.0.0/8 [::ffff:127.0.0.0]/104 [::1]/128
```

  * **myhostname** : Specify the fully qualified domain name of the Virtual Server on which Postfix is running.

  * **mydestination** : Specify the domains for which mail will be delivered locally instead of being forwarded to another host.

  * **mynetworks** : Specify a list of networks or IP addresses from which mail messages are allowed to be sent through this server.



<Frame>![Configure Postfix](/images/docs/cloud/use-cases/4.png)</Frame>



Save the changes and restart Postfix:

```sh
sudo service postfix restart
```

More information about Postfix is available in the [Postfix documentation](https://www.postfix.org/documentation.html).

6. Install Dovecot.

[Dovecot](https://www.dovecot.org/) is an IMAP (Internet Message Access Protocol) and POP3 (Post Office Protocol 3) server that provides access to mailboxes. To install Dovecot, run the following command:

```sh
sudo apt-get install dovecot-imapd dovecot-pop3d
```

7. Configure Dovecot.

Add the following SSL parameters:

```sh
listen = *, ::
ssl = required
ssl_cert = </etc/ssl/certs/ssl-cert-snakeoil.pem
ssl_key = </etc/ssl/private/ssl-cert-snakeoil.key
```

Insert the required SSL parameters below, as noted in the screenshot, after `listen = *, ::`.


<Frame>![required SSL parameters](/images/docs/cloud/use-cases/5.png)</Frame>



Save the changes and restart Dovecot:

```sh
sudo service dovecot restart
```

8. Create mailboxes.

After configuring Postfix and Dovecot, create mailboxes for users. For each user, run the following commands:

```sh
sudo useradd -m -s /usr/sbin/nologin username
sudo passwd username
```

Create a password for each user and retype it. The password will be saved automatically.


<Frame>![Create a password](/images/docs/cloud/use-cases/6.png)</Frame>

9. Add DNS records.

Configure DNS settings for the mail server. Refer to the [DNS setup guide](/dns/dns-records/set-up-dns-for-sending-email) for step-by-step instructions.

10. Test and verify.

Restart Postfix and Dovecot to apply the changes. 

```sh
sudo service postfix restart
sudo service dovecot restart
```

Use an email client (for example, [Thunderbird](https://www.thunderbird.net/)) to send a test email from one of the created email addresses. After sending a test email, verify receipt in the email client or webmail. The Postfix and Dovecot logs are located in the following directory:

```sh
/var/log/mail.log
```
