A Load Balancer distributes incoming requests across Virtual Machines and Bare Metal servers to improve infrastructure fault tolerance.

Gcore Load Balancers provide various configuration options to fit different network requirements. Multiple performance tests across available flavors help select the most effective solution for a given infrastructure.

Load Balancers support long (keepalive) connections through Server-Sent Events (SSE), long polling, and WebSockets. To keep connections stable, adjust the data timeout to values appropriate for the application's requirements.

## Performance analysis

The following results are from tests measuring the performance of different flavors.

Each flavor was tested in multithreading mode with 36 concurrent threads and 400 connections over 30 seconds.

The results cover:

- Throughput: the number of requests per second (RPS) a Load Balancer can handle under concurrent load.
- Latency: response times for both HTTP and HTTPS traffic across different Load Balancer flavors.

| 4 vCPU - 8 GiB | 91k | 5 | 51k | 8 |
| 8 vCPU - 16 GiB | 142k | 3 | 117k | 4 |

## Network optimised

For workloads that require high network throughput, network optimised Load Balancers (lb2 flavors) provide guaranteed bandwidth through QoS policies: up to 10&nbsp;Gbps.

These Load Balancers run on dedicated infrastructure with hardware-accelerated networking. The lb2 flavors deliver lower CPU overhead and higher packet rates than standard flavors. Use them when the workload requires guaranteed high throughput or consistent low packet-delivery latency.

| Flavor | vCPU | RAM (GiB) | Network bandwidth |
| ---------- | -------- | ------------- | --------------------- |
| lb2-2-2    | 2        | 2             | 1.5&nbsp;Gbps         |
| lb2-4-4    | 4        | 4             | 5&nbsp;Gbps           |
| lb2-8-8    | 8        | 8             | 10&nbsp;Gbps          |
| lb2-16-16  | 16       | 16            | 10&nbsp;Gbps          |

<Info>
The lb2-2-2 flavor provides baseline 1.5 Gbps bandwidth while retaining the low-latency benefits of network optimised infrastructure.
</Info>
