This guide explains how to enable and configure logging for an existing Load Balancer using the Gcore Logging service. It also provides guaranteed throughput data for different combinations of Load Balancer regions and Logging destination regions.

To create a new Load Balancer, refer to [Create and configure a load balancer](/cloud/networking/create-and-configure-a-load-balancer).

## Step 1. Open load balancers settings

In the **Cloud** menu, select **Networking** and then **Load Balancers**. Click the Load Balancer name to open its settings.


<Frame>
  <img src="/images/docs/cloud/networking/load-balancers/logging-for-load-balancers/logging-load-balancer-10.png" alt="Create Load Balancer" width="70%"/>
</Frame>



## Step 2. Enable logging

Go to the **Logging** tab and turn on the **Enable Logging** toggle.


<Frame>
  <img src="/images/docs/cloud/networking/load-balancers/logging-for-load-balancers/logging-load-balancer-20.png" alt="Enable Logging" width="70%"/>
</Frame>



## Step 3. Configure logging settings

1\. Select a **Destination region** where the logs will be stored. The available destination depends on the region of the Load Balancer.

2\. Create a namespace for the topics.

<Warning>
This setting is available only if [Logging](/cloud/logging-as-a-service/about-logging-as-a-service) has not been activated for the project in the selected destination region.
</Warning> 
<Accordion title="What is a namespace">
Previously, when activating Logging, you could see a non-human-readable string, e.g., `cbc25c71af8d4f7ca6cbbcdc5a4d68fe.your-topic` as a _namespace_ that is used as prefixed for topics. To make this process more user-friendly, we created the possibility of making human-readable namespaces, e.g., `namespace123.your-topic`.

<Caution>
A namespace can only be created for a new project where the default value has not been set.
</Caution>


</Accordion>


To create a namespace, enter the namespace name according to the following restrictions:

  * Lowercase Latin letters and digits can be used
  * Underscore and spaces are prohibited
  * Length is between 2–32
  * The name should be unique for the destination region. If it's already used, the system will suggest inputting another name.



Click **Create**.


<Frame>
  <img src="/images/docs/cloud/networking/load-balancers/logging-for-load-balancers/logging-namespace.png" alt="Configure logging" width="70%"/>
</Frame>



3\. Add a topic. Select an existing one from the list or create a new one. For a new topic, specify the name and the retention period. 

4\. Click **Save changes**.

The Load Balancer will start sending logs to the topic.
