Load Balancer logging records the access logs generated by each listener — incoming connection details, request metadata, and response codes — and sends them to the Gcore Logging service, where they are stored in OpenSearch and accessible through OpenSearch Dashboards. Configuring logging requires an existing [Load Balancer](/cloud/networking/create-and-configure-a-load-balancer).

## Step 1. Open Load Balancer settings

In the **Cloud** menu, select **Networking** and then **Load Balancers**. Click the Load Balancer name to open its settings.


<Frame>
  ![Load Balancers page](/images/docs/cloud/networking/load-balancers/logging-for-load-balancers/logging-load-balancer-10.png)
</Frame>



## Step 2. Enable logging

Go to the **Logging** tab and turn on the **Enable Logging** toggle.


<Frame>
  ![Logging tab](/images/docs/cloud/networking/load-balancers/logging-for-load-balancers/logging-load-balancer-20.png)
</Frame>



## Step 3. Configure logging settings

1\. If no namespace exists for the destination region, the **Create namespace** dialog appears. Enter the namespace name and click **Create**.

<Warning>
The namespace name cannot be changed after creation.
</Warning>

<Accordion title="What is a namespace">
Previously, namespaces appeared as non-human-readable strings — `cbc25c71af8d4f7ca6cbbcdc5a4d68fe.your-topic` — used as prefixes for topics. We added support for human-readable namespaces — `namespace123.your-topic` — to make them easier to identify.
</Accordion>

The namespace name must follow these rules:

  * Use only lowercase Latin letters and digits
  * Spaces and underscores are prohibited
  * Length must be between 2–32 characters
  * The name must be unique for the destination region


<Frame>
  ![Create namespace dialog](/images/docs/cloud/networking/load-balancers/logging-for-load-balancers/logging-create-namespace.png)
</Frame>

2\. Select a **Destination region** where the logs will be stored. The available destinations depend on the region of the Load Balancer.

3\. Add a topic. Select an existing one from the list or create a new one. For a new topic, specify the name and the retention period.

<Frame>
  ![Configure logging settings](/images/docs/cloud/networking/load-balancers/logging-for-load-balancers/logging-namespace.png)
</Frame>

4\. Click **Save changes**.

The Load Balancer will start sending logs to the topic.
