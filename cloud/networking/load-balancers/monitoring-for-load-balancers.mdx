This guide explains the monitoring features available for Gcore Load Balancers and how to configure them for an existing load balancer.

## Prometheus listener

Load balancers use listeners to associate a port with a pool of instances. In addition to the regular balancing listeners, a Prometheus listener exposes an endpoint with the balancer's metrics for scraping with Prometheus and viewing with Grafana.

<Tip>
  Import [this pre-made Grafana dashboard](https://grafana.com/grafana/dashboards/15828-openstack-octavia-amphora-load-balancer/) to display the metrics.
</Tip>

### Enable Prometheus listener

To enable the Prometheus listener, use the Customer Portal:

<Steps>
  <Step title="Open the listener settings">
    To enable the metrics endpoint, open the **Gcore Customer Portal**, navigate to **Cloud** \> **Networking** \> **Load Balancers**, and click the name of a load balancer in the list.

    <img
      src="/images/load-blanacers-monitoring-1.png"
      alt="Load balancers list"
      style={{ width:"62%" }}
    />

    The **Load Balancer** page opens.
  </Step>
  <Step title="Initialize the listener creation">
    Select the **Listeners** tab, scroll down until you see the **Add Listener** button, and click it.

    <img
      src="/images/load-balancers-monitoring-2.png"
      alt="Load balancers details dialog"
      style={{ width:"36%" }}
    />

    The **Add Listener** panel opens.
  </Step>
  <Step title="Configure the listener">
    Enter a **Name**, select **Prometheus** as the **Protocol**, set a **Port** and a **Connection limit**.

    The following options are available to secure the Prometheus listener:

    - Define an allowed IP range with **Allowed CIDRs**.
    - Define a list of users with access through **Basic Auth** (**User List**).
    - Enable SSL/TLS by [adding certificates](/cloud/networking/load-balancers/add-certificates-to-load-balancer).
    - Enable mTLS [via the Gcore Cloud API](/cloud/networking/load-balancers/configure-mtls-authentication).

    <img
      src="/images/load-balancers-monitoring-3.png"
      alt="Add listener dialog"
      style={{ width:"57%" }}
    />
  </Step>
  <Step title="Finalize the listener">
    Click the **Create Listener** button and wait until the Prometheus listener appears in the list.

    Add the listener as a target to a Prometheus instance by adding the following lines to `prometheus.yml` under `scrape_configs`:

    ```yaml
    - job_name: <LOAD_BALANCER_NAME>
        metrics_path: /metrics
        static_configs:
        - targets:
          - <LOAD_BALANCER_IP>:<PROMETHEUS_LISTENER_PORT>
    ```
  </Step>
</Steps>

### Disable Prometheus listener

<Info>
  Before deleting a Prometheus listener, delete the corresponding job from all `prometheus.yml` files.
</Info>

To disable the Prometheus listener, use the **Customer Portal**:

<Steps>
  <Step title="Open the load balancer settings">
    Open the **Gcore Customer Portal**, navigate to **Cloud** \> **Networking** \> **Load Balancers**, and click a load balancer's name in the list.

    <img
      src="/images/load-blanacers-monitoring-1.png"
      alt="Load Balancers Monitoring 1 Pn"
      style={{ width:"58%" }}
    />
  </Step>
  <Step title="Delete the listener">
    Select the **Listeners** tab, scroll down to the Prometheus listener, click the action menu on the right, and click **Delete**.
  </Step>
</Steps>

## Logging as a service

In addition to Prometheus metrics, load balancers can forward listener access logs to Gcore Managed Logging (LaaS). Logs are stored in the same region as the load balancer.

### Enable logging

To enable logging, use the **Customer Portal**:

<Steps>
  <Step title="Open the load balancer settings">
    To enable the logging, open the **Gcore Customer Portal**, navigate to **Cloud** \> **Networking** \> **Load Balancers**, and click the name of a load balancer in the list.

    <img
      src="/images/load-blanacers-monitoring-1.png"
      alt="Load balancers list"
      style={{ width:"63%" }}
    />
  </Step>
  <Step title="Initialize logging">
    Select the **Logging** tab, and activate the **Enable Logging** toggle.

    Select an existing log **Topic** or create a new one to define a **Retention** period. For full configuration details, see [Logging for load balancers](/cloud/networking/load-balancers/logging-for-load-balancers).

    <img
      src="/images/load-balancers-monitoring-4.png"
      alt="Load balancers settings"
      style={{ width:"73%" }}
    />
  </Step>
  <Step title="Finalize logging">
    Click **Save changes**. The Logging service configures log collection for the load balancer.
  </Step>
</Steps>

### View the logs

Logs can be viewed in the **Customer Portal** or in [a Grafana instance](/cloud/logging-as-a-service/view-opensearch-logs-in-Grafana). To view the logs in the **Customer Portal**:

<Steps>
  <Step title="Open Managed Logging">
    Open the **Gcore Customer Portal** and navigate to **Cloud** \> **Managed Logging**.

    <img
      src="/images/load-balancers-monitoring-5.png"
      alt="Managed logging settings"
      style={{ width:"36%" }}
    />
  </Step>
  <Step title="Open OpenSearch dashboard">
    Select a log topic from the list and click the corresponding **View Logs** link.
  </Step>
</Steps>

### Disable logging

To disable logging, use the **Customer Portal**:

<Steps>
  <Step title="Open the load balancer settings">
    Open the **Gcore Customer Portal**, navigate to **Cloud** \> **Networking** \> **Load Balancers**, and click a load balancer's name in the list.

    <img
      src="/images/load-blanacers-monitoring-1.png"
      alt="Load Balancers Monitoring 1 Pn"
      style={{ width:"50%" }}
    />
  </Step>
  <Step title="Deactivate logging">
    Select the **Logging** tab, deactivate the **Enable Logging** toggle, and click the **Save changes** button.

    <img
      src="/images/load-balancers-monitoring-6.png"
      alt="Load Balancers Monitoring 6 Pn"
      style={{ width:"49%" }}
    />
  </Step>
</Steps>

The status endpoints provide details about a load balancer's operating and provisioning status, its listeners, and their pools.
