Gcore offers Bare Metal server types optimized for specific use cases and workload demands. Available configurations depend on the [deployment region](https://gcore.com/cloud/bare-metal-servers). To compare types in the [Gcore Customer Portal](https://portal.gcore.com), open **Cloud** > **Bare Metal**, click **Create Bare Metal**, select a region, and review the **Server type** groups.

- **High frequency** — single-socket servers with high clock-speed CPUs for latency-sensitive workloads — real-time analytics, low-latency applications, and gaming servers.
- **Infrastructure** — multi-socket servers with high core counts for enterprise workloads, large databases, and virtualized environments.
- **Storage** — multi-socket servers with large-capacity drives for data-heavy workloads — archival storage, backup, and big data processing.

## High frequency

High frequency Bare Metal servers suit applications that depend on processor clock speed rather than core count: multiplayer gaming, CAD applications, media processing, and rendering tasks.

## Infrastructure

Infrastructure Bare Metal servers target workloads that need high multi-core performance and extensive multi-threading: large-scale databases, virtualization platforms, container orchestration, and high-performance microservices.

Storage Bare Metal servers combine high core counts with large disk capacity for workloads that read and write large datasets: archival storage, backup servers, media libraries, and big data analytics.
