Gcore offers Bare Metal server types optimized for specific use cases and workload demands. Available configurations depend on the [deployment region](https://gcore.com/cloud/bare-metal-servers). To compare types in the [Gcore Customer Portal](https://portal.gcore.com), open **Cloud** > **Bare Metal**, click **Create Bare Metal**, select a region, and review the **Server type** groups.

* **High frequency** — single-socket servers with high clock-speed CPUs for latency-sensitive workloads such as real-time analytics, low-latency applications, and gaming servers.
* **Infrastructure** — multi-socket servers with high core counts for enterprise workloads, large databases, and virtualized environments.
* **Storage** — multi-socket servers with large-capacity drives for data-heavy workloads such as archival storage, backup, and big data processing.

The sections below describe each type and list representative configurations verified in the Luxembourg-3 region.

## High frequency

Bare Metal servers with high-frequency CPUs suit applications that depend on processor clock speed rather than core count alone.

### Use cases

Multiplayer gaming, CAD applications, media processing, and rendering tasks.

## Infrastructure

These Bare Metal servers target large-scale databases, virtualization platforms, and enterprise applications that need high multi-core performance and extensive multi-threading.

### Use cases

Large-scale databases, virtualization, big data processing, container orchestration, and high-performance microservices.

Type | CPU | CPU cores/threads | RAM | Disk | Network
---|---|---|---|---|---

## Storage

Storage-type servers combine high core counts with large disk capacity for workloads that read and write large datasets.

### Use cases

Archival storage, backup servers, media libraries, and big data analytics that require high local storage capacity.
