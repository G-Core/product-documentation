Link aggregation (LACP bonding) combines multiple Ethernet interfaces into a single logical channel. The feature distributes server load and increases fault tolerance. Gcore uses the standard LACP protocol (IEEE 802.3ad) for channel bonding. The service is available on Dedicated Servers only and is paid — prices are listed on the [LACP&nbsp;tab](https://gcore.com/hosting/miscellaneous).

To activate link aggregation, order the **Port Channel** service from the [Control&nbsp;Panel](https://hosting.gcore.com/billmgr), prepare the server, and then open a support ticket — Gcore configures the network-side LACP after receiving the request.

To order the Port Channel service:

1. Log in to the [Control&nbsp;Panel](https://hosting.gcore.com/billmgr).
2. In the left sidebar, navigate to **Products/Services** > **Port Channel**.

3. Click **Order** next to the Port Channel service.

After ordering, prepare the server as described in the [Server&nbsp;setup](#server-setup) section below, then open a support ticket to request network-side LACP configuration.

## Server setup

Install the `ifenslave` package and configure the kernel bonding module **before opening the support ticket**. The example below uses Ubuntu 14.04.

## Setup example for Ubuntu 14.04

Install the `ifenslave` package **before contacting support**:

```sh
apt-get install ifenslave
```

Add "bonding" to `/etc/modules`.

Load the bonding module:

```sh
modprobe bonding
```

Verify it is loaded — the output should include a line like "bonding 147456 0":

```sh
# lsmod | grep bonding 
bonding 147456 0
```

## Configure LACP on the server

After Gcore has configured LACP on the network side, configure it on the server.

Stop network services:

```sh
stop networking
```

Edit network interfaces settings:

```sh
vim /etc/network/interfaces
```

using the following example (replace em1 and em2 with the actual network interface names):

```
auto em1 
iface em1 inet manual 
bond-master bond0 
bond-primary em1 
 
auto em2 
iface em2 inet manual 
bond-master bond0 
 
auto bond0 
iface bond0 inet static 
 address 92.223.103.25 
 gateway 92.223.103.1 
 netmask 255.255.255.0 
 
bond-mode 4 
bond-miimon 100 
bond-lacp-rate 1 
bond-slaves em1 em2
```

Changes will apply after running:

```sh
start networking
```

If there is no network connection in the bonding, reboot the server.

Check the bond0 status:

```sh
cat /proc/net/bonding/bond0
```
