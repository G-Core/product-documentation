---

SSH provides remote access to CentOS, Debian, Ubuntu, and FreeBSD servers. For Windows servers, use [RDP](/hosting/dedicated-servers/manage/connect/connect-to-a-windows-server) instead.

<Steps>
  <Step title="Open the Instructions tab">
    In the Gcore hosting panel, navigate to **Products/Services** → **Virtual private servers**. Select the server in the list, then click **Instructions** in the toolbar.

    <Frame>![Virtual private servers list in BillMgr with the Instructions button in the toolbar](/images/docs/hosting/virtual-servers/manage/connect/linux-server/connect-to-linux-server-via-ssh/connect-to-linux-server-via-ssh-image1.png)</Frame>

    A new browser tab opens with the server's access credentials. Note the **Server IP address**, **User**, and **Password** values.

    <Frame>![Virtual server activation page showing Server IP address, User, and Password fields](/images/docs/hosting/virtual-servers/manage/connect/linux-server/connect-to-linux-server-via-ssh/connect-to-linux-server-via-ssh-image2.png)</Frame>
  </Step>
  <Step title="Run the SSH command">
    Open a terminal (Linux) or command line (Windows) and run:

    ```sh
    ssh [username]@[server IP]
    ```

    For example, for a server with user root and IP 185.14.67.190, the command is:

    <Frame>![PowerShell terminal with the SSH command: ssh root@185.14.67.190](/images/docs/hosting/virtual-servers/manage/connect/linux-server/connect-to-linux-server-via-ssh/connect-to-linux-server-via-ssh-image3.png)</Frame>
  </Step>
  <Step title="Enter the password">
    When prompted for a password, enter the value from the **Password** field in the instructions. For example, if the password shown is YHr4mhn7hFJeN, enter YHr4mhn7hFJeN.

    <Note>
    In some SSH clients, the password cannot be pasted — it must be typed manually. Entered characters are not echoed to the screen.
    </Note>

    <Frame>![SSH password prompt in the terminal after running the SSH command](/images/docs/hosting/virtual-servers/manage/connect/linux-server/connect-to-linux-server-via-ssh/connect-to-linux-server-via-ssh-image4.png)</Frame>
  </Step>
</Steps>

Once connected, the virtual server is accessible for remote management.
