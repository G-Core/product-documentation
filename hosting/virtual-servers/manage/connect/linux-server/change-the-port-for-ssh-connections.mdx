title: Change the port for SSH connections
sidebarTitle: Change port for SSH connections
ai-navigation: Configure the SSH port by editing the Port directive in sshd_config, restarting the SSH service, and updating the firewall rules.
---

Port 22 is the default for SSH and a frequent target of automated scans — moving to a non-standard port reduces that exposure. The change requires editing `/etc/ssh/sshd_config`, restarting the SSH service, and updating the firewall.

  1. Log in to the server.

  2. Navigate to the SSH directory using the following command:

    ```sh
    cd /etc/ssh/
    ```

  3. Open the SSH configuration file with root privileges using this command:

    ```sh
    sudo nano sshd_config
    ```

  4. Find the **Port** line in the open file and replace the value 22 with the desired port number. Also, remove the `#` symbol if it is present before the **Port** line.


    <Frame>![sshd_config file with the Port line commented out as #Port 22](/images/docs/hosting/virtual-servers/manage/connect/linux-server/change-the-port-for-ssh-connections/change-the-port-for-ssh-connections-image1.png)</Frame>

    <Frame>![sshd_config file with the Port value changed to 2233 and the comment symbol removed](/images/docs/hosting/virtual-servers/manage/connect/linux-server/change-the-port-for-ssh-connections/change-the-port-for-ssh-connections-image2.png)</Frame>


  5. Press **Ctrl+O** to save the changes.

  6. Restart the SSH service using the command for the OS:

    - Ubuntu or Debian:

      ```sh
      sudo systemctl restart ssh
      ```

    - CentOS or AlmaLinux:

      ```sh
      sudo systemctl restart sshd
      ```

    - FreeBSD:

      ```sh
      sudo service sshd restart
      ```

  7. Add a firewall rule to permit traffic on the new port:

    - Ubuntu or Debian (UFW):

      ```sh
      sudo ufw allow 2233
      ```

    - CentOS or AlmaLinux (firewalld):

      ```sh
      sudo firewall-cmd --permanent --add-port=2233/tcp
      sudo firewall-cmd --reload
      ```

    Replace `2233` with the new port number.

  8. From a separate local terminal, verify that the new port is open:

    ```sh
    telnet <server-ip> 2233
    ```

    Replace `<server-ip>` with the server's IP address and `2233` with the new port number.

    <Tip>
    If `telnet` is not installed: run `apt install telnet` on Debian or Ubuntu, or `yum install telnet` on CentOS.
    </Tip>

    A successful response shows the SSH service banner — a line like `SSH-2.0-OpenSSH_8.2p1`; the connection closes immediately after:

    <Frame>![Terminal showing a telnet port probe and a successful SSH login on the new port](/images/docs/hosting/virtual-servers/manage/connect/linux-server/change-the-port-for-ssh-connections/change-the-port-for-ssh-connections-image3.png)</Frame>



  9. In a new terminal window, connect to the server on the new port:

    ```sh
    ssh -p 2233 root@<server-ip>
    ```

    Replace `2233` with the new port number and `<server-ip>` with the server's IP address.

    <Warning>
    Do not close the current session until the new connection is confirmed. If the new port is unreachable, revert the **Port** value to `22` in `sshd_config`, save the file, and restart the SSH service.
    </Warning>
