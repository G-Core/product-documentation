VMmanager 6 includes a set of pre-configured OS templates for reinstalling a virtual server's operating system.

<Warning>
Reinstalling the OS erases all data on the server. Back up any data before proceeding.
</Warning>

To reinstall the OS on a virtual server from a template:

1. Log in to [BillMgr](https://hosting.gcore.com/billmgr) and navigate to **Products/Services** > **Virtual private servers**.

2. Select the server row and click **To panel** — VMmanager opens in a new tab.

<Frame>![Virtual machines list in VMmanager 6 showing one active VM](/images/docs/hosting/virtual-servers/manage/operating-system/install-a-linux-os-from-a-template/install-a-linux-os-from-a-template-image1.png)</Frame>

3. In the **Virtual machines** list, click the VM name to expand the row.

4. Click the hamburger menu icon and select **Reinstall OS** — the reinstall form opens.

<Frame>![Reinstall OS form in VMmanager 6 showing OS templates, Applications and scripts, and VM password fields](/images/docs/hosting/virtual-servers/manage/operating-system/install-a-linux-os-from-a-template/install-a-linux-os-from-a-template-image2.png)</Frame>

5. In the **Operating system** section, select a template:

   - Ubuntu 20.04 (ISPsystem)
   - FreeBSD 13 (ISPsystem)
   - Alma Linux 8 (ISPsystem)
   - Debian 11 (ISPsystem)

6. (Optional) In the **Applications and scripts** section, select one or more scripts to run after the OS is installed. The available options are: Without script, Django, LAMP, LEMP, Nagios, OpenVPN, Route Reflector, Teamspeak, and Tomcat.

7. In the **VM password** field, enter a root password or keep the auto-generated value.

8. Click **Reinstall** to confirm.

Gcore also supports [ISO image installation](/hosting/virtual-servers/manage/operating-system/install-a-linux-os-from-your-iso-image) for operating systems not available as templates.
