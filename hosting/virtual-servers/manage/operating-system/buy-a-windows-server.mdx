Windows Server is available as an operating system option when ordering a Virtual Private Server. It carries an additional monthly license fee and is not available on the smallest plan tiers.

<Note>
Windows Server is not available on KVM-SSD-1, KVM-SSD-2, or KVM-SSD-3 plans. Select any other plan to use Windows Server.
</Note>

## Order a Windows Server

The ordering flow has two stages: select a plan, then configure the server and select the operating system.

<Steps>
  <Step title="Open the plan catalog">
    Log in to the [Hosting&nbsp;portal](https://hosting.gcore.com/billmgr). In the left sidebar, navigate to **Products/Services** > **Virtual private servers**, then click **Order** in the toolbar.
  </Step>
  <Step title="Select a plan and data center">
    Use the **Data center** filter to select a location, then optionally set the **Payment period** filter to compare prices across billing periods.

    Select a plan other than KVM-SSD-1, KVM-SSD-2, or KVM-SSD-3 — Windows Server is not available on those tiers. Click **Order** on the chosen plan card.
  </Step>
  <Step title="Select Windows Server as the operating system">
    On the configuration page, find the **Operating system** field and select **Windows Server**.

    Fill in the remaining fields — domain name, number of IPv4 addresses, and payment period — then click **Pay** to complete the order or **To cart** to save and pay later.
  </Step>
</Steps>

After payment, the server status changes to **Activation in progress**. Provisioning takes up to three hours. When complete, the status changes to **Active** and the server is ready to use.

## Windows Server notes

Keep these limitations in mind before ordering.

- Windows Server can only be installed at order time. It cannot be added to a server that was ordered with a Linux OS.
- Switching from Windows to a Linux OS is possible via an [OS&nbsp;template](/hosting/virtual-servers/manage/operating-system/install-a-linux-os-from-a-template), but the Windows license fee continues to be charged after reinstalling. Ordering a new server with the required OS is the recommended approach instead.
