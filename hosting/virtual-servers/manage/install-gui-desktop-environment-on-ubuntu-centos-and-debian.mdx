A desktop environment provides a graphical interface that displays the system in its familiar appearance — with folders, shortcuts, and a menu. The terminal remains accessible and can be opened to run commands at any time. XFCE is a lightweight desktop environment that runs well on low-resource servers and is compatible with all distributions covered here.

## Install XFCE desktop environment

The steps differ by operating system — follow the section that matches the server OS.

### Install XFCE on Ubuntu

1\. Connect to the server via [SSH](/hosting/virtual-servers/manage/connect/linux-server/connect-to-linux-server-via-ssh) or [VNC](/hosting/virtual-servers/manage/connect/connect-to-a-virtual-server-via-vnc). The login credentials are available on the **Instructions** tab in the [Hosting&nbsp;portal](https://hosting.gcore.com/billmgr).

2\. Update the package list:

```sh
sudo apt-get update
```

3\. Install the XFCE environment and additional components (panel, file manager, and various plugins):

```sh
sudo apt-get install xfce4-session xfce4-goodies
```

The system warns that the installation requires several hundred megabytes of disk space.

<Frame>![Disk space warning](/images/docs/hosting/virtual-servers/manage/install-gui-desktop-environment-on-ubuntu-centos-and-debian/image8.png)</Frame>

Confirm with: `y`

<Frame>![Confirmation prompt](/images/docs/hosting/virtual-servers/manage/install-gui-desktop-environment-on-ubuntu-centos-and-debian/image12.png)</Frame>

4\. Install a display manager to launch XFCE.

On some OS versions, the installer prompts to select between **gdm3** and **lightdm**. Select **lightdm** and press **Enter**.

<Frame>![Display manager selection: lightdm selected](/images/docs/hosting/virtual-servers/manage/install-gui-desktop-environment-on-ubuntu-centos-and-debian/4411242128145.png)</Frame>

On other OS versions, install the display manager manually:

```sh
sudo apt install xinit
```

The system warns that the installation requires several megabytes of disk space.

<Frame>![xinit disk space warning](/images/docs/hosting/virtual-servers/manage/install-gui-desktop-environment-on-ubuntu-centos-and-debian/image20.png)</Frame>

Confirm with: `y`

<Frame>![Confirmation prompt](/images/docs/hosting/virtual-servers/manage/install-gui-desktop-environment-on-ubuntu-centos-and-debian/image24.png)</Frame>

5\. Start the desktop environment.

If `lightdm` was installed in step 4, reboot the server — the graphical interface starts automatically:

```sh
shutdown -r now
```

If `xinit` was installed in step 4, start the graphical interface directly:

```sh
startx
```

6\. On first start, XFCE prompts to select a panel configuration. Select **Use default config** to get the taskbar, clock, and pre-configured Applications menu. The empty panel option requires manual configuration.

<Frame>![XFCE first-start panel configuration dialog](/images/docs/hosting/virtual-servers/manage/install-gui-desktop-environment-on-ubuntu-centos-and-debian/image11.png)</Frame>

The desktop with the default configuration looks as follows (minor details may vary):

<Frame>![XFCE desktop with default configuration](/images/docs/hosting/virtual-servers/manage/install-gui-desktop-environment-on-ubuntu-centos-and-debian/image2.png)</Frame>

If only folders and widgets are visible and the panel with the **Applications** button is missing, the panel is hidden at the top — move the slider up to reveal it.

<Frame>![Slide up to reveal the hidden panel](/images/docs/hosting/virtual-servers/manage/install-gui-desktop-environment-on-ubuntu-centos-and-debian/image33.png)</Frame>

XFCE is now ready. If `lightdm` was installed, the desktop starts automatically on each server boot. If `xinit` was used instead, the terminal opens after a reboot — run `startx` to start XFCE again.

### Install XFCE on CentOS

<Note>
CentOS 8 reached End of Life on December 31, 2021. The official CentOS 8 package repositories are no longer maintained, so installation commands will fail on a stock CentOS 8 system. For new servers, use AlmaLinux or Rocky Linux as supported alternatives. The instructions below apply to CentOS 7.
</Note>

1\. Connect to the server via SSH or VNC. The login credentials are available on the **Instructions** tab in the Hosting portal.

2\. Install EPEL (Extra Packages for Enterprise Linux) — an open repository that contains the XFCE desktop environment packages:

```sh
yum -y install epel-release
```

3\. Install the X Window System — the foundation for graphical interfaces on Linux.

For CentOS 7:

```sh
yum -y groupinstall x11
```

For CentOS 8:

```sh
yum -y groupinstall base-x
```

4\. Install the XFCE desktop environment from the EPEL repository:

```sh
yum --enablerepo=epel -y groups install "Xfce"
```

5\. Create a script for xinit to tell it which desktop environment to launch:

```sh
echo "exec /usr/bin/xfce4-session" >> ~/.xinitrc
```

Start the desktop environment:

```sh
startx
```

<Note>
If `startx` returns an error, re-run step 5 and then run `startx` again.
</Note>

On first start, XFCE prompts to select a panel configuration. Select **Use default config** to get the taskbar, clock, and pre-configured Applications menu.

<Frame>![XFCE first-start panel configuration dialog](/images/docs/hosting/virtual-servers/manage/install-gui-desktop-environment-on-ubuntu-centos-and-debian/image28.png)</Frame>

The desktop with the default configuration looks as follows:

<Frame>![XFCE desktop with default configuration on CentOS](/images/docs/hosting/virtual-servers/manage/install-gui-desktop-environment-on-ubuntu-centos-and-debian/image22.png)</Frame>

If only folders and widgets are visible and the panel is missing, move the slider up to reveal it.

<Frame>![Slide up to reveal the hidden panel](/images/docs/hosting/virtual-servers/manage/install-gui-desktop-environment-on-ubuntu-centos-and-debian/image34.png)</Frame>

XFCE is now ready. After a server restart, the terminal opens by default — run `startx` to start XFCE again.

### Install XFCE on Debian

1\. Connect to the server via SSH or VNC. The login credentials are available on the **Instructions** tab in the Hosting portal.

2\. Install XFCE:

```sh
apt-get install xfce4
```

The system warns that the installation requires several hundred megabytes of disk space.

<Frame>![Disk space warning](/images/docs/hosting/virtual-servers/manage/install-gui-desktop-environment-on-ubuntu-centos-and-debian/image16.png)</Frame>

Confirm with: `y`

3\. Install additional components for XFCE (panel, file manager, and various plugins):

```sh
apt-get install xfce4-goodies
```

The system warns that the installation requires several megabytes of disk space.

<Frame>![xfce4-goodies disk space warning](/images/docs/hosting/virtual-servers/manage/install-gui-desktop-environment-on-ubuntu-centos-and-debian/image27.png)</Frame>

Confirm with: `y`

Start XFCE:

```sh
startx
```

On first start, XFCE prompts to select a panel configuration. Select **Use default config** to get the taskbar, clock, and pre-configured Applications menu.

<Frame>![XFCE first-start panel configuration dialog](/images/docs/hosting/virtual-servers/manage/install-gui-desktop-environment-on-ubuntu-centos-and-debian/image30.png)</Frame>

The desktop with the default configuration looks as follows:

<Frame>![XFCE desktop with default configuration on Debian](/images/docs/hosting/virtual-servers/manage/install-gui-desktop-environment-on-ubuntu-centos-and-debian/image21.png)</Frame>

XFCE is now ready. After a server restart, the terminal opens by default — run `startx` to start XFCE again.

## Terminal in XFCE

Click **Applications** in the upper-left corner and select **Terminal Emulator**. The terminal opens.

<Frame>![XFCE Applications menu with Terminal Emulator selected](/images/docs/hosting/virtual-servers/manage/install-gui-desktop-environment-on-ubuntu-centos-and-debian/image7.png)</Frame>

## Logging out of XFCE

### Ubuntu (lightdm)

Press **Ctrl+Alt+F1** — the session returns to the terminal. The desktop restarts automatically on the next server boot.

### All other distributions

1\. Click **Applications** in the upper-left corner and select **Log Out** from the menu.

<Frame>![XFCE Applications menu with Log Out selected](/images/docs/hosting/virtual-servers/manage/install-gui-desktop-environment-on-ubuntu-centos-and-debian/image19.png)</Frame>

2\. Select **Log Out** in the confirmation window.

<Frame>![Log Out confirmation window](/images/docs/hosting/virtual-servers/manage/install-gui-desktop-environment-on-ubuntu-centos-and-debian/image29.png)</Frame>

The session returns to the terminal. Start XFCE again with `startx`.
