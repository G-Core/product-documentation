An additional IP address provisioned through the Gcore hosting Control Panel does not activate automatically — the operating system must be configured to route traffic through it. Configuration steps differ between Debian/Ubuntu, CentOS, and Windows Server.

## Configure an additional IP address on Debian/Ubuntu

<Info>
These examples use `eth0` as the interface name. The actual name may differ — for example, `ens3` on KVM servers. To confirm the interface name, run `ip link show` in the server's terminal or open **Network settings** in VMmanager.
</Info>

1. Open the network configuration file `/etc/network/interfaces`.
2. Add the following lines to the file:

```sh
auto eth0:N
iface eth0:N inet static
    address X.X.X.X
    netmask 255.255.255.255
```

Replace

- `N` with the interface index, starting at 0 for the first additional address
- `X.X.X.X` with the additional IP address
- `255.255.255.255` with the netmask. To find the netmask, navigate to the Control Panel, select **Virtual private servers** or **Dedicated servers**, select the server, and click **IP addresses**.

To add an IPv6 address as well, append the following:

```sh
iface eth0:N inet6 static
    address 2X01:X:X:X::X
```

3. Save the file.
4. Reboot the server or run the following command:

```sh
systemctl restart networking
```

## Configure an additional IP address on CentOS

<Info>
These examples use `eth0` as the interface name. The actual name may differ — for example, `ens3` on KVM servers. To confirm the interface name, run `ip link show` in the server's terminal or open **Network settings** in VMmanager.
</Info>

1. Create the `ifcfg-eth0:N` file in the `/etc/sysconfig/network-scripts/` directory.
2. Add the following lines to the file:

```sh
DEVICE=eth0:N
ONBOOT=yes
BOOTPROTO=static
IPADDR=X.X.X.X
NETMASK=255.255.255.255
```

Replace `N` with the interface index (starting at 0 for the first additional address), `X.X.X.X` with the additional IP address, and `255.255.255.255` with the netmask. To find the netmask, navigate to the Control Panel, select **Virtual private servers** or **Dedicated servers**, select the server, and click **IP addresses**.

<Frame>
  ![Virtual private servers list with the IP addresses button highlighted](/images/docs/hosting/virtual-servers/manage/networking/additional-ip-addresses/configure-an-additional-ip-address/configure-an-additional-ip-address-image3.png)
</Frame>

<Frame>
  ![IP addresses page showing IP address, Mask, Gateway, Domain, and Type columns](/images/docs/hosting/virtual-servers/manage/networking/additional-ip-addresses/configure-an-additional-ip-address/configure-an-additional-ip-address-image4.png)
</Frame>

To add an IPv6 address as well, append the following:

```sh
IPV6ADDR_SECONDARIES="2X01:X:X:X::X/64"
```

Replace `"2X01:X:X:X::X/64"` with the IPv6 address to add.

3. Save the file.
4. Reboot the server or run the `service network restart` command.

## Configure an additional IP address on Windows Server

<Warning>
Windows Server 2012 reached end of life in October 2023. All steps in this section also apply to Windows Server 2016, 2019, and 2022.
</Warning>

1. Connect to the server via RDP.
2. Click **Start** and open **Server Manager**.
3. In the Server Manager window, click **Local Server**.
4. In the **Server Configuration** page, right-click the network connection to configure and select **Properties**.
5. Select the IP version: **Internet Protocol Version 4 (TCP/IPv4)** for IPv4, or **Internet Protocol Version 6 (TCP/IPv6)** for IPv6.
6. Click **Advanced** and then select **Add** to specify the IP address and the subnet mask.
7. Click **OK** to save the changes, then click **Close**.
