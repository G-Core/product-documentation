Individual settings on a virtual server plan cannot be changed separately — the plan is always replaced as a whole. To upgrade, select a new plan with matching location and equal or higher specifications, then submit a support request.

A plan defines the server's location, vCPU count, RAM, disk size, bandwidth limit, and included traffic. These specifications are listed on each plan card in the plan catalog.


<Frame>![Virtual server plan catalog showing plan specs](/images/docs/hosting/virtual-servers/upgrade-your-virtual-server/upgrade-your-virtual-server-image1.png)</Frame>



<Steps>
  <Step title="Select a new plan">
    Its location must match the current server's location, and its specifications must be equal to or higher than the current plan.
  </Step>
  <Step title="Submit a support request">
    Navigate to **Support** > **Support tickets** in the left menu and click **Add**. Fill in the form:

    - In the **Subject** field, enter a description of the request.
    - In the **Service** dropdown, select the server.
    - In the message body, specify the desired tariff plan name.

    <Frame>![Support tickets section with Add button](/images/docs/hosting/virtual-servers/upgrade-your-virtual-server/upgrade-your-virtual-server-image2.png)</Frame>
  </Step>
</Steps>



We will replace the plan as requested. The following details apply to all upgrades:

  * All data on the server is retained, and the IP address remains unchanged.
  * Plan changes are performed on weekdays, while requests can be submitted any day.
  * A plan upgrade takes 30–60 minutes, during which the server is unavailable while settings are changed and the server is rebooted. The upgrade time can be agreed on in the chat. If no specific time is requested, we will perform the upgrade during the day.
  * The change is permanent — the original plan cannot be restored.
  * If the new plan has more disk space than the current one, new partitions are not automatically mounted — the logical disk partitions must be expanded inside the OS, or a new partition must be created manually.
  * The server price changes when the plan is upgraded; the new price is listed in the plan catalog. If the _minimum order period_ (set when ordering the server and paid in advance, shown in the **Expiration date** column) has not yet expired, the remaining prepaid time is reduced proportionally — a server ordered at $20/month with $60 prepaid for three months, then upgraded to $40/month immediately, leaves only one and a half months of prepaid time at the new rate. The account balance will need to be topped up.
