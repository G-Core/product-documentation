Log in to the [Gcore Hosting Platform](https://hosting.gcore.com/billmgr) and order a virtual server by selecting a plan, configuring it, and completing payment. After payment, the server activates within three hours.

## Steps

The ordering flow has two stages: first select a plan and data center from the plan catalog, then configure the server and complete payment.

1. In the left sidebar, navigate to **Products/Services** > **Virtual private servers**.

<Frame>![Virtual private servers list with the Order button highlighted in the toolbar](/images/docs/hosting/virtual-servers/order-a-virtual-server/order-a-virtual-server-image1.png)</Frame>

2. Click **Order** in the toolbar. The plan selection page opens.

3. Use the **Data center** filter to choose a location. More than 30 data centers are available across Europe, Asia, the Americas, and other regions.

4. (Optional) Use the **Payment period** filter to narrow plans by billing period.

<Frame>![Plan selection page showing plan cards with specs, price, and Order buttons](/images/docs/hosting/virtual-servers/order-a-virtual-server/order-a-virtual-server-image2.png)</Frame>

5. Review the plan cards. Each card shows the plan name, data center, hardware specs (vCPU, RAM, disk, network speed), included IPv4 and IPv6, bandwidth, and monthly price.

6. Click **Order** on the plan you want. The order configuration page opens.

7. On the configuration page, fill in the fields:

   - **Domain name** — enter a domain registered with a third-party provider. The domain must be at least a second-level domain, contain no more than 63 characters per level, and have a total length of no more than 255 characters. If left blank, it defaults to `example.com` and can be changed later.
   - **Operating system** — select an OS. Debian, FreeBSD, Ubuntu, and AlmaLinux are available on all plans at no extra cost. Windows Server is available on select plans at an additional charge.
   - **Public IPv4-address, pcs** — use the slider to add extra IPv4 addresses. The slider range depends on the plan (typically 1 to 3 or 1 to 10).
   - **IPv6 network** — included at no extra cost and enabled by default. Uncheck to disable.
   - **Number of services available for order** — set to more than one to order multiple identically configured servers in a single transaction.

8. In the **Payment period** dropdown on the right, select a billing period. A note shows the minimum order period and the equivalent daily charge.

<Frame>![Order configuration form showing OS selection, IPv4 slider, IPv6 checkbox, payment period, and Pay button](/images/docs/hosting/virtual-servers/order-a-virtual-server/order-a-virtual-server-image3.png)</Frame>

9. Click **Pay** to proceed to payment immediately, or **To cart** to save the order and pay later from the cart.

After payment, the server status changes to **Activation in progress**. Activation takes between 15 minutes and three hours. Once activation completes, the status changes to **Active** and the server is ready to use.

## Server statuses

A newly ordered server passes through three statuses before becoming available.

| Status | Meaning |
|--------|---------|
| Ordered | Payment has not been received yet. Add funds to your account balance and complete payment to proceed. |
| Activation in progress | Payment received. The server is being provisioned. |
| Active | The server is ready to use. |

If the server remains in **Activation in progress** for more than three hours, contact support via **Support** > **Support tickets** in the left sidebar.

## Renewal

After the initial paid period ends, the server switches to [automatic daily billing](/hosting/payments/renew-your-server). The daily charge equals the monthly plan price divided by the number of days in that month.
