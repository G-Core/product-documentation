DCI manager handles ISO images for dedicated servers under **Templates** > **ISO-images**. Upload and mounting errors appear during the [ISO&nbsp;installation](/hosting/dedicated-servers/manage/operating-system/install-a-linux-os-from-your-iso-image) process when the file doesn't meet format or size requirements, or when the IPMI proxy fails to attach the image.

## Upload failure

If the ISO image upload fails, check that the file meets all requirements:

- The file extension must be lowercase: `.iso`, not `.ISO`. Use only letters and numbers in the filename.
- The file size must not exceed 9 GB.
- No more than four ISO images are allowed per account.
- ISO images are automatically deleted after 24 hours.

If the upload still fails, delete the failed image and try again. DCI manager shows a **Delete the ISO-image** button in the error dialog.

## Mounting error

If DCI manager shows the following error when mounting an ISO image on a server:

```
Error occurred when mounting ISO-image on the IPMI proxy server. Script output: ''
```

Delete the ISO image and upload it again. If the error persists, [contact&nbsp;support](/hosting/contact-our-technical-support).
