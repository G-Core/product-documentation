When a server does not respond to ping requests, work through the following checks to identify the cause.

<Steps>
  <Step title="Check for Great Firewall blocking">
    If the server's IP address is unreachable from China, verify whether it is blocked by the Great Firewall using these tools:

    - [PingChinaz](https://ping.chinaz.com) — shows connectivity status in China only
    - [ping.pe](https://ping.pe) — shows worldwide connectivity status, including China

    Due to Chinese network restrictions, Gcore cannot guarantee service availability in China. The Great Firewall randomly blocks IP addresses, but the issue can be resolved by purchasing an [additional&nbsp;IP](/hosting/dedicated-servers/manage/networking/additional-ip-addresses/buy-an-additional-ip-address).
  </Step>
  <Step title="Verify network settings">
    If the OS was installed from an ISO image, the network configuration must be set up manually — otherwise, the IP address will not be available.

    To find the required settings, in the [Hosting&nbsp;portal](https://hosting.gcore.com/billmgr), navigate to **Dedicated servers**, select a server, and click **IP addresses**.

    <Frame>
      <img src="/images/docs/hosting/dedicated-servers/troubleshooting/troubleshoot-a-server-that-is-not-responding-to-ping-requests/troubleshoot-a-server-that-is-not-responding-to-ping-requests-image1.png" alt="Dedicated servers list with the IP addresses button highlighted" width="70%" />
    </Frame>

    Note the IP address, the mask, and the gateway.
  </Step>
  <Step title="Configure the additional IP address">
    If an additional IP address is not responding to ping, confirm it is [configured](/hosting/virtual-servers/manage/networking/additional-ip-addresses/configure-an-additional-ip-address) on the server. To configure an additional IP address, reinstall the OS.
  </Step>
</Steps>
