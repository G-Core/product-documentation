Gcore MCP Server provides approximately 60 tools for managing live streams, video on demand, players, and broadcasts through natural language. Configuration and client setup are covered in the [Gcore MCP Server overview](/account-settings/integrations/gcore-mcp-server-overview).

## Configure for Streaming

The `GCORE_TOOLS` environment variable controls which tools are exposed. Setting `GCORE_TOOLS="streaming.*"` loads all Streaming tools.

```bash
GCORE_TOOLS="streaming.*"
```

Streaming tools follow the `streaming.<resource>.<operation>` naming pattern. The `streams` tools handle live stream management with list, create, get, update, and delete operations. The `videos` tools manage video on demand (VOD) content. The `broadcasts` tools handle scheduled broadcasts, while `players` tools configure video player appearance and behavior. The `playlists` tools organize videos into sequences, and `directories` tools provide content organization. The `quality_sets` tools manage encoding presets, `restreams` tools configure multi-platform distribution, and `statistics` tools retrieve viewing metrics.

## Streaming operations

Natural language requests trigger the corresponding API operations. Listing streams requires no parameters—a request like "List all my live streams" returns stream names, IDs, status, push URLs, and current viewer counts.

Stream creation requires a name and optional quality settings. Requesting "Create a new live stream named 'Product Launch' with HD quality" provisions the stream and returns the push URL, stream key, and playback URL for use in streaming software such as OBS.

VOD management includes video listing and upload. A request to "List all uploaded videos" returns the video library. Uploading with "Upload a video from URL https://example.com/video.mp4 named 'Tutorial Part 1'" initiates processing that typically takes 5-15 minutes depending on video length. Processing status can be checked with "Check the processing status of video 56789".

Broadcast scheduling configures timed stream events. Requesting "Create a scheduled broadcast for stream 12345 starting at 2024-03-15 14:00 UTC" creates a broadcast entry that activates the stream at the specified time.

Player configuration customizes playback experience. A request to "Create a custom video player with autoplay disabled and controls visible" provisions a player configuration. Listing players with "List all player configurations" shows available presets.

Playlist management organizes content sequences. Creating playlists with "Create a playlist named 'Tutorial Series' with videos 56789, 56790, and 56791" groups videos for sequential playback. Adding videos with "Add video 56792 to playlist 'Tutorial Series'" extends existing playlists.

Restreaming distributes streams to external platforms. Requesting "Create a restream from stream 12345 to YouTube with RTMP URL rtmp://a.rtmp.youtube.com/live2" configures simultaneous distribution.

Statistics queries return viewing metrics. "Show viewing statistics for the last 7 days" returns total views, unique viewers, average watch time, peak concurrent viewers, and bandwidth consumption.

## Multi-step workflows

Complex streaming configurations involve multiple tools executed in sequence. Setting up a live event requires creating a stream, configuring quality settings, customizing the player, and scheduling the broadcast. A combined request like "Set up a live streaming event named 'Annual Conference' with HD 1080p quality, custom branded player, and scheduled broadcast for March 20, 2024 at 10:00 UTC" triggers the appropriate tool sequence.

VOD library organization uses directories and batch operations. Requesting "Create directories for Tutorials, Webinars, and Product Demos, then organize existing videos into appropriate directories" structures the content library.

Multi-platform streaming combines restream configuration. A request to "Set up multi-platform restreaming from my main stream to YouTube, Twitch, and Facebook Live" creates restream entries for each destination platform.

## Tool naming patterns

Streaming tool names follow the `streaming.<resource>.<operation>` pattern. The `streaming.streams.list` tool lists all live streams without parameters and returns stream IDs, names, status, and URLs. The `streaming.streams.create` tool requires `name` (the stream name) with optional `quality` (SD, HD, or FHD preset). The `streaming.videos.create` tool requires `name` (the video name) and `url` (the source video URL) for VOD ingestion.
