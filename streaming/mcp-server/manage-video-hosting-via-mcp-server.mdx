Gcore MCP Server provides video-on-demand management through natural language. AI assistants can upload videos, organize libraries, create playlists, and generate subtitles.

<Info>
Before starting, [install and configure Gcore MCP Server](/account-settings/integrations/install-and-configure-gcore-mcp-server) with `GCORE_TOOLS="streaming.*"` to load streaming tools.
</Info>

## How it works

When a request is made in natural language, the AI client calls the appropriate MCP tool and formats the JSON response for readability.

Example request:

```
List my videos
```

The MCP server returns structured data:

```json
{
  "id": 56789,
  "name": "Product Tutorial",
  "duration": 845,
  "status": "ready",
  "views": 12456
}
```

The AI client presents this as readable text:

```
Found 24 videos:

1. Product Tutorial (ID: 56789)
   Duration: 14:05
   Views: 12,456
   Status: ready
   Qualities: 360p, 480p, 720p, 1080p

2. Onboarding Webinar (ID: 56790)
   Duration: 58:32
   Views: 3,892
   Status: ready

3. Q1 Product Demo (ID: 56791)
   Duration: 22:15
   Status: processing (78%)
```

## Upload videos

```
Upload video from https://example.com/training/module1.mp4
```

Response:

```
Video upload started

Source: https://example.com/training/module1.mp4
Name: module1 (from filename)
Status: downloading

Processing pipeline:
  1. Download: in progress
  2. Transcode: pending
  3. Ready: pending

Estimated time: 10-15 minutes
I'll let you know when it's ready.
```

Check progress:

```
What's the status of my video uploads?
```

Response:

```
Video processing status:

Completed:
  ✓ module1.mp4 → Product Tutorial (ID: 56789)
  ✓ module2.mp4 → Advanced Features (ID: 56790)

In progress:
  ⟳ module3.mp4 - transcoding 720p (65%)
  
All videos will be ready in ~5 minutes.
```

For upload options, see [Upload videos](/streaming/video-hosting/upload-a-video-and-embed-it-to-your-app).

## Troubleshooting

```
My video has been processing for 2 hours
```

Response:

```
Checking video status...

Video: Conference Recording
Size: 8.5 GB
Duration: 3 hours 20 minutes
Format: 4K H.265
Status: transcoding 1080p (45%)

This is expected for large 4K files:
  - 4K = 4x more pixels than 1080p
  - H.265 requires more processing
  - 3+ hour duration

Estimated completion: 1-2 more hours

For faster processing next time:
  - Upload in 1080p if 4K not needed
  - Use H.264 codec
  - Split long recordings into parts
```
