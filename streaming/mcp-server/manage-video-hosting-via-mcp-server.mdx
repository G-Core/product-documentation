Gcore MCP Server provides video-on-demand (VOD) management through natural language. AI assistants can upload videos, organize libraries, create playlists, manage subtitles, and track viewing analytics.

<Info>
Before starting, [install and configure Gcore MCP Server](/account-settings/integrations/install-and-configure-gcore-mcp-server) with `GCORE_TOOLS="streaming.*"` to load streaming tools.
</Info>

## How it works

When a request is made in natural language, the AI client calls the appropriate MCP tool and formats the JSON response for readability.

Example request:

```
List my videos
```

The MCP server returns structured data:

```json
{
  "id": 56789,
  "name": "Product Tutorial Part 1",
  "status": "ready",
  "duration": 845,
  "size": 524288000,
  "views": 12456,
  "created_at": "2026-03-15T10:30:00Z",
  "qualities": ["360p", "480p", "720p", "1080p"]
}
```

The AI client presents this as readable text:

```
Found 24 videos in library:

1. Product Tutorial Part 1
   Duration: 14:05
   Size: 500 MB
   Views: 12,456
   Qualities: 360p, 480p, 720p, 1080p
   Status: ready

2. Onboarding Webinar Recording
   Duration: 58:32
   Size: 2.1 GB
   Views: 3,892
   Qualities: 360p, 480p, 720p, 1080p
   Status: ready

3. Q1 Product Demo
   Duration: 22:15
   Size: 890 MB
   Views: 567
   Status: processing (78% complete)
```

## Upload videos

### Upload from URL

The most common method for bulk uploads or migration.

```
Upload a video from https://example.com/training/module1.mp4
```

Response:

```
Starting video upload...

Video queued:
  Source: https://example.com/training/module1.mp4
  Name: module1 (auto-detected from filename)
  Status: downloading

Progress:
  Download: started
  Processing: pending
  Encoding: pending

I'll let you know when it's ready.

Check status: "What's the status of my uploading videos?"
Upload more: "Upload videos from https://example.com/training/module2.mp4 
             and module3.mp4"
```

## Troubleshooting

### Video stuck processing

```
My video has been processing for 2 hours, is something wrong?
```

Response:

```
Checking video processing status...

Video: Large Conference Recording
ID: 56794
Status: processing (stuck at 45%)

Analysis:
  File size: 8.5 GB
  Duration: 3 hours 20 minutes
  Source format: 4K H.265
  
Issue identified: Large 4K file with complex codec

This is expected for very large files. Processing time scales with:
  - File size (8.5 GB is large)
  - Resolution (4K = 4x more pixels than 1080p)
  - Duration (3+ hours)
  - Source codec (H.265 is compute-intensive)

Estimated completion: 1-2 more hours

For faster processing in future:
  - Upload in 1080p when 4K isn't needed
  - Use H.264 codec for faster processing
  - Split long recordings into parts

No action needed - processing will complete.
Check again: "Check status of video 56794"
```
