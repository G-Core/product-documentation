Gcore MCP Server provides video-on-demand management through natural language. AI assistants can upload videos, organize libraries, create playlists, and generate subtitles.

<Info>
Before starting, [install and configure Gcore MCP Server](/account-settings/integrations/install-and-configure-gcore-mcp-server) with `GCORE_TOOLS="streaming.*"` to load streaming tools.
</Info>

## How it works

When a request is made in natural language, the AI client calls the appropriate MCP tool, receives JSON data from the Gcore API, and presents it in readable format.

Example request:

```
List my videos
```

The AI client responds:

```
Found 2 videos:

1. Free DEMO: CHARGE - Blender Open Movie (ID: 13755849)
   Duration: 4:22
   Resolution: 3840x1608 (Ultra-wide 4K)
   Status: ready
   Qualities: 1080p, 720p, 480p, 360p
   HLS: https://1000503.gvideo.io/videos/2675_cC4PEC0AWy1qu03R/master.m3u8

2. Free DEMO: Coffee Run - Blender Open Movie (ID: 13755852)
   Duration: 3:04
   Resolution: 1920x804
   Status: ready
   Qualities: 720p, 480p, 360p
```

## Upload videos

```
Upload video from https://example.com/training/module1.mp4
```

Response:

```
Video upload started

Source: https://example.com/training/module1.mp4
ID: 13756001
Name: module1 (from filename)
Status: downloading

Processing pipeline:
  1. Download: in progress
  2. Transcode: pending
  3. Ready: pending

Estimated time: 10-15 minutes depending on file size.
```

Check progress:

```
What's the status of video 13756001?
```

Response:

```
Video: module1 (ID: 13756001)
Status: ready

Duration: 3:04
Resolution: 1920x1080
Qualities: 1080p, 720p, 480p

Playback: https://1000503.gvideo.io/videos/2675_abc123/master.m3u8
```

For upload options, see [Upload videos](/streaming/video-hosting/upload-a-video-and-embed-it-to-your-app).

## Troubleshooting

```
My video has been processing for 2 hours
```

Response:

```
Checking video status...

Video: Conference Recording
Size: 8.5 GB
Duration: 3 hours 20 minutes
Format: 4K H.265
Status: transcoding 1080p (45%)

This is expected for large 4K files:
  - 4K = 4x more pixels than 1080p
  - H.265 requires more processing
  - 3+ hour duration

Estimated completion: 1-2 more hours

For faster processing next time:
  - Upload in 1080p if 4K not needed
  - Use H.264 codec
  - Split long recordings into parts
```
