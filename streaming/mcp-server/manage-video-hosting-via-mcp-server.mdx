Gcore MCP Server enables AI assistants to manage video-on-demand content through natural language. This article covers video upload, library organization, playlists, subtitles, and analytics.

<Info>
Before starting, [install and configure Gcore MCP Server](/account-settings/integrations/install-and-configure-gcore-mcp-server) with `GCORE_TOOLS="streaming.*"` to load streaming tools.
</Info>

## Video hosting concepts

Video Hosting provides a complete VOD workflow: upload source files, transcode to multiple quality levels, organize in a library, and deliver via global CDN.

**Supported upload methods:**
- URL download - fetch video from any accessible URL
- TUS protocol - resumable uploads for large files
- Google Drive import - direct import from Google Drive

**Delivery formats:**
- HLS (MPEG-TS and CMAF) - adaptive streaming for all devices
- MPEG-DASH - industry standard for adaptive delivery
- MP4 - direct download for offline viewing

**Quality renditions:** Videos are automatically transcoded to multiple quality levels (360p, 480p, 720p, 1080p) for adaptive bitrate playback.

## Upload videos

### Upload from URL

The most efficient method for bulk uploads or migration from other platforms.

```
Upload video from https://example.com/videos/tutorial.mp4
```

The video enters a processing queue:
1. **Download** - fetches the source file
2. **Transcoding** - converts to multiple quality renditions
3. **Ready** - available for playback

Processing time depends on video length and source quality. A 10-minute 1080p video typically processes in 5-10 minutes.

### Batch upload

Upload multiple videos in a single request:

```
Upload videos from https://cdn.example.com/training/module1.mp4, module2.mp4, module3.mp4
```

Videos process in parallel (up to 3 simultaneously) and appear in the library as each completes.

### Check upload status

Monitor processing progress:

```
Show video processing status
```

The response shows each video's current stage (downloading, transcoding, ready) and completion percentage.

### Upload with metadata

Specify name, description, and tags during upload:

```
Upload video from https://example.com/demo.mp4 named "Product Demo Q1" with tags demo, product, 2026
```

Metadata helps organize and search the video library.

## Organize video library

### Create directories

Structure the library with folders:

```
Create directories: Tutorials, Webinars, Product Demos
```

Directories help organize large video collections and can be nested for hierarchical structures.

### Move videos

Organize existing videos into directories:

```
Move "Product Demo" to Product Demos directory
```

Move multiple videos at once:

```
Move all videos tagged "tutorial" to Tutorials directory
```

### Search and filter

Find videos by name, tag, or metadata:

```
Find videos containing "onboarding"
```

```
List videos in Webinars directory
```

```
Show videos uploaded in the last 7 days
```

### Delete videos

Remove videos from the library:

```
Delete video "Old Demo"
```

<Warning>
Deleted videos cannot be recovered. Associated analytics data is also removed.
</Warning>

## Playlists

Playlists group videos for sequential playback. A single playlist can contain up to 512 videos.

### Create a playlist

```
Create playlist "New Employee Training" with videos: Onboarding, Company Overview, Tools Setup
```

Videos play in the specified order. The playlist has its own embed code and playback URL.

### Manage playlist content

Add videos to an existing playlist:

```
Add "Compliance Training" to playlist "New Employee Training"
```

Reorder videos:

```
Move "Company Overview" to position 1 in playlist "New Employee Training"
```

Remove videos:

```
Remove "Old Video" from playlist "New Employee Training"
```

### Playlist playback

Playlists use the same player as individual videos. The player displays:
- Current video title
- Navigation controls to skip between videos
- Progress through the playlist

Embed a playlist on a website:

```
Show embed code for playlist "New Employee Training"
```

## Subtitles and captions

### Generate AI subtitles

Automatic Speech Recognition (ASR) generates subtitles from the audio track:

```
Generate subtitles for video "Product Tutorial"
```

The AI:
1. Extracts audio from the video
2. Performs speech recognition
3. Synchronizes text with video timing
4. Adds subtitles to the video player

Processing time is approximately 1 minute per 10 minutes of video. Subtitles appear in the player's caption menu once ready.

### Translate subtitles

Translate existing subtitles to other languages:

```
Translate subtitles for "Product Tutorial" to Spanish, German, French
```

Translations process in parallel. Each language becomes available in the player's language selector as it completes.

### Supported languages

ASR supports speech recognition for major languages including English, Spanish, German, French, Portuguese, Italian, Dutch, Polish, Russian, Chinese, Japanese, and Korean.

Translation supports additional languages. Check available options:

```
Show available subtitle languages
```

## Video analytics

### View statistics

Analyze viewing patterns and engagement:

```
Show analytics for video "Product Tutorial" for the last 30 days
```

Available metrics:
- Total views and unique viewers
- Average watch time and completion rate
- Viewer geography (country breakdown)
- Device types (desktop, mobile, tablet)
- Quality distribution (what resolution viewers watched)

### Identify engagement issues

Low completion rates indicate content issues. Analyze drop-off points:

```
Show viewer retention for video "Onboarding Webinar"
```

The retention curve shows where viewers stop watching. Common patterns:
- Sharp drop at beginning - thumbnail/title mismatch with content
- Gradual decline - content too long or loses focus
- Drop at specific point - problematic section (too technical, boring, etc.)

### Compare videos

Benchmark performance across the library:

```
Show top performing videos this month
```

```
Compare analytics for videos in Tutorials directory
```

## Content moderation

AI-powered content moderation scans videos for inappropriate content before publishing.

### Run moderation check

```
Check video "User Upload" for inappropriate content
```

Moderation scans for:
- **NSFW content** - nudity and inappropriate imagery
- **Violence** - violent scenes or imagery
- **Offensive language** - profanity in audio

Results indicate whether the video passed or was flagged, with timestamps for any detected issues.

### Automatic moderation

Enable automatic scanning for all uploads:

```
Enable automatic content moderation for new uploads
```

Flagged videos require manual review before becoming publicly available.

## Troubleshooting

### Video stuck in processing

Videos occasionally stall during transcoding. Check status:

```
Show detailed status for video "Large Conference Recording"
```

Processing time scales with:
- File size (larger = longer)
- Resolution (4K takes 4x longer than 1080p)
- Duration (linear scaling)
- Source codec (H.265 is slower than H.264)

If processing exceeds expected time, the video may need re-uploading. Contact support if issues persist.

### Playback buffering

Viewer buffering typically results from:
- Slow viewer connection - ensure all quality renditions are available
- Initial quality too high - reduce default quality setting
- CDN issues - rare, check status page

Analyze a specific video:

```
Check delivery status for video "Product Tutorial"
```

### Missing quality renditions

Videos should have multiple quality levels. If renditions are missing:

```
Show available qualities for video "Product Tutorial"
```

If lower qualities are missing, the source may have been too low resolution. Re-upload with higher quality source material if needed.
