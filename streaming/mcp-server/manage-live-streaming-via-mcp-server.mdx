Gcore MCP Server provides live streaming management through natural language. AI assistants can create streams, configure encoders, set up restreams to social platforms, and monitor broadcasts.

<Info>
Before starting, [install and configure Gcore MCP Server](/account-settings/integrations/install-and-configure-gcore-mcp-server) with `GCORE_TOOLS="streaming.*"` to load streaming tools.
</Info>

## How it works

When a request is made in natural language, the AI client calls the appropriate MCP tool and formats the JSON response for readability.

Example request:

```
List my live streams
```

The MCP server returns structured data:

```json
{
  "id": 12345,
  "name": "Weekly Webinar",
  "status": "active",
  "push_url": "rtmp://vp-push-ed1.gvideo.co/in",
  "stream_key": "12345?s=abc123def456",
  "playback_url": "https://12345.gvideo.io/cmaf/12345/master.m3u8",
  "viewers": 234,
  "recording": true,
  "dvr": true
}
```

The AI client presents this as readable text:

```
Found 3 live streams:

1. Weekly Webinar
   Status: LIVE (234 viewers)
   Push URL: rtmp://vp-push-ed1.gvideo.co/in
   Stream Key: 12345?s=abc123def456
   Recording: enabled
   DVR: enabled

2. Product Launch Event
   Status: offline
   Last broadcast: 2 days ago
   Recording: enabled

3. Training Sessions
   Status: offline
   Scheduled: March 25, 2026 at 14:00 UTC
```

The AI assistant provides step-by-step encoder setup.

```
How do I configure OBS for my stream 45678?
```

Response:

```
OBS Studio configuration for stream 45678:

Step 1: Open OBS Settings → Stream
  Service: Custom
  Server: rtmp://vp-push-ed1.gvideo.co/in
  Stream Key: 45678?s=xyz789abc123

Step 2: Output settings (recommended)
  Output Mode: Advanced
  Encoder: x264 or NVENC (if available)
  Rate Control: CBR
  Bitrate: 4500-6000 kbps (for 1080p)
  Keyframe Interval: 2 seconds

Step 3: Video settings
  Base Resolution: 1920x1080
  Output Resolution: 1920x1080
  FPS: 30 (or 60 for gaming)

Step 4: Audio settings
  Sample Rate: 48 kHz
  Channels: Stereo

Click "Start Streaming" when ready.

Your stream will appear at:
https://player.gvideo.io/streams/45678

```

### Alternative protocols

```
My network is unstable, what's the best protocol?
```

Response:

```
For unstable networks, SRT (Secure Reliable Transport) is recommended.

SRT advantages:
  - Built-in error correction
  - Handles packet loss up to 30%
  - Lower latency than RTMP over bad connections
  - Encrypted by default

SRT configuration for stream 45678:
  URL: srt://vp-push-ed1.gvideo.co:5001
  Stream ID: 45678,s=xyz789abc123

OBS SRT settings:
  Service: Custom
  Server: srt://vp-push-ed1.gvideo.co:5001
  Stream Key: streamid=45678,s=xyz789abc123

Latency buffer:
  Default: 120ms (good for most connections)
  High packet loss: 200-500ms
  
  In OBS: Settings → Advanced → Sources → SRT Latency

For browser-based streaming without encoder software:
"Set up WebRTC streaming for stream 45678"
```

## Set up restreaming

Broadcast simultaneously to multiple platforms.

```
I want to stream to YouTube and Twitch at the same time
```

Response:

```
Setting up multi-platform restreaming...

To restream, I need the RTMP credentials from each platform:

For YouTube:
  1. Go to YouTube Studio → Go Live → Stream
  2. Copy the Stream URL and Stream Key
  
For Twitch:
  1. Go to Twitch Dashboard → Settings → Stream
  2. Copy your Primary Stream Key

Provide the credentials in this format:
"Add YouTube restream with URL rtmp://a.rtmp.youtube.com/live2 
 and key xxxx-xxxx-xxxx-xxxx"
```

After providing credentials:

```
Creating restreams for stream 45678...

Restream 1: YouTube
  ID: rs_101
  Target: rtmp://a.rtmp.youtube.com/live2
  Status: ready
  
Restream 2: Twitch  
  ID: rs_102
  Target: rtmp://live.twitch.tv/app
  Status: ready

Both restreams are configured. When you start your main stream:
  - Gcore receives your stream once
  - Automatically distributes to YouTube and Twitch
  - Each platform shows your content simultaneously

Benefits:
  - Single encoder output (saves bandwidth and CPU)
  - Synchronized across platforms
  - If one platform has issues, others continue

Start streaming to see content on all platforms.
Restream limit: 2 of 3 available (contact support to increase)
```

## Troubleshooting

### Stream not starting

```
I started OBS but the stream isn't showing up
```

Response:

```
Troubleshooting stream connection...

Checking stream 45678:
  Status: waiting for encoder
  Last connection attempt: none in last 5 minutes

Common issues:

1. Incorrect stream key
   Your stream key: 45678?s=xyz789abc123
   Verify this matches exactly in OBS Settings → Stream
   
2. Wrong server URL
   Correct: rtmp://vp-push-ed1.gvideo.co/in
   Check for typos or extra spaces

3. Firewall blocking RTMP
   RTMP uses port 1935
   Ensure your firewall allows outbound on this port
   
4. Stream not enabled
   Status in Gcore: enabled ✓

5. OBS not actually streaming
   Check OBS shows "LIVE" in bottom right
   If "Connecting..." for >30 seconds, check network

Test connection:
  Try the test button in OBS Settings → Stream

If RTMP fails, try SRT:
  srt://vp-push-ed1.gvideo.co:5001?streamid=45678,s=xyz789abc123
```

### Poor video quality

```
My stream looks pixelated even though I set 1080p
```

Response:

```
Analyzing stream 45678 quality...

Current ingest:
  Resolution: 1920x1080 ✓
  Bitrate: 2,500 kbps ⚠️
  
Issue: Bitrate too low for 1080p

Recommended bitrates:
  1080p60: 6,000-8,000 kbps
  1080p30: 4,500-6,000 kbps
  720p60: 4,000-5,000 kbps
  720p30: 2,500-4,000 kbps

Your 2,500 kbps is suitable for 720p, not 1080p.

Solutions:

Option 1: Increase bitrate (if upload allows)
  In OBS: Settings → Output → Bitrate: 5000

Option 2: Reduce resolution
  In OBS: Settings → Video → Output Resolution: 1280x720
  Better 720p quality than pixelated 1080p

Option 3: Check encoder preset
  Slower presets = better quality at same bitrate
  OBS: Settings → Output → Encoder Preset: slow

After changing: restart the stream for settings to apply.
```
