Gcore MCP Server provides live streaming management through natural language. AI assistants can create streams, configure features, set up restreaming, and monitor broadcasts.

<Info>
Before starting, [install and configure Gcore MCP Server](/account-settings/integrations/install-and-configure-gcore-mcp-server) with `GCORE_TOOLS="streaming.*"` to load streaming tools.
</Info>

## How it works

When a request is made in natural language, the AI client calls the appropriate MCP tool and formats the JSON response for readability.

Example request:

```
List my live streams
```

The MCP server returns structured data:

```json
{
  "id": 12345,
  "name": "Weekly Webinar",
  "active": true,
  "push_url": "rtmp://vp-push-ed1.gvideo.co/in",
  "stream_key": "12345?s=abc123",
  "hls_url": "https://12345.gvideo.io/cmaf/12345/master.m3u8"
}
```

The AI client presents this as readable text:

```
Found 2 live streams:

1. Weekly Webinar (ID: 12345)
   Status: active
   Push URL: rtmp://vp-push-ed1.gvideo.co/in
   Stream key: 12345?s=abc123
   Features: recording enabled, DVR enabled

2. Product Demos (ID: 12346)
   Status: offline
   Last broadcast: March 20, 2026
```

## Create a stream

```
Create a live stream for our company town hall
```

Response:

```
Stream created: Company Town Hall
ID: 12347

Encoder settings:
  Server: rtmp://vp-push-ed1.gvideo.co/in
  Stream key: 12347?s=xyz789

Playback URL: https://player.gvideo.io/streams/12347

Default settings applied:
  - Low latency: enabled (2-3 second delay)
  - Recording: disabled
  - DVR: disabled

Configure OBS or your encoder with these settings to start broadcasting.
```

For encoder configuration details, see [OBS setup](/streaming/live-streaming/broadcasting-software/obs) or [SRT protocol](/streaming/live-streaming/protocols/srt).

## Enable recording and DVR

```
Enable recording and DVR for stream "Weekly Webinar"
```

Response:

```
Updated stream: Weekly Webinar

Recording: enabled
  - Stream will be saved as VOD after broadcast ends
  - Available in Video Hosting within 5-15 minutes

DVR: enabled  
  - Viewers can pause and rewind up to 1 hour
  - Takes ~10 minutes after stream starts to accumulate buffer

Remember to clear DVR cache before each new broadcast to prevent 
viewers from seeing previous content.
```

For feature details, see [Recording](/streaming/live-streaming/record-your-live-streams-and-save-them-as-videos) and [DVR](/streaming/live-streaming/pause-and-rewind-the-live-streams).

## Set up restreaming

Broadcast simultaneously to YouTube, Twitch, or other platforms.

```
Set up restream from "Weekly Webinar" to YouTube
```

Response:

```
To create a restream, I need the RTMP credentials from YouTube.

In YouTube Studio:
1. Go Live → Stream → copy Stream URL and Stream key

Then tell me:
"Create restream with URL rtmp://a.rtmp.youtube.com/live2 and key xxxx-xxxx"
```

After providing credentials:

```
Restream created for Weekly Webinar → YouTube

When you start streaming:
  - Gcore receives your stream
  - Automatically forwards to YouTube
  - Both platforms show your content simultaneously

Status: ready (will activate when main stream starts)
```

For platform setup, see [Restreaming](/streaming/live-streaming/create-and-configure-a-restream-to-social-media).

## Create clips

Capture highlights during a live broadcast.

```
Create a 60-second clip from the current stream
```

Response:

```
Clip created from Weekly Webinar

Duration: 60 seconds
Status: processing (ready in 1-2 minutes)

Playback URL: https://player.gvideo.io/clips/789

The clip is saved permanently as VOD.
```

## Troubleshooting

```
My stream isn't showing up after I started OBS
```

Response:

```
Checking stream "Weekly Webinar"...

Status: waiting for encoder connection

Troubleshooting steps:
1. Verify stream key: 12345?s=abc123
   - Check for typos or extra spaces in OBS

2. Verify server URL: rtmp://vp-push-ed1.gvideo.co/in
   - Ensure "in" is included at the end

3. Check OBS is actually streaming
   - Bottom right should show "LIVE" not "Connecting..."

4. Firewall
   - RTMP uses port 1935, ensure it's not blocked

If using SRT instead:
  srt://vp-push-ed1.gvideo.co:5001?streamid=12345,s=abc123
```
