Gcore MCP Server provides live streaming tools that AI clients can invoke through natural language. The server returns structured data that the AI client interprets and presents in readable format.

<Info>
Before starting, [install and configure Gcore MCP Server](/account-settings/integrations/install-and-configure-gcore-mcp-server) with `GCORE_TOOLS="streaming.*"` to load streaming tools.
</Info>

## Available tools

| Tool | Description |
|------|-------------|
| `streaming_streams_ls` | List all live streams |
| `streaming_streams_get` | Get stream details by ID |
| `streaming_streams_new` | Create a new stream |
| `streaming_streams_upd` | Update stream settings |
| `streaming_streams_del` | Delete a stream |
| `streaming_restreams_ls` | List restreams |
| `streaming_restreams_new` | Create restream to external platform |
| `streaming_restreams_del` | Delete a restream |
| `streaming_broadcasts_ls` | List scheduled broadcasts |
| `streaming_broadcasts_new` | Schedule a broadcast |
| `streaming_statistics_get` | Get viewing statistics |

## View streams

List all live streams:

```
List my live streams
```

Get details for a specific stream:

```
Show details for stream 12345
```

The response includes: stream ID, name, status (active/ready/offline), push URL, stream key, playback URLs, and enabled features (recording, DVR, low latency).

## Create a stream

```
Create a live stream named "Weekly Webinar"
```

The response includes push URL and stream key for encoder configuration. For encoder setup details, see [OBS configuration](/streaming/live-streaming/broadcasting-software/obs) or [SRT protocol](/streaming/live-streaming/protocols/srt).

## Enable features

Enable recording to save the stream as VOD:

```
Enable recording for stream 12345
```

Enable DVR to allow viewers to pause and rewind:

```
Enable DVR for stream 12345
```

Clear DVR cache before a new broadcast:

```
Clear DVR cache for stream 12345
```

For feature details, see [Recording](/streaming/live-streaming/record-your-live-streams-and-save-them-as-videos) and [DVR](/streaming/live-streaming/pause-and-rewind-the-live-streams).

## Restreaming

Create a restream to broadcast simultaneously on external platforms:

```
Create restream for stream 12345 to rtmp://a.rtmp.youtube.com/live2 with key xxxx-xxxx
```

List configured restreams:

```
List restreams for stream 12345
```

For platform setup, see [Restreaming](/streaming/live-streaming/create-and-configure-a-restream-to-social-media).

## Schedule broadcasts

Schedule a stream to start at a specific time:

```
Schedule broadcast for stream 12345 on April 15 at 14:00 UTC
```

## View statistics

Get viewing metrics:

```
Show statistics for stream 12345 for the last 7 days
```

Available metrics include viewer counts, peak concurrent viewers, watch time, and geographic distribution.

## Common requests

| Request | Tool called |
|---------|-------------|
| "List my live streams" | `streaming_streams_ls` |
| "Create stream named X" | `streaming_streams_new` |
| "Enable recording for stream 123" | `streaming_streams_upd` |
| "Create restream to YouTube" | `streaming_restreams_new` |
| "Show stream statistics" | `streaming_statistics_get` |
| "Delete stream 123" | `streaming_streams_del` |

## Troubleshooting

If a request fails, the AI client displays the error message. Common issues:

- **Stream not found**: Verify the stream ID exists
- **Permission denied**: Check API token permissions
- **Feature unavailable**: Some features require plan upgrade

For operations not available through MCP, use the [Gcore Customer Portal](https://streaming.gcore.com) or [Streaming API](/api-reference/streaming).
