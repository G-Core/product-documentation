## Introduction

**[Gcore Video Streaming](https://gcore.com/streaming-platform)** is a global high-load streaming PaaS for streaming at massive scale.

The platform enables live and on-demand delivery at global scale, handling over a million concurrent viewers with ultra-low latency. It supports LL-HLS, LL-DASH, HLS MPEG-TS, HLS CMAF, MPEG-DASH CMAF, and MP4. And adds value with AI and computer vision, live overlays, playlists, subtitles, restreams, recording, live clipping, and full playback statistics. Video streams are available from SD and HD up to 8K, ensuring reliable delivery across every device and network.

For small business and startups:
	- Quick start with iFrame or REST API integration.
	- Free Full-HD transcoding and PAYG pricing reduce entry cost.
	- Upgrade path from trial to scale—grow at your own pace.
	- Basic analytics available.

For enterprise:
	- Proven ability to support 1M+ concurrent live viewers, 100M+ total.
	- Enterprise-grade security (DRM, tokens, encryption, GDPR).
	- Anti-DDoS, WAF/WAAP, global edge protection.
	- Observability & QoE, SLA, uptime guarantees, priority enterprise support.

## Architecture 

Gcore combines a global CDN with 200+ PoPs and a high-load streaming PaaS to deliver flawless live and on-demand video to audiences from thousands to millions.

The Gcore Video Streaming has Cloud layered architecture = control plane + ingest gateways + processing/transcoding + packaging + origin/storage + CDN delivery + player SDKs + security + analytics. All layers are managed internally by Gcore.


## API

The Video Streaming API is accessible through Gcore’s public API endpoints. 
All endpoints are available over HTTPS and require authentication with your Gcore API key.

Full reference documentation, including request/response schemas and usage examples, is available at https://gcore.com/docs/api-reference/streaming/
