The Application Catalog provides pre-built open-source AI models that you can deploy in a few clicks without building or configuring a container image. For a full list of available models, see the [Application Catalog](/edge-ai/everywhere-inference/application-catalog).

<Info>
**Info**

You might have to [request a quota increase](/edge-ai/everywhere-inference/quotas/request-quota-increase) before deploying a new model.
</Info>

## Start a deployment

There are two ways to open the **Create Deployment** form:

- **From the Deployments page**: navigate to **Everywhere Inference** > **Deployments** and click **Deploy application from catalog**.
- **From the Application Catalog**: open a model detail page and click **Deploy Application**. The form opens with the selected model pre-filled.

<Frame>![Deployments page showing the Deploy application from catalog button](/images/docs/edge-ai/everywhere-inference/ai-models/deploy-an-ai-model/deploy-an-ai-model-1.png)</Frame>

## Configure the deployment

1\. In the **Create Deployment** form, under **Deployment Configuration**, click the **Application** dropdown and select the model you want to deploy.

<Frame>![Create Deployment form with empty Application dropdown](/images/docs/edge-ai/everywhere-inference/ai-models/deploy-an-ai-model/deploy-an-ai-model-2.png)</Frame>

<Frame>![Application dropdown open showing the list of available models](/images/docs/edge-ai/everywhere-inference/ai-models/deploy-an-ai-model/deploy-an-ai-model-3.png)</Frame>

2\. Under **Routing placement**, click **Select region** and choose the regions where the model will run. You can select up to six regions.

3\. Under **Application modules**, configure compute resources:

- **Flavor type** — select **CPU-optimized** or **GPU-optimized** depending on the model requirements.
- **Flavor** — select the hardware configuration from the dropdown (for example, 1xA100 / 16 vCPU / 232 GiB RAM).
- **Minimum pods** — the minimum number of pods to keep running during low-traffic periods.
- **Maximum pods** — the maximum number of pods that can be provisioned during peak traffic.

<Info>
**Info**

The main application module is pre-selected and required. The **Expose** toggle controls whether the module endpoint is publicly accessible.
</Info>

4\. Under **Deployment details**, enter a name for the deployment.

5\. (Optional) Under **Additional options**, enable the **Enable API Key authentication** toggle to restrict access using [API keys](/edge-ai/everywhere-inference/api-keys/create-inference-deployment-with-auth).

6\. Review the estimated cost in the right panel, then click **Deploy model**.

After the deployment is created, Gcore redirects you to the **Deployments** page where you can monitor the deployment status.
