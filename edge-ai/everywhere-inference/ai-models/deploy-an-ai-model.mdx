---

With Gcore Everywhere Inference, you can deploy pre-built open-source models from the [Application Catalog](/edge-ai/everywhere-inference/application-catalog) or deploy a custom model from a Docker container image.

<Info>
**Info**

You might have to [request a quota increase](/edge-ai/everywhere-inference/quotas/request-quota-increase) before deploying a new model.
</Info>

<Tabs>
  <Tab title="Deploy from catalog">

  The catalog deployment flow uses the **Create Deployment** form, where you select a model from the Application Catalog and configure compute resources, regions, and deployment settings.

  You can also start a catalog deployment directly from the Application Catalog — navigate to a model detail page and click **Deploy Application**. The form opens with the selected model pre-filled.

  1\. In the Gcore Customer Portal, navigate to **Everywhere Inference** > **Deployments**.

  <Frame>![Deployments page with Deploy application from catalog and Deploy custom inference buttons](/images/docs/edge-ai/everywhere-inference/ai-models/deploy-an-ai-model/deploy-an-ai-model-1.png)</Frame>

  2\. Click **Deploy application from catalog** in the top-right corner.

  The **Create Deployment** form opens.

  <Frame>![Create Deployment form with empty Application dropdown](/images/docs/edge-ai/everywhere-inference/ai-models/deploy-an-ai-model/deploy-an-ai-model-2.png)</Frame>

  3\. Under **Deployment Configuration**, click the **Application** dropdown and select the model you want to deploy.

  <Frame>![Application dropdown open showing list of available models](/images/docs/edge-ai/everywhere-inference/ai-models/deploy-an-ai-model/deploy-an-ai-model-3.png)</Frame>

  4\. Under **Routing placement**, select the regions where the model will run. You can select up to six regions.

  5\. Under **Application modules**, configure compute resources for the model:

  - **Flavor type** — select **CPU-optimized** or **GPU-optimized**.
  - **Flavor** — select the hardware configuration from the dropdown (for example, 1xA100 / 16 vCPU / 232 GiB RAM).
  - **Minimum pods** — the minimum number of pods to maintain during low-traffic periods.
  - **Maximum pods** — the maximum number of pods that can be provisioned during peak traffic.

  6\. Under **Deployment details**, enter a name for the deployment.

  7\. (Optional) Under **Additional options**, enable the **Enable API Key authentication** toggle to restrict access using [API keys](/edge-ai/everywhere-inference/api-keys/create-inference-deployment-with-auth).

  8\. Review the estimated cost in the right panel, then click **Deploy model**.

  After the deployment is created, Gcore redirects you to the **Deployments** page where you can monitor the deployment status.

  </Tab>
  <Tab title="Deploy a custom model">

  The custom model deployment flow lets you deploy any Docker container image from a public or private registry. Before you start, make sure your container image is ready — refer to [Prepare a custom model for deployment](/edge-ai/everywhere-inference/ai-models/prepare-a-custom-ai-model-for-deployment) for requirements and instructions.

  1\. In the Gcore Customer Portal, navigate to **Everywhere Inference** > **Deployments**.

  <Frame>![Deployments page with Deploy application from catalog and Deploy custom inference buttons](/images/docs/edge-ai/everywhere-inference/ai-models/deploy-an-ai-model/deploy-an-ai-model-1.png)</Frame>

  2\. Click **Deploy custom inference** in the top-right corner.

  The **Deploy custom model** form opens.

  3\. Under **Model image**, configure the container image source:

  **Public registry**: Select **Public**, then enter the **Model image URL (docker tag)** and the **Container port** where the model listens for requests.

  <Frame>![Model image section with Public registry type selected, showing URL and port fields](/images/docs/edge-ai/everywhere-inference/ai-models/deploy-an-ai-model/deploy-an-ai-model-4.png)</Frame>

  **Private registry**: Select **Private**, select the registry from the **Registry** dropdown, then enter the **Model image URL (docker tag)** and the **Container port**.

  <Frame>![Model image section with Private registry type selected, showing Registry dropdown and Add registry link](/images/docs/edge-ai/everywhere-inference/ai-models/deploy-an-ai-model/deploy-an-ai-model-8.png)</Frame>

  If you haven't added a private registry yet, click **+ Add registry**. This opens a dialog where you can enter the registry URL and credentials. Refer to the [registry guide](/edge-ai/everywhere-inference/container-image-registries/add-a-registry) for details.

  (Optional) Enable the **Set startup command** toggle to specify a command that runs when the container starts.

  4\. Under **Pod configuration**, select the compute resources for the deployment:

  - **Flavor type** — select **CPU-optimized** or **GPU-optimized**.
  - **Flavor** — select the hardware configuration from the dropdown.

  The following flavor parameters are recommended based on model size:

  | **Billion parameters** | **Recommended flavor** |
  |---|---|
  | < 21 | 1 x L40S 48 GB |
  | 21–41 | 2 x L40S 48 GB |
  | > 41 | 4 x L40S 48 GB |

  5\. Under **Routing placement**, select the regions where the model will run. You can select up to six regions.

  <Frame>![Routing placement and Autoscaling limits sections](/images/docs/edge-ai/everywhere-inference/ai-models/deploy-an-ai-model/deploy-an-ai-model-5.png)</Frame>

  6\. Under **Autoscaling limits**, configure pod scaling:

  - **All selected regions** — applies the same autoscaling settings to all selected regions.
  - **Custom** — applies different settings per region.
  - **Minimum pods** — the minimum number of pods to maintain during low-traffic periods.
  - **Maximum pods** — the maximum number of pods that can be provisioned during peak traffic.
  - **Cooldown period** — the time (in seconds) the autoscaler waits after a scaling event before making another adjustment.
  - **Pod lifetime** — the time (in seconds) before an idle pod is deleted after its last request. A value of zero means the pod scales down after approximately one minute.

  Under **Autoscaling triggers**, define the conditions that trigger pod provisioning. By default, CPU utilization and GPU utilization triggers are included with an 80% threshold. Click **Add trigger** to add more triggers.

  <Frame>![Autoscaling triggers and Health checks sections](/images/docs/edge-ai/everywhere-inference/ai-models/deploy-an-ai-model/deploy-an-ai-model-6.png)</Frame>

  7\. (Optional) Under **Health checks**, enable probes to monitor container availability:

  - **Liveness probe** — restarts the container if it becomes unresponsive.
  - **Readiness probe** — removes the container from load balancing until it is ready to serve traffic.
  - **Startup probe** — delays other probes until the container finishes starting up.

  8\. Under **Deployment details**, enter a deployment name. You can also add an optional description.

  <Frame>![Deployment details and Additional options sections](/images/docs/edge-ai/everywhere-inference/ai-models/deploy-an-ai-model/deploy-an-ai-model-7.png)</Frame>

  9\. (Optional) Under **Additional options**:

  - Enable **Set environment variables** to pass key-value pairs to the container at runtime.
  - Enable **Enable API Key authentication** to restrict access using [API keys](/edge-ai/everywhere-inference/api-keys/create-inference-deployment-with-auth).

  10\. Review the estimated cost in the right panel, then click **Deploy model**.

  After the deployment is created, Gcore redirects you to the **Deployments** page where you can monitor the deployment status.

  </Tab>
</Tabs>
