# Gclaw overview

Gclaw is a Gcore-hosted, fully managed version of [OpenClaw](https://openclaw.ai/) — a popular open-source AI agent framework with over 220,000 GitHub stars — with built-in inference powered by Gcore GPU infrastructure.

Click **Start** and immediately receive a fully functional OpenClaw instance with inference already connected. No VPS setup, no SSH, no separate inference provider, no token management required.

<Info>
Gclaw is currently in beta. During the beta period, each account is limited to one Gclaw instance, and service levels, guarantees, or support commitments do not apply.
</Info>

## Key features

Gclaw provides the following capabilities:

- **One-click launch**: Get a fully configured OpenClaw instance in seconds. Click **Start** and begin chatting within approximately one minute.

- **Built-in inference**: Inference is already connected and running using the Kimi model on Gcore H200 GPUs. You do not need to find a provider, manage API tokens, or optimize usage.

- **Dedicated environment**: Each user gets an isolated virtual machine with a personal OpenClaw instance. Your data and configuration are fully separated from others.

- **EU data residency**: The inference model runs on Gcore GPUs on-premise in the EU. All chat history and personal data reside solely on your dedicated VM. Gcore does not store your data centrally.

- **Messaging integrations**: Connect your AI agent to Telegram, Discord, WhatsApp, and other messaging platforms.

## What's included

Gclaw includes the following third-party products:

- [OpenClaw](https://openclaw.ai/) - An open-source AI agent framework
- [Kimi](https://github.com/MoonshotAI/Kimi-K2.5) - A large language model for inference

These are third-party products and should be considered Third Party Products under the Master Services Agreement. Gcore assumes no liability for any technical errors, security vulnerabilities, incompatibilities, instabilities, or actions of such Third Party Products.

## Support scope

Gcore provides support for infrastructure and billing issues. For OpenClaw-specific questions, refer to the [official OpenClaw documentation](https://docs.openclaw.ai/).

| Issue type | Gcore support |
|------------|---------------|
| Service down or 500 errors | Yes |
| Billing and subscription issues | Yes |
| OpenClaw configuration questions | No - refer to OpenClaw docs |
| Feature requests | No - logged for consideration |

If your instance is not responding, try clicking **Restart Instance** in the Settings panel before contacting support.
