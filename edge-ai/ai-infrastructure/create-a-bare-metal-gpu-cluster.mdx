<p>Bare Metal GPU clusters provision one or more dedicated GPU servers with no virtualization layer. They are intended for AI training, inference, distributed workloads, and other applications that require full hardware performance. Two cluster types are available: **Bare Metal GPU** for on-demand dedicated servers, and **[Spot&nbsp;GPU](/edge-ai/ai-infrastructure/spot-bare-metal-gpu)** for discounted interruptible capacity that may be reclaimed with 24 hours' notice. Available GPU models and cluster types vary by region.</p>

## Cluster architecture

<p>A cluster consists of one or more bare metal GPU servers provisioned from an identical template. All nodes share the same image, flavor, and network configuration.</p>

```
Cluster
 ├─ Node 1  (GPU server)
 ├─ Node 2  (GPU server)
 ├─ ...
 ├─ InfiniBand fabric  (auto-configured when the flavor supports it)
 └─ File Share         (optional — shared storage across all nodes)
```

<p>InfiniBand inter-node networking is configured automatically when the selected flavor supports it, enabling efficient distributed training without manual network setup. File shares mount across all nodes and persist independently of cluster state.</p>

## Create a GPU cluster

<p>In the [Gcore Customer Portal](https://portal.gcore.com), navigate to **GPU Cloud**, open **GPU Clusters** > **Bare Metal GPU Clusters**, and click **Create Cluster**.</p>

<Steps>
  <Step title="Select region">
    In the **Region** section, select the data center location for the cluster. Regions are grouped by geography (Asia-Pacific, EMEA). Each region card shows its availability status — file share integration, for instance, is available only in select regions.

    <Frame>
      <img src="/images/docs/edge-ai/ai-infrastructure/create-a-bare-metal-gpu-cluster/gpu-cluster-region.png" alt="Region selection section showing available regions grouped by geography" />
    </Frame>

    <Info>
    GPU model availability and pricing vary by region. Check other regions if a required GPU model is not available. For help with availability, contact the [sales&nbsp;team](https://gcore.com/contact-sales).
    </Info>
  </Step>
  <Step title="Configure cluster capacity">
    In the **Cluster capacity** section, select the GPU model and flavor that define the hardware configuration for every node. Available options depend on the selected region.

    - **GPU Cluster type**: Select **Bare Metal GPU** for dedicated servers, or **Spot Bare Metal GPU** for discounted interruptible capacity — pricing and reclamation details are on the [Spot&nbsp;GPU](/edge-ai/ai-infrastructure/spot-bare-metal-gpu) page.
    - **GPU Model**: A100, H100, or H200 — availability depends on the region.
    - **Show out of stock**: Toggle to show or hide unavailable flavors.
    - **Flavor**: Each card displays GPU configuration, CPU type, RAM, storage, network connectivity, and pricing.

    <Frame>
      <img src="/images/docs/edge-ai/ai-infrastructure/create-a-bare-metal-gpu-cluster/gpu-cluster-capacity.png" alt="Cluster capacity section showing GPU Cluster type, GPU Model selector, and flavor card with specifications" />
    </Frame>
  </Step>
  <Step title="Set the number of instances">
    In the **Number of Instances** section, specify how many servers to provision. Each instance is a separate physical server with the selected flavor configuration. The maximum depends on regional stock availability — there is no fixed per-cluster limit.

    <Frame>
      <img src="/images/docs/edge-ai/ai-infrastructure/create-a-bare-metal-gpu-cluster/gpu-cluster-instances.png" alt="Number of Instances section with instance counter" />
    </Frame>

    <Info>
    After creation, the cluster can be resized. Scaling up adds nodes with the same configuration used at creation. Scaling down removes a random node — to delete a specific node, use the per-node delete action in the cluster details. Deleting the last node deletes the entire cluster.
    </Info>
  </Step>
  <Step title="Select image">
    The image defines the operating system and pre-installed software for cluster nodes.

    In the **Image** section, select the operating system:

    - **Public**: Pre-configured images with NVIDIA drivers and CUDA toolkit (recommended)
    - **Custom**: Custom images uploaded to the account

    Images with the `eni` suffix are configured for InfiniBand interconnect.

    <Frame>
      <img src="/images/docs/edge-ai/ai-infrastructure/create-a-bare-metal-gpu-cluster/gpu-cluster-image.png" alt="Image section with Public and Custom tabs and image selector" />
    </Frame>
  </Step>
  <Step title="Configure file share integration (optional)">
    File shares provide shared storage accessible from all cluster nodes. GPU-enabled regions use VAST-backed high-performance shares. This option appears only in regions where file shares are available — — [file&nbsp;shares](/cloud/file-shares/configure-file-shares) documentation describes share types, sizing, and access rules.

    1. Enable the **File Share integration** checkbox.
    2. Select an existing file share or create a new one.
    3. Specify the mount path on cluster nodes (default: `/home/ubuntu/mnt/nfs`).

    <Frame>
      <img src="/images/docs/edge-ai/ai-infrastructure/create-a-bare-metal-gpu-cluster/gpu-cluster-file-share.png" alt="File share integration section with Enable File Share checkbox" />
    </Frame>

    <Info>
    When file share integration is enabled, the cluster automatically adds mount commands to the user data script. If **User data** is also configured in Additional options, do not remove these commands — they handle automatic mounting on first boot.
    </Info>
  </Step>
  <Step title="Configure network settings">
    Network settings define how the cluster communicates with external services and other resources. At least one interface is required.

    In the **Network settings** section, select the interface type:

    | Type | Access | Use case |
    |------|--------|----------|
    | **Public** | Direct internet access with dynamic public IP | Development, testing, quick access to cluster |
    | **Private** | Internal network only, no external access | Production workloads, security-sensitive environments |
    | **Dedicated public** | Reserved static public IP | Production APIs, services requiring stable endpoints |

    To add more interfaces, click **Add Interface**. All public interfaces include Basic DDoS Protection at no additional cost.

    Each interface can be expanded to configure [security&nbsp;groups](/cloud/networking/add-and-configure-a-firewall). The `default` security group is pre-selected — if no security group is selected, it is attached automatically.

    <Frame>
      <img src="/images/docs/edge-ai/ai-infrastructure/create-a-bare-metal-gpu-cluster/network-settings-security-groups.png" alt="Security Groups field with default group selected per interface" />
    </Frame>
  </Step>
  <Step title="Configure SSH key">
    In the **SSH key** section, select an existing key from the dropdown or create a new one. If generating a new key pair, save the private key immediately as it cannot be retrieved later.

    <Frame>
      <img src="/images/docs/edge-ai/ai-infrastructure/create-a-bare-metal-gpu-cluster/gpu-cluster-ssh-key.png" alt="SSH key section with dropdown and options to add or generate keys" />
    </Frame>
  </Step>
  <Step title="Set additional options">
    The **Additional options** section provides optional settings: user data scripts for automated configuration and metadata tags for resource organization.

    <Frame>
      <img src="/images/docs/edge-ai/ai-infrastructure/create-a-bare-metal-gpu-cluster/gpu-cluster-additional-options.png" alt="Additional options section with User data and Add tags checkboxes" />
    </Frame>
  </Step>
  <Step title="Name and create the cluster">
    In the **GPU Cluster Name** section, enter a name or use the auto-generated one. Review the estimated cost in the right panel, then click **Create Cluster**.

    <Frame>
      <img src="/images/docs/edge-ai/ai-infrastructure/create-a-bare-metal-gpu-cluster/gpu-cluster-name.png" alt="GPU Cluster Name section with name input field" />
    </Frame>

    Once all instances reach **Power on** status, the cluster is ready for use.

    <Warning>
    Cluster-level settings (image, file share integration, default networks) cannot be changed after creation. New nodes added via scaling inherit the original configuration. To change these settings, create a new cluster.
    </Warning>
  </Step>
</Steps>

## Connect and verify the cluster

<p>After the cluster is created, connect to a node via SSH and verify that GPUs are available.</p>

<Steps>
  <Step title="Connect via SSH">
    Open a terminal and connect using the default username `ubuntu`:

    ```bash
    ssh ubuntu@<instance-ip-address>
    ```

    Replace `<instance-ip-address>` with the public or floating IP shown in the cluster details. For nodes with only private interfaces, connect through a bastion host, VPN, or the Customer Portal [web&nbsp;console](/cloud/virtual-instances/connect/connect-to-your-instance-via-control-panel).
  </Step>
  <Step title="Verify GPU detection">

    ```bash
    nvidia-smi
    ```

    A successful output shows all available GPUs, driver version, and CUDA version. If no GPUs appear, check that the image includes the correct NVIDIA drivers for the GPU model.

    If file share integration was enabled, verify the mount is accessible:

    ```bash
    ls /home/ubuntu/mnt/nfs
    ```
  </Step>
</Steps>

## Quickstart

<p>Complete scripts that list flavors and images, create a one-node cluster, wait for provisioning, and print the SSH command.</p>

<Tabs>
  <Tab title="Python SDK">
    ```python
    import os
    import time
    from gcore import Gcore

    client = Gcore()
    ssh_key_name = os.environ["GCORE_SSH_KEY_NAME"]

    # Step 1. List flavors — select the one with the fewest GPUs
    flavors = client.cloud.gpu_baremetal.clusters.flavors.list()
    flavor = min(
        flavors.results,
        key=lambda f: f.hardware_properties.gpu_count,
    )
    print(f"Flavor: {flavor.name}  ({flavor.hardware_properties.gpu_count} GPU(s))")

    # Step 2. List images — select the first (latest) image
    images = client.cloud.gpu_baremetal.clusters.images.list()
    image = images.results[0]
    print(f"Image: {image.name}")

    # Step 3. Create the cluster
    task_id_list = client.cloud.gpu_baremetal.clusters.create(
        name="my-gpu-cluster",
        flavor=flavor.name,
        image_id=image.id,
        servers_count=1,
        servers_settings={
            "interfaces": [{"type": "external"}],
            "credentials": {"ssh_key_name": ssh_key_name},
        },
    )
    task_id = task_id_list.tasks[0]
    print(f"Provisioning — task: {task_id}")

    # Step 4. Poll task until FINISHED
    while True:
        task = client.cloud.tasks.get(task_id)
        if task.state == "FINISHED":
            cluster_id = task.created_resources.clusters[0]
            print(f"Cluster ID: {cluster_id}")
            break
        if task.state == "ERROR":
            raise RuntimeError(f"Cluster creation failed: {task.error}")
        print(f"  state={task.state} — retrying in 15s…")
        time.sleep(15)

    # Step 5. Get the public IP and print SSH command
    ifaces = client.cloud.gpu_baremetal.clusters.interfaces.list(
        cluster_id=cluster_id,
    )
    for iface in ifaces.results:
        if iface.network_details.external:
            ip = iface.ip_assignments[0].ip_address
            print(f"SSH: ssh ubuntu@{ip}")
    ```
  </Tab>
  <Tab title="Go SDK">
    ```go
    package main

    import (
        "context"
        "fmt"
        "os"
        "sort"
        "strconv"
        "time"

        "github.com/G-Core/gcore-go"
        "github.com/G-Core/gcore-go/cloud"
    )

    func main() {
        projectIDInt, _ := strconv.ParseInt(os.Getenv("GCORE_CLOUD_PROJECT_ID"), 10, 64)
        regionIDInt, _ := strconv.ParseInt(os.Getenv("GCORE_CLOUD_REGION_ID"), 10, 64)
        sshKeyName := os.Getenv("GCORE_SSH_KEY_NAME")

        client := gcore.NewClient()
        ctx := context.Background()

        // Step 1. List flavors — select the one with fewest GPUs
        flavorList, err := client.Cloud.GPUBaremetal.Clusters.Flavors.List(ctx,
            cloud.GPUBaremetalClusterFlavorListParams{
                ProjectID: gcore.Int(projectIDInt),
                RegionID:  gcore.Int(regionIDInt),
            })
        if err != nil {
            panic(err)
        }
        sort.Slice(flavorList.Results, func(i, j int) bool {
            return flavorList.Results[i].HardwareProperties.GPUCount <
                flavorList.Results[j].HardwareProperties.GPUCount
        })
        flavor := flavorList.Results[0]
        fmt.Printf("Flavor: %s  (%d GPU(s))\n", flavor.Name, flavor.HardwareProperties.GPUCount)

        // Step 2. List images — select the first (latest) image
        imageList, err := client.Cloud.GPUBaremetal.Clusters.Images.List(ctx,
            cloud.GPUBaremetalClusterImageListParams{
                ProjectID: gcore.Int(projectIDInt),
                RegionID:  gcore.Int(regionIDInt),
            })
        if err != nil {
            panic(err)
        }
        image := imageList.Results[0]
        fmt.Printf("Image: %s\n", image.Name)

        // Step 3. Create the cluster
        taskIDList, err := client.Cloud.GPUBaremetal.Clusters.New(ctx,
            cloud.GPUBaremetalClusterNewParams{
                ProjectID:    gcore.Int(projectIDInt),
                RegionID:     gcore.Int(regionIDInt),
                Name:         "my-gpu-cluster",
                Flavor:       flavor.Name,
                ImageID:      image.ID,
                ServersCount: 1,
                ServersSettings: cloud.GPUBaremetalClusterNewParamsServersSettings{
                    Interfaces: []cloud.GPUBaremetalClusterNewParamsServersSettingsInterfaceUnion{{
                        OfExternal: &cloud.GPUBaremetalClusterNewParamsServersSettingsInterfaceExternal{},
                    }},
                    Credentials: cloud.GPUBaremetalClusterNewParamsServersSettingsCredentials{
                        SSHKeyName: gcore.String(sshKeyName),
                    },
                },
            })
        if err != nil {
            panic(err)
        }
        taskID := taskIDList.Tasks[0]
        fmt.Printf("Provisioning — task: %s\n", taskID)

        // Step 4. Poll task until FINISHED
        var clusterID string
        for {
            task, err := client.Cloud.Tasks.Get(ctx, taskID)
            if err != nil {
                panic(err)
            }
            if task.State == "FINISHED" {
                clusterID = task.CreatedResources.Clusters[0]
                fmt.Printf("Cluster ID: %s\n", clusterID)
                break
            }
            if task.State == "ERROR" {
                panic(fmt.Sprintf("Cluster creation failed: %v", task.Error))
            }
            fmt.Printf("  state=%s — retrying in 15s…\n", task.State)
            time.Sleep(15 * time.Second)
        }

        // Step 5. Get the public IP and print SSH command
        ifaces, err := client.Cloud.GPUBaremetal.Clusters.Interfaces.List(ctx,
            clusterID,
            cloud.GPUBaremetalClusterInterfaceListParams{
                ProjectID: gcore.Int(projectIDInt),
                RegionID:  gcore.Int(regionIDInt),
            })
        if err != nil {
            panic(err)
        }
        for _, iface := range ifaces.Results {
            if iface.NetworkDetails.External {
                ip := iface.IPAssignments[0].IPAddress
                fmt.Printf("SSH: ssh ubuntu@%s\n", ip)
            }
        }
    }
    ```

</Tabs>

### Step 1. List available flavors

<p>Returns all Bare Metal GPU flavors available in the region. Use the response to pick a flavor name — required when creating the cluster.</p>

| Field | Description |
|-------|-------------|
| `name` | Flavor identifier — pass this value as `flavor` when creating the cluster |
| `hardware_description.gpu` | GPU model name and VRAM |
| `hardware_properties.gpu_count` | Number of GPUs per node |
| `hardware_properties.nic_ib` | InfiniBand configuration, if present |
| `capacity` | Available node count in the region (`0` means currently out of stock) |

### Step 3. Create the cluster

<p>Submits the cluster creation request and returns a task ID for tracking provisioning progress.</p>

| Parameter | Required | Description |
|-----------|----------|-------------|
| `name` | Yes | Cluster name (two–63 chars, alphanumeric, hyphens, underscores) |
| `flavor` | Yes | Flavor name from Step 1 |
| `image_id` | Yes | Image UUID from Step 2 |
| `servers_count` | Yes | Number of nodes to provision |
| `servers_settings.interfaces` | Yes | At least one interface — use `"type": "external"` for public internet access |
| `servers_settings.credentials.ssh_key_name` | Yes | Name of an existing SSH key in the project |
| `tags` | No | Key-value metadata tags |

### Step 4. Wait for provisioning

<p>Poll the task endpoint every 15 seconds until `state` is `FINISHED`. Bare Metal GPU nodes typically provision in 15–20 minutes.</p>

```bash
curl "https://api.gcore.com/cloud/v1/tasks/e4cecbfc-5258-40bb-b78f-3cd92621c3b2" \
  -H "Authorization: APIKey ${GCORE_API_KEY}"
```

<p>While provisioning:</p>

```json
{
  "id": "e4cecbfc-5258-40bb-b78f-3cd92621c3b2",
  "state": "RUNNING",
  "created_resources": null
}
```

<p>When complete:</p>

```json
{
  "id": "e4cecbfc-5258-40bb-b78f-3cd92621c3b2",
  "state": "FINISHED",
  "created_resources": {
    "clusters": ["4146c059-501a-409f-8fe1-04de091d917a"]
  }
}
```

<p>Read the cluster ID from `created_resources.clusters[0]`.</p>

    ```go
            fmt.Println("Public IP:", iface.IPAssignments[0].IPAddress)
        }
    }
    ```

<p>After the cluster reaches `active` status, connect to a node via SSH using the `ubuntu` user:</p>

```bash
ssh ubuntu@{NODE_PUBLIC_IP}
```

<p>Verify that GPUs are detected:</p>

```bash
nvidia-smi
```

<p>A successful output shows the list of GPUs, driver version, and CUDA version. For nodes with only private interfaces, connect through a bastion host or VPN.</p>

## Create a Bare Metal GPU cluster

<p>Provisions one bare metal node with a public interface. Use the flavors and images endpoints from the REST&nbsp;API tab to identify the right `flavor` and `image_id` for the target region before applying.</p>

```hcl
resource "gcore_cloud_gpu_baremetal_cluster" "example" {
  project_id    = var.project_id
  region_id     = var.region_id
  name          = "my-bm-gpu-cluster"
  flavor        = "bm3-ai-1xlarge-a100-80-8"  # 8× A100 80 GB; list available flavors via the flavors API
  image_id      = "d9400449-0ee8-4c60-adb9-fc545a80325c"  # Ubuntu 22.04, driver 580, CUDA 13.0 — Luxembourg-2; list current images via the images API
  servers_count = 1

  # servers_settings is an attribute (nested type), not a block — use = assignment
  servers_settings = {
    interfaces = [
      { type = "external" }
    ]
    credentials = {
      ssh_key_name = "my-ssh-key"  # name of an existing SSH key in the project
    }
  }
}

output "cluster_id" {
  value = gcore_cloud_gpu_baremetal_cluster.example.id
}

output "cluster_status" {
  value = gcore_cloud_gpu_baremetal_cluster.example.status
}

# terraform import gcore_cloud_gpu_baremetal_cluster.example '<project_id>/<region_id>/<cluster_id>'
```

<p>Bare Metal nodes take 15–20 minutes to provision. After `terraform apply` completes, the cluster status is `active`. Connect via SSH to the public IP from the cluster's network interfaces.</p>

<Info>
`ssh_key_name` must be a literal string in the HCL file — a variable reference causes the provider to reject the configuration at plan time. Replace `"my-ssh-key"` with the name of an existing SSH key in the project.
</Info>
