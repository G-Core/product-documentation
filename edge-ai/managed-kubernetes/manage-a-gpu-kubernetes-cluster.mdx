GPU Kubernetes cluster management covers day-to-day operations: connecting with kubectl, managing node pools, configuring observability and authentication, upgrading the Kubernetes version, and deleting the cluster when it is no longer needed.

| Topic | Description |
|-------|-------------|
| [Connect](/edge-ai/managed-kubernetes/connect-to-a-gpu-kubernetes-cluster) | Download kubeconfig and configure kubectl |
| [Node&nbsp;pools](/edge-ai/managed-kubernetes/manage-node-pools) | View, scale, add GPU pools, replace nodes, and delete pools |
| [Autoscaling](/edge-ai/managed-kubernetes/configure-cluster-autoscaling) | Configure the Cluster Autoscaler |
| [Logging](/edge-ai/managed-kubernetes/configure-cluster-logging) | Enable log collection to OpenSearch Dashboards |
| [OIDC&nbsp;authentication](/edge-ai/managed-kubernetes/configure-oidc-authentication) | Set up SSO via an external identity provider |
| [Upgrade](/edge-ai/managed-kubernetes/upgrade-a-gpu-kubernetes-cluster) | Upgrade to the next Kubernetes minor version |
| [Audit&nbsp;log](/edge-ai/managed-kubernetes/view-cluster-audit-log) | View cluster operation history |
| [Delete](/edge-ai/managed-kubernetes/delete-a-gpu-kubernetes-cluster) | Remove the cluster and its resources |
