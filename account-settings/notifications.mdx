Notification settings control which email alerts each user in a Gcore account receives for product events, billing changes, and promotional offers. Important notifications are configured individually per user, while general notifications are shared across all users of the account.

## Manage notifications

To view and configure notification settings:

1. In the [Gcore Customer Portal](https://portal.gcore.com/accounts/reports/dashboard), click the account icon in the top-right corner, then click **Profile**.

<Frame>![Profile section in the Customer Portal](/images/docs/account-settings/my-profile/overview/profile-menu.png)</Frame>

2. Open the **Notifications** page.

<Frame>![Notifications page in account settings](/images/docs/account-settings/my-profile/overview/notifications-page.png)</Frame>

3. Use the **Allow notifications** toggle to enable or disable all email notifications at once, or configure individual notifications and groups below it.

Notifications are organized into two account-wide groups, plus a card for each active product:

* **Important notifications**, configured individually per user: service emails about API changes and other information that might affect service availability, and promotional emails with tips and special offers.
* **General notifications**, shared across all account users: alerts including technical support requests, payment results, API token expiration, and product trial status.
* Product-specific cards appear for each active product and list notifications for product-specific events, including SSL certificate changes for CDN and DDoS attacks for Cloud.

<Tip>
Keep the **Service emails** toggle enabled to stay updated on changes that could affect the performance of the account's products.
</Tip>

## Limitation notifications

Limitation notifications are available for CDN and Object Storage, and warn account users when usage approaches a configured limit.

Usage counters reset on the first day of each month, and the traffic volume is checked hourly, at the top of the hour with a variance of up to three minutes.

Setting a traffic consumption level below the amount already used in the current month sends a notification to all account users at the next check. Enabling traffic control at 12:21 with a 1 TB level, after 2 TB of traffic has already been used that month, triggers the notification for all account users at 13:00, the next hourly check.

### CDN traffic consumption

The **CDN notifications** card also includes toggles for SSL certificate errors, SSL certificate expiration, and CDN resource suspension due to 90 days of inactivity. To configure the shared traffic consumption limit:

1. In the **Traffic consumption level** field, specify the traffic limit in GB.

<Frame>![CDN notifications card with SSL certificate and traffic consumption toggles](/images/docs/account-settings/my-profile/overview/notifications-cdn.png)</Frame>

2. Click **Save** to apply the settings.

<Info>
The traffic consumption level is not configured individually. It is the same for all account users who use this notification.
</Info>

Suppose three users share an account: User A, User B, and User C. User A is the first to enable this notification and sets the limit to 100 GB, so User B and User C see the same 100 GB limit already applied. User B leaves the limit unchanged, while User C changes it to 70 GB. After the change, all three users receive a notification once traffic reaches 70 GB.

### Object Storage limits

The **Object Storage notifications** card sends alerts about the number of requests to a bucket and the number of objects it contains. To configure a limit:

1. On the Notifications page, open the **Object Storage notifications** section.

2. Click **Add limit** next to the number of requests or number of objects setting.

3. Enter the required limit.

4. Click **Save** to apply the changes.

<Frame>![Object Storage notification limits for requests and objects in a bucket](/images/docs/account-settings/my-profile/overview/notifications-object-storage.png)</Frame>
