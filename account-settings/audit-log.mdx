The Audit log in Settings records actions performed on the Gcore account. This log provides visibility into user activity, API calls, and system events for security monitoring and compliance purposes.

To access the Audit log, navigate to **Settings** > **Audit log**.

<img src="/images/docs/account-settings/audit-log.png" alt="Audit log page" width="80%"/>

## Log entries

Each audit log entry contains the following information:

| Column | Description |
|--------|-------------|
| Date time | Timestamp of the event in local timezone |
| Actor ID | User ID that performed the action, with context type (common, API) |
| Actor context | IP address and browser/client information |
| Source | Service or system component (User management, CDN, Notifications, Cloud) |
| Event name | Specific action performed (login, API call, configuration change) |
| Trace ID | Unique identifier for request tracing and troubleshooting |

## Event sources

The audit log captures events from multiple sources:

- **User management**: Login attempts, user invitations, role changes
- **CDN**: Resource configuration, statistics queries, client operations
- **Cloud**: Instance operations, network changes, storage actions
- **Notifications**: Subscription updates, preference changes
- **API**: Token usage, programmatic operations

## Filter log entries

The audit log supports filtering to locate specific events:

### Available filters

| Filter | Description |
|--------|-------------|
| Source | Filter by service (User management, CDN, Cloud, Notifications) |
| Event | Search by event name or action type |
| Actor ID | Filter by specific user ID |
| Trace ID | Search by trace identifier |
| Date range | Select predefined range or custom dates |

### Apply filters

1. Select a filter from the dropdown or enter search text in the filter fields.
2. The table updates automatically to show matching entries.
3. Click **Reset all** to clear all active filters.

### Date range selection

The date picker supports:

- **Predefined ranges**: Last 24 hours, Last 7 days, Last 30 days
- **Custom range**: Select specific start and end dates using the calendar

## Common event types

### Authentication events

| Event | Description |
|-------|-------------|
| iam.auth.login | User login (successful or failed) |
| iam.auth.logout | User logout |
| iam.auth.2fa | Two-factor authentication verification |

### CDN events

| Event | Description |
|-------|-------------|
| /cdn/clients/me | Client profile access |
| GET /cdn/statistics/series | Statistics data retrieval |
| /cdn/resources | Resource configuration changes |

### Notification events

| Event | Description |
|-------|-------------|
| /user_subscription | Notification preference updates |

## Use cases

The audit log serves several operational purposes:

- **Security monitoring**: Detect unauthorized access attempts by reviewing login events and unusual IP addresses
- **Compliance**: Maintain records of configuration changes for audit requirements
- **Troubleshooting**: Use trace IDs to correlate issues with specific actions
- **User activity review**: Track actions performed by team members

## Data retention

Audit log entries are retained according to the account's data retention policy. For extended retention requirements, export log data periodically or contact support for enterprise retention options.

<Info>
Audit log access is available to account Administrators. Users with limited roles may not see the full event history.
</Info>
