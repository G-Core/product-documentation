Gcore MCP Server organizes tools by product category. This reference describes each product and its key tools.

## Tool naming convention

Tools follow a hierarchical naming pattern:

```
<product>.<resource>.<operation>
```

Examples:
- `cdn.cdn_resources.list` - List CDN resources
- `dns.zones.create` - Create a DNS zone
- `cloud.instances.create` - Create a virtual machine

## CDN

Content Delivery Network for accelerating static and dynamic content delivery.

**Filter pattern:** `cdn.*`

### Key resources

| Resource | Tools | Description |
|----------|-------|-------------|
| `cdn_resources` | list, create, get, update, delete | CDN resource configuration |
| `origin_groups` | list, create, get, update, delete | Origin server groups |
| `certificates` | list, create, get, delete | SSL/TLS certificates |
| `rule_templates` | list, get | Caching and behavior rules |
| `shields` | list, get | Origin shielding locations |

### Common operations

**List CDN resources:**

```
cdn.cdn_resources.list
```

No parameters required. Returns all CDN resources in the account.

**Create a CDN resource:**

```
cdn.cdn_resources.create
```

Parameters:
- `cname` (required): Domain name for the CDN resource
- `origin_group` (required): Origin group ID
- `origin_protocol`: HTTP or HTTPS

**Purge cache:**

```
cdn.cdn_resources.purge
```

Parameters:
- `resource_id`: CDN resource ID
- `paths`: List of paths to purge, or `/*` for all

## DNS

Managed DNS service with global Anycast network.

**Filter pattern:** `dns.*`

### Key resources

| Resource | Tools | Description |
|----------|-------|-------------|
| `zones` | list, create, get, update, delete | DNS zones |
| `zones.records` | list, create, get, update, delete | DNS records within zones |
| `locations` | list | DNS server locations |

### Common operations

**List DNS zones:**

```
dns.zones.list
```

**Create a DNS zone:**

```
dns.zones.create
```

Parameters:
- `name` (required): Zone name (for example, `example.com`)

**Add a DNS record:**

```
dns.zones.records.create
```

Parameters:
- `zone_name` (required): Parent zone
- `name` (required): Record name
- `type` (required): Record type (A, AAAA, CNAME, MX, TXT, etc.)
- `content` (required): Record value
- `ttl`: Time to live in seconds

## Cloud

Cloud infrastructure services including compute, storage, and networking.

**Filter pattern:** `cloud.*`

### Compute resources

| Resource | Tools | Description |
|----------|-------|-------------|
| `instances` | list, create, get, delete, start, stop, resize | Virtual machines |
| `baremetal` | list, create, get, delete, power_on, power_off | Bare metal servers |
| `gpu_baremetal` | clusters.list, clusters.create, clusters.delete | GPU bare metal clusters |
| `gpu_virtual` | clusters.list, clusters.create, clusters.delete | GPU virtual clusters |

### Storage resources

| Resource | Tools | Description |
|----------|-------|-------------|
| `volumes` | list, create, get, delete, attach, detach | Block storage volumes |
| `volume_snapshots` | list, create, get, delete | Volume snapshots |
| `file_shares` | list, create, get, delete | Shared file systems |

### Networking resources

| Resource | Tools | Description |
|----------|-------|-------------|
| `networks` | list, create, get, delete | Virtual networks |
| `floating_ips` | list, create, get, delete, assign, unassign | Public IP addresses |
| `load_balancers` | list, create, get, delete, update | Load balancers |
| `security_groups` | list, create, get, delete, add_rule, remove_rule | Firewall rules |

### Security resources

| Resource | Tools | Description |
|----------|-------|-------------|
| `ssh_keys` | list, create, get, delete | SSH public keys |
| `secrets` | list, create, get, delete | Secret storage |

### Management resources

| Resource | Tools | Description |
|----------|-------|-------------|
| `projects` | list, get | Cloud projects |
| `regions` | list, get | Available regions |
| `quotas` | get | Resource quotas |

### Common operations

**List projects:**

```
cloud.projects.list
```

**List instances in a project:**

```
cloud.instances.list
```

Parameters:
- `project_id` (required): Project ID
- `region_id` (required): Region ID

**Create an instance:**

```
cloud.instances.create
```

Parameters:
- `project_id` (required): Project ID
- `region_id` (required): Region ID
- `name` (required): Instance name
- `flavor` (required): Instance type
- `image_id` or `volume_id` (required): Boot source
- `networks`: Network configuration

## Storage

### Key resources

| Resource | Tools | Description |
|----------|-------|-------------|
| `buckets` | list, create, get, delete | Storage buckets |
| `bucket.objects` | list, upload, download, delete | Objects within buckets |

**List buckets:**

```
storage.buckets.list
```

**Create a bucket:**

```
storage.buckets.create
```

- `name` (required): Bucket name

## Discover tool names

To find exact tool names for a product:

1. Use MCP Inspector with the product pattern:

```bash
GCORE_TOOLS="cdn.*" npx @modelcontextprotocol/inspector
```

2. Connect and browse the Tools panel.

3. Note the full tool names for use in configuration.
