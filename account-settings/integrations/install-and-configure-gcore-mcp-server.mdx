Gcore MCP Server requires a [Gcore API token](/account-settings/api-tokens) and the `uv` package manager. The `uv` tool handles Python dependencies and provides isolated execution environments—installation instructions are available in the [uv documentation](https://docs.astral.sh/uv/getting-started/installation/).

## Create an API token

The API token authenticates requests to the Gcore API. Token permissions determine which operations are available through the MCP server.

1. In the Gcore Customer Portal, navigate to **Profile** > **API tokens**.
2. Click **Create token**.
3. Select the permissions required for the intended operations.
4. Copy the generated token.

<Warning>
The token is displayed once and cannot be retrieved later. Store it securely and do not commit to version control.
</Warning>

## Installation methods

### Temporary environment (uvx)

The `uvx` command runs the server in a temporary environment without persistent installation:

```bash
uvx --from "gcore-mcp-server@git+https://github.com/G-Core/gcore-mcp-server.git" gcore-mcp-server
```

A specific version can be pinned by appending the tag:

```bash
uvx --from "gcore-mcp-server@git+https://github.com/G-Core/gcore-mcp-server.git@v0.1.1" gcore-mcp-server
```

### Persistent installation (uv tool)

Installing as a tool makes the command globally available:

```bash
uv tool install "gcore-mcp-server@git+https://github.com/G-Core/gcore-mcp-server.git"
```

After installation, the server starts with:

```bash
gcore-mcp-server
```

If the command is not found, run `uv tool update-shell` or verify that the `uv` tool bin directory is in `PATH`.

## Environment variables

| Variable | Required | Description |
|----------|----------|-------------|
| `GCORE_API_KEY` | Yes | API token for authentication |
| `GCORE_TOOLS` | No | Tool filter pattern. Default: `management,instances` (HTTP) or `management` (stdio) |
| `GCORE_BASE_URL` | No | API endpoint. Default: `https://api.gcore.com` |
| `GCORE_CLOUD_PROJECT_ID` | No | Default project ID for Cloud operations |
| `GCORE_CLOUD_REGION_ID` | No | Default region ID for Cloud operations |

### Set environment variables

**Linux/macOS:**

```bash
export GCORE_API_KEY="your-api-key"
export GCORE_TOOLS="cdn.*"
```

**Windows PowerShell:**

```powershell
$env:GCORE_API_KEY = "your-api-key"
$env:GCORE_TOOLS = "cdn.*"
```

**Windows Command Prompt:**

```cmd
set GCORE_API_KEY=your-api-key
set GCORE_TOOLS=cdn.*
```

## Verify installation

After setting environment variables, start the server:

```bash
gcore-mcp-server
```

The server outputs its version and the number of loaded tools when connected to an MCP client. To verify API key validity independently, the Gcore Python SDK can list Cloud projects:

```python
from gcore import Gcore
from dotenv import load_dotenv

load_dotenv()

client = Gcore()
projects = client.cloud.projects.list()
for project in projects:
    print(f"Project: {project.name} (ID: {project.id})")
```

A successful response confirms the API key has Cloud access.

## Transport modes

### stdio (standard input/output)

The default mode reads JSON-RPC messages from stdin and writes responses to stdout. Most MCP clients use this mode:

```bash
gcore-mcp-server
```

### HTTP (SSE)

HTTP mode with Server-Sent Events listens on `http://localhost:8000` by default. This mode is useful for debugging with [MCP Inspector](/account-settings/integrations/test-tools-with-mcp-inspector):

```bash
gcore-mcp-server --http
```

## Configuration files

MCP clients store server configuration in JSON files:

| Client | Configuration file |
|--------|-------------------|
| Claude Code | `~/.claude.json` |
| Cursor IDE | `~/.cursor/mcp.json` |

Claude Code configuration example (`~/.claude.json`):

```json
{
  "mcpServers": {
    "gcore-mcp-server": {
      "command": "uvx",
      "args": [
        "--from",
        "gcore-mcp-server@git+https://github.com/G-Core/gcore-mcp-server.git",
        "gcore-mcp-server"
      ],
      "env": {
        "GCORE_API_KEY": "your-api-key",
        "GCORE_TOOLS": "*"
      }
    }
  }
}
```

Setting `GCORE_TOOLS=*` is appropriate for Claude Code due to its deferred schema loading. Other clients require [filtering to fewer tools](/account-settings/integrations/filter-tools-with-gcore-tools) to avoid context window issues.
