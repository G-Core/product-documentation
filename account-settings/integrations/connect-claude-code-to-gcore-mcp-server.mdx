Claude Code is the recommended client for Gcore MCP Server. Its tool search feature handles large tool sets without context window issues.

## Why Claude Code

Claude Code differs from other MCP clients in how it handles tools:

- **Deferred schema loading**: Tool schemas are fetched on demand when they match a query
- **Tool search**: Claude searches for relevant tools instead of loading all schemas upfront
- **No context bloat**: Registering many tools does not affect the context window

These features allow Claude Code to work with all 780 Gcore tools simultaneously.

## Configure the server

1. Open the Claude Code configuration file at `~/.claude.json`.
2. Add the Gcore MCP Server entry to `mcpServers`:

```json
{
  "mcpServers": {
    "gcore-mcp-server": {
      "command": "uvx",
      "args": [
        "--from",
        "gcore-mcp-server@git+https://github.com/G-Core/gcore-mcp-server.git",
        "gcore-mcp-server"
      ],
      "env": {
        "GCORE_API_KEY": "your-api-key",
        "GCORE_TOOLS": "*"
      }
    }
  }
}
```

3. Replace `your-api-key` with a valid Gcore API token.
4. Save the file and restart Claude Code.

<Info>
Setting `GCORE_TOOLS=*` enables all available tools. This configuration is safe for Claude Code due to deferred schema loading.
</Info>

## Verify the connection

After configuring, start a new Claude Code session and ask:

```
List my CDN resources
```

Claude Code searches for CDN-related tools, loads their schemas, and executes the appropriate tool. The response shows CDN resource details from the Gcore account.

Example response:

```
Found 1 CDN resource:

- ID: 888473
- CName: example.com
- Status: active
- Origin Group: 1298426
```

## Tool search behavior

When a query mentions a specific product or operation, Claude Code:

1. Searches tool names and descriptions for matches
2. Loads schemas for matched tools
3. Selects the most appropriate tool
4. Executes the tool with extracted parameters

If multiple tools match, Claude may ask for clarification or choose based on context.

## Handling tool warnings

Claude Code displays a warning when tools exceed 10% of the context window. With Gcore MCP Server configured as above, this warning does not appear because schemas are loaded on demand.

If the warning appears:
- Verify `GCORE_TOOLS=*` is set
- Restart Claude Code to clear cached schemas
- Check that no other MCP servers are loading many tools

## Troubleshooting

### Server not connecting

Verify the configuration syntax:

```bash
cat ~/.claude.json | python -m json.tool
```

Check that `uvx` is installed and available:

```bash
uvx --version
```

### API authentication errors

Confirm the API key is valid:

```bash
curl -H "Authorization: Bearer your-api-key" https://api.gcore.com/cdn/resources
```

A 200 response indicates the key is valid. A 401 response indicates invalid credentials.

### Tools not found

If Claude cannot find Gcore tools:

1. Restart Claude Code
2. Check that `GCORE_TOOLS=*` is set
3. Verify the server starts without errors by running it manually:

```bash
GCORE_API_KEY=your-key uvx --from "gcore-mcp-server@git+https://github.com/G-Core/gcore-mcp-server.git" gcore-mcp-server
```
