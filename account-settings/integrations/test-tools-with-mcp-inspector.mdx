MCP Inspector provides a web interface for exploring and testing MCP server tools without an AI client. It displays available tools, their parameters, and allows manual tool execution.

## Prerequisites

Node.js 18 or later is required. Verify the installation:

```bash
node --version
npx --version
```

## Start MCP Inspector

Run the inspector using npx:

```bash
npx @modelcontextprotocol/inspector
```

The inspector starts at `http://localhost:6274`. A browser window opens automatically.

## Connect to Gcore MCP Server

MCP Inspector connects to servers via stdio or HTTP transport.

### Option 1: Configure in the Inspector UI

1. Set **Transport Type** to `STDIO`.
2. Enter the command: `uvx`
3. Enter arguments: `--from gcore-mcp-server@git+https://github.com/G-Core/gcore-mcp-server.git gcore-mcp-server`
4. Click **Environment Variables** and add:
   - `GCORE_API_KEY`: your API key
   - `GCORE_TOOLS`: tool filter (for example, `cdn.*`)
5. Click **Connect**.

### Option 2: Run server in HTTP mode

Start the server with HTTP transport in one terminal:

```bash
GCORE_API_KEY=your-key GCORE_TOOLS="cdn.*" gcore-mcp-server --http
```

In the Inspector:
1. Set **Transport Type** to `SSE`.
2. Enter URL: `http://localhost:8000/sse`
3. Click **Connect**.

## Explore tools

After connecting, the Inspector displays:

- **Tools**: List of available MCP tools
- **Resources**: MCP resources (if any)
- **Prompts**: MCP prompts (if any)

Click a tool to view:
- **Description**: What the tool does
- **Input Schema**: Required and optional parameters
- **Test Interface**: Form to execute the tool with custom parameters

## Execute tools manually

1. Select a tool from the list.
2. Fill in the required parameters.
3. Click **Execute**.
4. View the response in the Results panel.

### Example: List CDN resources

1. Connect with `GCORE_TOOLS="cdn.*"`.
2. Find `cdn.cdn_resources.list` in the tools list.
3. Click to open the tool details.
4. No parameters are required for listing.
5. Click **Execute**.
6. View the JSON response containing CDN resources.

### Example: Get DNS zone details

1. Connect with `GCORE_TOOLS="dns.*"`.
2. Find `dns.zones.get` in the tools list.
3. Enter the zone name parameter.
4. Click **Execute**.
5. View zone configuration in the response.

## Filter tools for exploration

The `GCORE_TOOLS` variable controls which tools appear in the Inspector.

**Explore CDN tools:**

```bash
GCORE_TOOLS="cdn.*"
```

**Explore Cloud instance tools:**

```bash
GCORE_TOOLS="cloud.instances.*"
```

**Explore multiple products:**

```bash
GCORE_TOOLS="cdn.*,dns.*"
```

<Warning>
Using `GCORE_TOOLS=*` loads all 780 tools. The Inspector interface may become slow or unresponsive. Filter to specific products for better performance.
</Warning>

## View tool schemas

Tool schemas define the expected input and output formats. In the Inspector:

1. Select a tool.
2. View the **Input Schema** section.
3. Expand nested objects to see all fields.

Schema information includes:
- **type**: Data type (string, number, object, array)
- **required**: Whether the field is mandatory
- **description**: Field purpose and constraints
- **default**: Default value if not provided

## Debug tool execution

The Inspector shows:

- **Request**: JSON-RPC message sent to the server
- **Response**: Server response with results or errors
- **Timing**: Execution duration

Common issues:

### Authentication error (401)

The API key is invalid or expired. Generate a new token in the Gcore Customer Portal.

### Permission denied (403)

The API key lacks permissions for the requested operation. Check token permissions.

### Resource not found (404)

The specified resource ID or name does not exist. Verify the parameter values.

### Validation error

Required parameters are missing or have invalid types. Check the tool schema for correct format.

## Use cases for MCP Inspector

### Discover tool names and parameters

Before documenting tools, explore them in the Inspector to understand:
- Exact tool names
- Required vs optional parameters
- Parameter types and formats
- Response structure

### Test tool filtering patterns

Verify that `GCORE_TOOLS` patterns match the intended tools:

```bash
GCORE_TOOLS="cloud.instances.*"
```

Connect and verify the tool list.

### Debug API issues

When Claude Code or other clients report errors, replicate the tool call in the Inspector to see:
- Exact request sent
- Full error response
- Server logs
