This guide describes how to create a FastEdge app. Check out our [FastEdge overview](/fastedge/getting-started) article to learn more about the product.

You can create a FastEdge app in two ways: by uploading a custom binary file (built using the [JavaScript SDK](https://github.com/G-Core/FastEdge-sdk-js) or [Rust](https://github.com/G-Core/FastEdge-sdk-rust)) or by using a preconfigured template. If you use a template, skip Stage 1.

## Stage 1. Create a Wasm binary file

To get started, create a .wasm file that you will later upload to the Gcore Customer Portal.

<Tabs>
  <Tab title="Via Rust">
    Scaffold a project with the Rust SDK and the `http-base` template:

    ```sh
    npm create fastedge-app@latest my-rust-app -- --rust --template http-base
    ```

    The template sets the compilation target to `wasm32-wasip2` in `.cargo/config.toml` and uses the `wstd` crate. More examples are in the [Gcore repository](https://github.com/G-Core/FastEdge-examples/tree/main/rust).

    Build the Wasm binary:

    ```sh
    cd my-rust-app
    cargo build --release
    ```

    The resulting Wasm file is written to `./target/wasm32-wasip2/release/basic_wasi_http.wasm`. Upload this file in Stage 2.

  </Tab>
  <Tab title="Via JavaScript SDK">
    A JavaScript code pattern closely resembles [Service Worker API](https://developer.mozilla.org/en-US/docs/Web/API/Service_Worker_API). You can also find multiple examples in the [Gcore repository](https://github.com/G-Core/FastEdge-examples/tree/main/javascript).

    The key aspect of the Wasm configuration is to set up the `addEventListener` that has to synchronously call `event.respondWith` with a callback. This callback can be asynchronous and this is where you'd usually include any custom code to generate a response.

    Here's the sample app source:

    ```js
    async function app(event) {
      const request = event.request;
      return new Response(`You made a request to ${request.url}`);
    }
    addEventListener('fetch', (event) => {
      event.respondWith(app(event));
    });
    ```

    Scaffold a project with the JavaScript SDK:

    ```sh
    npm create fastedge-app@latest my-first-app -- --javascript --template http-base
    ```

    Build the Wasm binary:

    ```sh
    cd my-first-app
    npm install
    npm run build
    ```

    The build command runs `npx fastedge-build ./src/index.js ./wasm/basic-http.wasm`. Upload `./wasm/basic-http.wasm` in Stage 2.

  </Tab>
</Tabs>

## Stage 2. Deploy an application

For detailed steps on how to deploy a FastEdge app, refer to the relevant sections below:

- [In the Customer Portal](/fastedge/getting-started/create-fastedge-apps#in-the-customer-portal). Follow the instructions if you created a custom Wasm using either the Rust or Javascript SDK, or if you want to create a FastEdge app from a preconfigured template.

- [Via command line](/fastedge/getting-started/create-fastedge-apps#via-command-line): Follow the instructions if you want to deploy a custom Wasm using cURL and our API.

### In the customer portal

<Tabs>
  <Tab title="Deploy an app from a binary">
    1\. In the [Gcore Customer Portal](https://portal.gcore.com/accounts/reports/dashboard), navigate to **FastEdge** > **HTTP Applications** > **Applications**.

    2\. In the top-right corner of the screen, click **Create new application**.

    <Frame>![HTTP Applications list with the Create new application button](/images/docs/fastedge/create-http-apps/http-apps-list-create-button.png)</Frame>

    3\. On the **Create an application** page, click the **Upload binary** card.

    <Frame>![Create an application page with the Upload binary card](/images/docs/fastedge/create-http-apps/http-apps-upload-binary.png)</Frame>

    4\. In the **Upload binary** dialog, click **Click to upload or drag and drop** and select your `.wasm` file. The portal uploads the file automatically and shows **Processing binary upload...**. When processing finishes, the **Deploy application** page opens.

    <Frame>![Upload binary dialog with file picker](/images/docs/fastedge/create-http-apps/upload-binary-dialog.png)</Frame>

    5\. On the **Deploy application** page, enter a name and an optional description.

    6\. (Optional) In the **Response headers** section, click **Add response header** to add fixed headers to responses. For example, include CORS (cross-origin resource sharing) headers in each response to allow cross-origin requests.

    7\. (Optional) In the **Environment variables** section, click **Add environment variable** and enter data as key-value pairs.

    <Frame>![Deploy application form with response headers and environment variables sections](/images/docs/fastedge/create-http-apps/deploy-form-config-sections.png)</Frame>

    <Info>
    **Info**

    **Secrets** and **Edge Storage** require existing resources. Create a secret in <a href="/fastedge/secrets-manager/manage-secrets" target="_blank" rel="noopener noreferrer">Secrets Manager</a> (set slot index `0`, click **Encrypt**, then **Save changes**) and an Edge Storage store on the **Edge Storage** page before you attach them here. In the **Secrets** section, click **Add secret**, enter an environment variable key, and select the secret from the dropdown.
    </Info>

    8\. Check all the settings. If everything is configured correctly, click **Save and deploy**.

    The portal opens the application details page with the deployment URL.

    <Frame>![Application details page with URL and Active status](/images/docs/fastedge/create-http-apps/http-app-details-url-active.png)</Frame>

    You can now test its configuration and adjust it as described in the following steps.

  </Tab>
  <Tab title="Deploy an app from a template">
    1\. In the [Gcore Customer Portal](https://portal.gcore.com/accounts/reports/dashboard), navigate to **FastEdge** > **HTTP Applications** > **Applications**.

    2\. In the top-right corner of the screen, click **Create new application**.

    <Frame>![HTTP Applications list with the Create new application button](/images/docs/fastedge/create-http-apps/http-apps-list-create-button.png)</Frame>

    3\. On the **Create an application** page, scroll to the **Create from a template** section and select a template. Built-in templates are **Geolocation-based redirect** and **Markdown renderer**. To deploy from a custom template, create it first on the **HTTP Applications** > **Templates** page as described in [Create and manage templates](/fastedge/create-manage-templates).

    <Frame>![Create from a template section with Geolocation-based redirect and Markdown renderer](/images/docs/fastedge/create-http-apps/http-apps-create-from-template.png)</Frame>

    4\. Enter a name for the application and, optionally, update its description.

    5\. In the **Template environment variables** section, fill in the fields required by the selected template:

    **Markdown renderer**

    - **BASE** (required): Base URL of the Markdown source. Example: `https://raw.githubusercontent.com/G-Core/FastEdge-sdk-js/main/`
    - **HEAD** (optional): HTML inserted into the `<head>` tag. Example: `<style>body{font-family:sans-serif;margin:2rem}</style>`

    After deployment, open `https://<app-name>-<id>.fastedge.app/README.md` to fetch `README.md` from the repository root as HTML.

    **Geolocation-based redirect**

    - **DEFAULT** (required): Fallback URL when the visitor country does not match any rule. Example: `https://example.com/fallback`

    In the **Environment variables** section (not **Template environment variables**), click **Add environment variable** and add ISO 3166-1 alpha-2 country codes as keys with redirect URLs as values. Example: key `LU`, value `https://example.com/luxembourg`. The app returns HTTP 302 with a `location` header.

    6\. (Optional) In the **Response headers**, **Environment variables**, **Secrets**, or **Edge Storage** sections, add any extra configuration the application needs.

    <Frame>![Deploy application form with template environment variables](/images/docs/fastedge/create-http-apps/template-deploy-env-vars-form.png)</Frame>

    7\. Click **Save and deploy**.

    The portal opens the application details page with the deployment URL.

    <Frame>![Application details page after template deployment](/images/docs/fastedge/create-http-apps/http-app-details-url-active.png)</Frame>

    You can now test its configuration and adjust it as described in the following steps.

### Via command line

1\. Upload the Wasm binary to our edge servers by running the following [API request](https://gcore.com/docs/api-reference/fastedge/binaries/store-compiled-wasm-binary) from the repo's root directory. Insert your [permanent API token](/account-settings/api-tokens) instead of the api_key:

```sh
curl -X POST 'https://api.gcore.com/fastedge/v1/binaries/raw' \
  -H 'accept: application/json' \
  -H 'Authorization: APIKey api_key' \
  -H 'Content-Type: application/octet-stream' \
  --data-binary '@./wasm/basic-http.wasm'
```

The response includes the uploaded binary ID (`<binary_id>`). Save this value for the next step.

2\. Create the app by running the following [API request](https://gcore.com/docs/api-reference/fastedge/apps/add-a-new-app):

```sh
curl -X POST 'https://api.gcore.com/fastedge/v1/apps' \
  -H 'accept: application/json' \
  -H 'Authorization: APIKey api_key' \
  -H 'Content-Type: application/json' \
  -d '{
    "name": "my-edge-app",
    "binary": <binary_id>,
    "status": 1
  }'
```

Where:

- `my-edge-app` is the unique application name (lowercase letters, numbers, and hyphens).
- `api_key` is the permanent API token.
- `<binary_id>` is the numeric ID from the upload response.

## Stage 3. Test an application

After deployment, the portal opens the application details page. Click the URL shown at the top of the page (for example, `https://<app-name>-<id>.fastedge.app`) to open the application in a browser.

<Frame>
  ![Application details page with deployment URL and Active
  status](/images/docs/fastedge/create-http-apps/http-app-details-url-active.png)
</Frame>

To return to an application later, use the applications list:

1\. In the Gcore Customer Portal, navigate to **FastEdge** > **HTTP Applications** > **Applications**.

<Frame>
  ![HTTP Applications list with Name, URL, and Status
  columns](/images/docs/fastedge/create-http-apps/http-apps-application-list-edge-storage.png)
</Frame>

2\. Find the application and click its name in the **Name** column to open the details page.

3\. Click the URL in the **URL** column on the list page, or the **URL** link on the details page, to test the application.

**Verified examples** (portal E2E, 2026-06-10):

- **Markdown renderer**: `curl https://<app-name>-<id>.fastedge.app/README.md` returns `HTTP/1.1 200 OK` with HTML converted from the Markdown file at BASE + path.
- **Geolocation-based redirect**: `curl -i https://<app-name>-<id>.fastedge.app/` returns `HTTP/1.1 302 Found` with a `location` header pointing to the matching country URL or **DEFAULT**.
- **Custom binary** (JavaScript SDK `http-base` template): the portal upload, deploy, and **Actions** > **Update binary** flows complete with **Active** status. If `curl https://<app-name>-<id>.fastedge.app/` returns `HTTP/1.1 530` with `fastedge: Execute error`, rebuild the Wasm with `npm run build` from a `create-fastedge-app --template http-base` project and upload again. See [Troubleshooting](/fastedge/troubleshooting).

To attach **Secrets** or **Edge Storage** at creation time, click **Add secret** or **Add Edge Storage** on the deploy form, enter an environment variable key, and select an existing resource from the dropdown.

## Stage 4 (Optional). Add more functionality

The following example prints incoming request headers and adds a custom response header from an environment variable.

<Tabs>
  <Tab title="Via Rust">
    1\. In the Rust project, update `src/lib.rs` to read the `CUSTOM_HEADER` environment variable and echo request headers in the response body.

    2\. Rebuild the Wasm binary:

    ```sh
    cargo build --release
    ```

    3\. In the Gcore Customer Portal, open the application details page and click **Actions** > **Update binary**.

    4\. Upload the new `.wasm` file from `./target/wasm32-wasip2/release/basic_wasi_http.wasm`, then click **Save and deploy**.

    5\. On the **Environment variables** tab, click **Add environment variable**, set key `CUSTOM_HEADER` and value `foo`, then click **Save changes**.

    6\. Test the deployment URL with `curl -i https://<app-name>-<id>.fastedge.app/`. The response includes the `custom-header` field and GeoIP headers injected by the edge network.

  </Tab>
  <Tab title="Via JavaScript SDK">
    1\. Replace the contents of `src/index.js` with the following code:

    ```js
    import { getEnv } from 'fastedge::env';

    async function eventHandler(event) {
      const request = event.request;
      const customEnvVariable = getEnv('MY_CUSTOM_ENV_VAR');
      const headersStr = JSON.stringify(Object.fromEntries(request.headers.entries()), null, 2);

      return new Response(`Headers: ${headersStr}\n`, {
        headers: {
          'Custom-Header': customEnvVariable,
        },
      });
    }

    addEventListener('fetch', (event) => {
      event.respondWith(eventHandler(event));
    });
    ```

    2\. Rebuild the Wasm binary:

    ```sh
    npm run build
    ```

    3\. In the Gcore Customer Portal, open the application details page and click **Actions** > **Update binary**.

    4\. Upload `./wasm/basic-http.wasm`, then click **Save and deploy**.

    5\. On the **Environment variables** tab, click **Add environment variable**, set key `MY_CUSTOM_ENV_VAR` and value `Custom-Header-Value`, then click **Save changes**.

    6\. Test the deployment URL:

    ```sh
    curl -i https://<app-name>-<id>.fastedge.app/
    ```

    The response body lists request headers as JSON text, and the `custom-header` response header reflects the environment variable value.

  </Tab>
</Tabs>
