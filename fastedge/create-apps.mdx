FastEdge applications compile to WebAssembly and run on Gcore edge nodes. The SDK depends on the application type: HTTP applications use the Rust or JavaScript SDK; CDN applications use the Proxy-Wasm spec.

## HTTP applications

HTTP applications handle requests independently — the edge node acts as the origin and the application returns its own HTTP response. Two SDKs are available.

### Rust SDK

The [FastEdge Rust SDK](https://github.com/G-Core/FastEdge-sdk-rust) provides the `#[fastedge::http]` macro that marks the application entry point, along with modules for outbound HTTP, key-value storage, secrets, and dictionary access.

Add the dependency and configure the crate type in `Cargo.toml`:

```toml
[dependencies]
fastedge = "0.4"
anyhow = "1.0"

[lib]
crate-type = ["cdylib"]
```

Add the WebAssembly target:

```bash
rustup target add wasm32-wasip1
```

A complete example with build and deployment steps is in [Build a Rust WASI app](/fastedge/getting-started/build-a-rust-wasi-app).

### JavaScript SDK

The [FastEdge JavaScript SDK](https://github.com/G-Core/FastEdge-sdk-js) (`@gcoredev/fastedge-sdk-js`) compiles JavaScript to WebAssembly using a bundled CLI. Applications use a fetch-like interface:

```js
addEventListener('fetch', (event) => {
  event.respondWith(new Response('Hello from FastEdge'));
});
```

Requirements: Node.js v20 or higher.

```bash
npm install --save-dev @gcoredev/fastedge-sdk-js
```

The SDK includes three CLI tools: `fastedge-init` scaffolds a new project, `fastedge-build` compiles it to WebAssembly, and `fastedge-assets` packages static files for edge serving.

### Deploy from a binary or template

Both SDKs produce a `.wasm` binary that is uploaded to FastEdge via the Customer Portal or the API. Pre-built templates are available for common use cases and do not require compiling from source. Deployment steps for both paths are covered in [Create a FastEdge app](/fastedge/getting-started/create-fastedge-apps).

## CDN applications

CDN applications run alongside a CDN resource. They intercept and modify HTTP requests before they reach the origin, and responses before they reach the client — without acting as an origin themselves.

CDN applications use the [Proxy-Wasm](https://github.com/proxy-wasm/spec) spec. The Rust SDK supports Proxy-Wasm through the `fastedge::proxywasm` module, which is enabled by default.

Integration steps and CDN resource configuration are covered in [Integrate CDN with FastEdge](/fastedge/getting-started/integrate-cdn-with-fastedge).
