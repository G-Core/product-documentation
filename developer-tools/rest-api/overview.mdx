The Gcore API provides programmatic access to every product available in the Customer Portal. Creating VMs, configuring CDN rules, managing DNS records, provisioning Kubernetes clusters — all of it can be done through API calls instead of browser clicks. This makes infrastructure manageable from scripts, CI/CD pipelines, and custom applications.

The API is a good fit when the goal is to automate repetitive tasks, integrate Gcore into existing tooling, or build workflows that the portal doesn't support directly.

## Base URLs

Each Gcore product has a dedicated base URL. The same permanent API token authenticates to all of them — no separate credentials per product:

| Product | Base URL |
|---------|----------|
| Cloud (VMs, Kubernetes, LB, ...) | `https://api.gcore.com/cloud` |
| IAM (users, tokens) | `https://api.gcore.com/iam` |
| CDN | `https://api.gcore.com/cdn` |
| Managed DNS | `https://api.gcore.com/dns` |
| Object Storage | `https://api.gcore.com/storage` |
| FastEdge | `https://api.gcore.com/fastedge` |
| Streaming | `https://api.gcore.com/streaming` |
| WAAP | `https://api.gcore.com/waap` |
| DDoS Protection | `https://api.gcore.com/security/iaas` |

Complete endpoint documentation — every parameter, response schema, and code sample — is in the [API reference](/api-reference/quick-access).
