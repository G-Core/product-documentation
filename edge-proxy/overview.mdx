## Edge Proxy

Edge Proxy is a DDoS protection solution that runs on the Gcore CDN edge infrastructure. Applications are protected by routing traffic through a globally distributed proxy layer.

Protection is active in minutes without deploying additional infrastructure or configuring complex networking.

The [setup&nbsp;guide](/edge-proxy/setup-and-configuration) covers creating a proxy and obtaining a protected IP. The [management&nbsp;guide](/edge-proxy/manage-edge-proxy) covers editing, deleting, and monitoring traffic.

## Traffic flow

Edge Proxy uses the CDN edge network to receive and filter incoming traffic before forwarding it to the origin server.

Key principles:

- Traffic is routed through Anycast IP addresses to the nearest Point of Presence (PoP)
- Filtering and proxying happen at the edge
- Clean traffic is forwarded to the origin server
- A protected IP address and port are assigned for user connections

## Key benefits

### High availability and scalability

Edge Proxy runs on the CDN infrastructure, which distributes traffic across multiple PoPs. This removes regional bottlenecks and enables multi-terabit filtering capacity.

### Lower latency

Traffic is processed closer to end users. This reduces round-trip time and improves application responsiveness.

### Simple setup

Protection is configured in a few steps without managing virtual machines or networking components.

### Efficient IP usage

Edge Proxy uses shared Anycast IP pools, reducing the number of dedicated IP addresses required.

### Flexible configuration

Multiple translation rules can be created for a single protected IP address.

## Use cases

Edge Proxy is suitable for:

- Game servers
- APIs and backend services
- Small and medium-sized applications
- Services that require fast DDoS protection without complex setup

## Core concepts

### Protected IP

A public IP address with a port assigned by Gcore. Users connect to this endpoint.

### Origin server

The backend server that receives filtered traffic.

### Translation rule

A configuration that maps:

- Origin IP
- Port
- Protocol
- Application type

One combination equals one rule.

### Application type

A predefined countermeasure profile optimized for specific applications.

## After setup

After enabling Edge Proxy:

- A protected IP address and port are assigned
- DDoS protection is active on all configured port mappings
- Traffic metrics are available in the Traffic Overview page
- Proxies can be edited or deleted at any time
