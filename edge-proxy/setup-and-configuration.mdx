Edge Proxy assigns a protected IP to the origin server and routes incoming traffic through Gcore's DDoS mitigation infrastructure. To configure Edge Proxy, specify the origin server's public IPv4 address, the application port, the protocol (TCP or UDP), and the application type.

<Steps>
<Step title="Open the Protected IPs page">

In the [Gcore Customer Portal](https://portal.gcore.com/), navigate to **Security → Edge Proxy → Protected IPs**.

- To add the first proxy, click **Add your first protection**.
- To add another proxy to an existing list, click **Add Protection**.

</Step>
<Step title="Configure proxy settings">

Fill in the configuration form.

<Frame>![Edge Proxy configuration form](/images/docs/edge-proxy/edge-proxy-configuration.png)</Frame>

**Proxy name** — Enter a name to identify this proxy. Example: `Main game server`.

**Origin server IP address** — Enter the origin server's public IPv4 address. Example: `195.14.50.0`.

**Port mappings** — Each mapping defines one protected application endpoint.

1. Enter the port the application listens on (1–65535). Example: `8080`.
2. Select the protocol: TCP or UDP.
3. Select the application type from the dropdown — predefined profiles include Minecraft, Counter-Strike, FiveM, Rust, TeamSpeak, and Unity.
4. Click **Add rule**.

Repeat to add additional mappings for applications that use more than one port.

</Step>
<Step title="Start protection">

Click **Start protecting**. The system processes the request in approximately 20 seconds. A progress indicator appears during setup.

</Step>
<Step title="Get protected endpoint">

After activation, the proxy appears in the Protected IPs list with the protected IP:port, active status, and origin details. Copy the protected IP:port value to use in the next step.

</Step>
<Step title="Update the service">

Update the application's client configuration to connect to the protected IP:port instead of the origin server's address directly. If clients connect using a domain name, update its A record to point to the protected IP.

</Step>
<Step title="Verify protection">

Verify that:

- the service is reachable
- traffic flows through the protected IP
- the proxy status is **Active**

If the status doesn't change or the service is unreachable, the [troubleshooting&nbsp;guide](/edge-proxy/faqs-and-trobleshooting) lists the most common causes.

</Step>
</Steps>

## Manage protected IPs

### View proxies

The **Protected IPs** list shows each proxy's name, status, protected IP and port, origin IP, protocol, application type, and creation date.

### Delete a proxy

Click **Delete** next to the proxy and confirm the action in the dialog.
