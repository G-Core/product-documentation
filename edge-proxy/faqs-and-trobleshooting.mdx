Common questions and troubleshooting steps for Edge Proxy.

## General questions

### Edge Proxy explained

Edge Proxy is a DDoS protection service that runs on the Gcore CDN edge network. It filters incoming traffic before forwarding it to the origin server.

### Supported application types

Any TCP or UDP application can be protected, including:

- Game servers — Counter-Strike, Minecraft, FiveM
- Web servers and APIs
- DNS servers
- Voice servers — TeamSpeak
- Custom TCP or UDP applications

### Setup duration

The setup process typically takes less than five minutes. After clicking **Start protecting**, the proxy is provisioned in approximately 3 minutes.

### Does Edge Proxy add latency?

Edge Proxy is designed to add minimal latency. Traffic is routed to the nearest Point of Presence (PoP), which may improve routing efficiency in some cases.

### Can I protect multiple applications?

Yes. Multiple proxy rules can be created. Several rules can share the same protected IP address.

### IP range requirements

No IP range or AS number is required. Gcore assigns a protected Anycast IP address automatically.

## Setup and configuration

### Can I select the protected IP address?

No. The system automatically assigns a protected IP from the Gcore Anycast pool.

### Can I reuse the same origin IP?

Yes. Multiple proxy rules can use the same origin IP with different ports, protocols, or application types.

### Wrong application type selection

Incorrect configuration may cause legitimate traffic to be blocked during an attack.

To resolve this:

- Delete the proxy
- Create a new one with the correct application type

### Application type not in list

If the application type is not available in the list:

- Select a matching engine or protocol — Unity, Unreal, RakNet
- Use TCP protection for TCP-based applications
- Contact support if the application type is unclear

### Can I edit an existing proxy?

Yes, but partially. The proxy name and port mappings can be changed at any time using the **Edit protection** form. The origin server IP address cannot be changed after creation — to use a different origin IP, delete the proxy and create a new one. A new protected IP may be assigned.

## DNS and connectivity

### Do I need to update DNS?

Yes, if the application uses a domain name.

- Update the A or AAAA record to the protected IP
- If using direct IP connections, share the new IP:PORT with users

### DNS propagation time

Propagation depends on the DNS TTL and can take from a few minutes to 48 hours.

To reduce downtime:

- Lower the TTL before making changes
- Update the record
- Increase the TTL after propagation completes

### Existing connections during migration

Existing connections to the origin continue until clients reconnect. New connections use the protected IP.

## Troubleshooting

### Clients cannot connect to the protected IP

Symptoms:

Users cannot connect to the application using the protected IP and port.

Steps to resolve:

1. Verify the protected IP and port
   - Check the values in the Customer Portal
   - Confirm the full IP:PORT pair is used

2. Check the origin server
   - Confirm the application is running
   - Verify it listens on the correct port and protocol

3. Verify protocol configuration
   - Confirm TCP or UDP matches the application

4. Check firewall rules
   - Allow traffic from Gcore edge IPs
   - Do not restrict access to old client IPs only

5. Check DNS propagation
   - Verify DNS resolution using tools like `dig` or `nslookup`

### High latency through the proxy

Symptoms:

Users experience increased latency or lag.

Steps to resolve:

1. Compare latency
   - Test direct connection vs protected IP

2. Check origin location
   - Confirm the origin server is close to Gcore PoPs

3. Verify application type
   - Incorrect settings may cause retransmissions

### Legitimate traffic is blocked during an attack

Symptoms:

Real users cannot connect or are disconnected during an attack.

Steps to resolve:

1. Verify application type
   - Use the correct countermeasure for the application type

2. Contact Gcore support
   - Open a ticket via the [support&nbsp;portal](#contact-gcore-support)

### Proxy is active but no traffic flows

Symptoms:

The proxy shows as active, but no traffic reaches the application.

Steps to resolve:

1. Verify client configuration
   - Confirm clients use the protected IP:PORT

2. Check the port
   - Confirm clients connect to the correct protected port

3. Test from another network
   - Avoid testing from the same network as the origin

## Contact Gcore support

If the issue persists, contact Gcore support via the [support&nbsp;portal](https://support.gcore.com).

Use the **Submit a request** form to create a support ticket, and provide:

- Account ID
- Proxy name and protected IP
- Description of the issue
- Steps already taken
- Screenshots or logs, if available
