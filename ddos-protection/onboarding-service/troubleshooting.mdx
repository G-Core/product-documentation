## **Common issues and solutions**

### **Issue: network submission fails**

**Possible causes:**

- Invalid CIDR format.
- AS Number outside valid ranges.
- LOA file exceeds 10 MB.
- Missing required fields.

**Solutions:**

- Verify CIDR format (minimum `/24`).
- Check AS Number is in valid ranges (`1-65534` or `131072-4294967294`).
- Compress or optimize LOA PDF if needed.
- Ensure all required fields are completed.

### **Issue: network stuck in "Pending" status**

**Possible causes:**

- NOC review in progress.
- Additional information required.
- Technical issues.

**Solutions:**

- Wait for email notification (typically within 24-48 hours).
- Check your email (including spam folder).
- Contact support if status doesn't change after 48 hours.

### **Issue: cannot save network configuration**

**Possible causes:**

- Invalid GRE endpoint format.
- Profile not selected.
- Network connection issues.

**Solutions:**

- Verify GRE endpoint is a valid IP address.
- Ensure a protection profile is selected.
- Check your internet connection and try again.

### **Issue: custom profile not appearing in dropdown**

**Possible causes:**

- Profile not saved properly.
- Browser cache issues.
- Profile creation failed.

**Solutions:**

- Verify profile was saved successfully.
- Refresh the page or clear browser cache.
- Recreate the profile if necessary.

## **Getting help**

If you encounter issues not covered in this guide:

1. **Check email notifications**
   - Review all emails from the support team.
   - Follow any instructions provided.
2. **Review documentation**
   - Check the Confluence documentation for updates.
   - Review protection profile templates and examples.
3. **Contact support**   
   - Use the support email provided in the notification.
   - Include relevant details (network CIDR, AS Number, error messages).
