## Overview

The DDoS protection profiles API provides access to protection profile data, including system-managed and user-created profiles. Each profile includes a `managed_by` field indicating the management type.

## Endpoint

`GET /ddos-protection/profiles`

### Query parameters

| Parameter    | Type   | Required | Description |
|-------------|--------|----------|-------------|
| `managed_by` | string | No | Filter by management type: `system` or `user` |

### Response fields

| Field        | Type   | Description |
|-------------|--------|-------------|
| `id`         | string | Unique profile identifier |
| `name`       | string | Profile display name |
| `managed_by` | string | Management type: `system` or `user` |
| `description`| string | Profile description |
| `settings`   | object | Protection configuration |

## Profile management types

### System-managed (`managed_by: "system"`)

Profiles created and maintained by Gcore:
- Cannot be modified or deleted via API
- Contain predefined protection configurations
- Updated automatically with security improvements

### User-created (`managed_by: "user"`)

Profiles created through the Customer Portal or API:
- Can be modified and deleted
- Allow custom protection configurations

## Requests

Retrieve all profiles:

```bash
curl -X GET "https://api.gcore.com/ddos-protection/profiles" \
  -H "Authorization: Bearer API_TOKEN" \
  -H "Content-Type: application/json"
```

Filter by system-managed profiles:

```bash
curl -X GET "https://api.gcore.com/ddos-protection/profiles?managed_by=system" \
  -H "Authorization: Bearer API_TOKEN"
```

Filter by user-created profiles:

```bash
curl -X GET "https://api.gcore.com/ddos-protection/profiles?managed_by=user" \
  -H "Authorization: Bearer API_TOKEN"
```

```json
      "name": "Basic Protection",

      "managed_by": "user",
      "description": "High-sensitivity custom profile",
      "settings": {
        "threshold": "1000",

      }
    }

}
```
