<p>API Discovery allows you to manage and protect REST API endpoints associated with your domain, including legacy and shadow APIs.</p>

<p>This feature scans your domain's traffic and presents a compiled list of APIs for your validation so you can decide which endpoints require WAAP protection. You can also add endpoints manually or import them in a Swagger file.</p>

<Info>
**Info**

The API Discovery feature is included in the *Enterprise* plan. To enable it for your domain, contact our [sales team](https://gcore.com/waap-contact-sales).
</Info>

## Overview

<p>API Discovery covers both public and private REST APIs. Public APIs are available to anyone, while private APIs are only accessible to authorized users.</p>

<Tip>
**Tip**

If API Discovery isn't available in your plan and you can't automatically map APIs with API Baseline, use the [API base path](/waap/api-discovery-and-protection/configure-api-base-path) feature. It lets you manually add API paths that belong to your domain and should be protected by WAAP.
</Tip>

<p>To access the feature:</p>

1. In the [Gcore Customer Portal](https://portal.gcore.com/accounts/reports/dashboard), navigate to **WAAP** > **Domains**.

<Frame>![Domains page in the Customer Portal](/images/docs/waap/api-discovery-and-protection/waap-domains.png)</Frame>

2. Find the needed domain and click its name to open it. You'll be directed to the **Overview** page.

3. In the sidebar, click **API Discovery**. You can protect and manage APIs on the following pages:

   * **API Baseline**: Contains all API endpoints protected by WAAP. On this page, you can add, edit, and view your APIs.

   * **Potential APIs**: Endpoints that have been auto-detected by the system and require verification. This is where you can reject or approve endpoints as valid APIs.

   * **Scan History**: A history of network scans run by the system.

   * **API Discovery Settings**: A page with available configuration and customization options where you can set up scan settings and upload a Swagger file.

<p>The functionality on these pages is described in more detail in the following sections.</p>

## Statuses of scanned endpoints

<p>Each API endpoint can have a specific status. This status reflects the results of a system scan or Swagger file check:</p>

- **Potential**: The endpoint has been added to the system through a network scan and is awaiting to be marked as a confirmed API or not an API. If you confirm the endpoint, it'll appear on the **API Baseline** page along with other APIs and will automatically get the **Confirmed** status.

- **Confirmed**: The endpoint is a valid API and should be protected by WAAP. An API endpoint automatically gets a "Confirmed" status if it was added to the API Baseline page or via Swagger file scan. You can change the status at any time.

- **Not an API**: The potential endpoint is not a valid API and doesn't require protection. You can manually assign this status to endpoints that were falsely detected during a network scan.

- **Delisted**: The endpoint was originally detected in the Swagger file, but it's subsequently missing after a later Swagger scan.

## API baseline

<p>All your protected API endpoints are listed on this page. Here, you can do the following:</p>

- Manually add endpoints that should be protected by WAAP

- Group endpoint using tags to manage access to your APIs

<Frame>![API Baseline page in the Customer Portal](/images/docs/waap/api-discovery-and-protection/api-baseline.png)</Frame>

### Add endpoints to API baseline

<p>If you manually add APIs on the **API Baseline** page, they will automatically get the confirmed status. You can change the status at any time.</p>

<p>To add a new API endpoint:</p>

1. In the top-right corner of the screen, click **Add an API**. This will open up a menu with the required settings on the right side of the page.

2. Configure your APIs as follows:

   * **Scheme**: Choose the relevant scheme – HTTP or HTTPS.
   * **Method**: Provide the relevant API method.
   * **Endpoint**: Enter a full path to the endpoint.
   * **Version** (optional): To maintain API efficiently and consistently, you can establish an API versioning to keep track of API changes, relevant fixes, or deprecations.
   * **API group** (optional): Organize endpoints into related groups.
   * **Tags**: Configure access to your endpoints. Add reserved tags to set up advanced API protection.

3. Click **Add** to save the changes.

### Group endpoints based on tags

<p>You can categorize endpoints by marking them with reserved tags and then create custom rules to specifically protect those tagged endpoints.</p>

<p>For example, you can mark certain APIs with the **Indicate API Privileged User** or **Indicate API Admin User** tags to make those paths accessible only to privileged or admin users. Refer to our [configure API access with reserved tags](/waap/api-discovery-and-protection/configure-api-access-with-reserved-tags) guide for instructions.</p>

### View endpoints

<p>To inspect existing APIs in more detail or modify endpoint status, API groups, or associated tags:</p>

1. In the [Gcore Customer Portal](https://portal.gcore.com/accounts/reports/dashboard), navigate to **WAAP** > **Domains**.

2. Find the domain where you want to configure APIs and click the domain name to open it.

3. In the sidebar menu, click **API Discovery**. You'll be directed to the **API Baseline** page.

4. Find the endpoint you want to inspect and click the three-dot icon next to it.

5. Select **View details**.

6. On the page that opens, you can check additional information about your existing endpoint or change endpoint status, API groups, and tags.

7. If you made any changes, click **Save** to apply them. Otherwise, close the menu.

### Edit endpoints

<p>If you want to update the actual endpoint path, you can do it as follows:</p>

1. In the [Gcore Customer Portal](https://portal.gcore.com/accounts/reports/dashboard), navigate to **WAAP** > **Domains**.

2. Find the domain where you want to modify APIs and click the domain name to open it.

3. In the sidebar menu, click **API Discovery**. You'll be directed to the **API Baseline** page.

4. Find the endpoint you want to edit and click the three-dot icon next to it.

5. Enter a new or updated path.

<p>When editing a path, you can use curly braces to signify query string parameters, as in <code>/api/v1/stacks/{stack_id}/sites/{site_id}</code>.</p>

## Potential APIs

<p>When the API Discovery feature performs a network scan, all detected endpoints appear on this page. Here, you can manage their status and determine whether or not they are confirmed APIs.</p>

<Frame>![Potential APIs page in the Customer Portal](/images/docs/waap/api-discovery-and-protection/potential-apis.png)</Frame>

### Verify detected endpoints

<p>After you identify the endpoint as a confirmed API, it will be moved to the API Baseline page. All rejected APIs will be removed from the system.</p>

## Scan history

<p>This page shows the history of scans performed by the system:</p>

- Automated network scans. The default scan frequency is set to 24 hours, but you can adjust it on the **API Discovery Settings** page.

- Scans of the uploaded Swagger file (if you added it to the **API Discovery Settings** page).

<Frame>![Scan History page in the Customer Portal](/images/docs/waap/api-discovery-and-protection/scan-history.png)</Frame>

<p>The scan history also displays information about the scan time, analyzed source, and whether a scan was successful or resulted in errors. You'll see information about the scan process and any related errors in the **Message** column.</p>

## API Discovery Settings

<p>On this page, you can configure specifications and network scan intervals for detecting potential API endpoints.</p>

<Frame>![API Discovery Settings page in the Customer Portal](/images/docs/waap/api-discovery-and-protection/settings.png)</Frame>

### Configure base path

<p>For guidelines on how to configure the API base path in Gcore WAAP, check out the [Manually add endpoints to API base path](/waap/api-discovery-and-protection/configure-api-base-path) guide.</p>

### Specify API specification files

<p>One of the ways to add APIs to the API Discovery feature is to parse an uploaded JSON or YAML Swagger file. Detected API endpoints will appear on the page with the **Confirmed** status.</p>

<p>Uploading a Swagger file can significantly reduce the work needed for categorizing potential APIs, as this will automatically classify most of the endpoints that comprise the API Baseline.</p>

<p>This is how you can do it:</p>

1. On the **API Discovery Settings** page, navigate to the **API specifications file** section.

2. Choose the preferred method for adding APIs:

   * **Path**: Provide a URL to the Swagger file
   * **Upload**: Import locally stored Swagger file

3. Click **Start Swagger scan**. This will trigger a scan of the listed paths or files for APIs. Once the scan is complete, potential endpoints will be added to the **API Baseline** page.

## Set up scan settings

<p>In this section, you can choose how the system will detect your APIs:</p>

- **Network scans**. When this toggle is enabled, API Discovery will automatically scan your site's URLs and detect new potential API endpoints.

- **API specification scans**. When this toggle is enabled, API Discovery will scan your uploaded API Specification file or specified API paths to see if any new endpoints were added or if any existing endpoints were removed (delisted).

<p>You can customize scan intervals by selecting the relevant option from the **Scan every** dropdown.</p>

## List API paths

<p>Returns all API paths registered in the domain's API Baseline, with optional filters for status, method, source, and group.</p>

<Tabs>
  <Tab title="Python SDK">
    ```python
    import os
    import gcore

    client = gcore.Gcore(api_key=os.environ["GCORE_API_KEY"])
    domain_id = int(os.environ["WAAP_DOMAIN_ID"])

    paths = client.waap.domains.api_paths.list(domain_id)
    print(f"Total paths: {paths.count}")
    for path in paths.results:
        print(f"{path.method} {path.path}  status={path.status}  source={path.source}")
    ```
  </Tab>
  <Tab title="Go SDK">
    ```go
    package main

    import (
        "context"
        "fmt"
        "os"
        "strconv"

        "github.com/G-Core/gcore-go"
        "github.com/G-Core/gcore-go/option"
        "github.com/G-Core/gcore-go/waap"
    )

    func main() {
        domainID, _ := strconv.ParseInt(os.Getenv("WAAP_DOMAIN_ID"), 10, 64)
        client := gcore.NewClient(option.WithAPIKey(os.Getenv("GCORE_API_KEY")))

        result, err := client.Waap.Domains.APIPaths.List(
            context.Background(),
            domainID,
            waap.DomainAPIPathListParams{},
        )
        if err != nil {
            panic(err)
        }
        fmt.Printf("Total paths: %d\n", result.Count)
        for _, p := range result.Results {
            fmt.Printf("%s %s  status=%s  source=%s\n", p.Method, p.Path, p.Status, p.Source)
        }
    }
    ```

## Add an API path

<p>Manually adds an endpoint to the API Baseline. Manually added paths receive `CONFIRMED_API` status and `USER_DEFINED` as source automatically.</p>

<Tabs>
  <Tab title="Python SDK">
    ```python
    import os
    import gcore

    client = gcore.Gcore(api_key=os.environ["GCORE_API_KEY"])
    domain_id = int(os.environ["WAAP_DOMAIN_ID"])

    path = client.waap.domains.api_paths.create(
        domain_id,
        http_scheme="HTTPS",
        method="GET",
        path="/api/v1/users",
        api_version="v1",
        api_groups=["user-service"],
    )
    print(f"Created: {path.id}  status={path.status}")
    ```
  </Tab>
  <Tab title="Go SDK">
    ```go
    package main

    import (
        "context"
        "fmt"
        "os"
        "strconv"

        "github.com/G-Core/gcore-go"
        "github.com/G-Core/gcore-go/option"
        "github.com/G-Core/gcore-go/waap"
    )

    func main() {
        domainID, _ := strconv.ParseInt(os.Getenv("WAAP_DOMAIN_ID"), 10, 64)
        client := gcore.NewClient(option.WithAPIKey(os.Getenv("GCORE_API_KEY")))

        path, err := client.Waap.Domains.APIPaths.New(
            context.Background(),
            domainID,
            waap.DomainAPIPathNewParams{
                HTTPScheme: waap.DomainAPIPathNewParamsHTTPSchemeHTTPS,
                Method:     waap.DomainAPIPathNewParamsMethodGet,
                Path:       "/api/v1/users",
                APIVersion: gcore.String("v1"),
                APIGroups:  []string{"user-service"},
            },
        )
        if err != nil {
            panic(err)
        }
        fmt.Printf("Created: %s  status=%s\n", path.ID, path.Status)
    }
    ```
  </Tab>
  <Tab title="curl">
    ```bash

         -d '{
           "http_scheme": "HTTPS",
           "method": "GET",
           "path": "/api/v1/users",
           "api_version": "v1",
    ```

    ```json
      "source": "USER_DEFINED"
    ```

## Update an API path

<p>Updates an existing API path's status, groups, or tags. The path ID is the UUID returned when the endpoint was created or listed. Returns `204 No Content` on success.</p>

<p>Available status values: `CONFIRMED_API`, `POTENTIAL_API`, `NOT_API`, `DELISTED_API`.</p>

<Tabs>
  <Tab title="Python SDK">
    ```python
    import os
    import gcore

    client = gcore.Gcore(api_key=os.environ["GCORE_API_KEY"])
    domain_id = int(os.environ["WAAP_DOMAIN_ID"])

    path_id = "3090f4ba-b2c4-4a7a-a15e-399e4c0abd23"

    client.waap.domains.api_paths.update(
        path_id,
        domain_id=domain_id,
        status="NOT_API",
        api_groups=[],
    )
    print("Updated")
    ```
  </Tab>
  <Tab title="Go SDK">
    ```go
    package main

    import (
        "context"
        "fmt"
        "os"
        "strconv"

        "github.com/G-Core/gcore-go"
        "github.com/G-Core/gcore-go/option"
        "github.com/G-Core/gcore-go/waap"
    )

    func main() {
        domainID, _ := strconv.ParseInt(os.Getenv("WAAP_DOMAIN_ID"), 10, 64)
        client := gcore.NewClient(option.WithAPIKey(os.Getenv("GCORE_API_KEY")))

        pathID := "3090f4ba-b2c4-4a7a-a15e-399e4c0abd23"

        err := client.Waap.Domains.APIPaths.Update(
            context.Background(),
            pathID,
            waap.DomainAPIPathUpdateParams{
                DomainID:  domainID,
                Status:    waap.DomainAPIPathUpdateParamsStatusNotAPI,
                APIGroups: []string{},
            },
        )
        if err != nil {
            panic(err)
        }
        fmt.Println("Updated")
    }
    ```
  </Tab>
  <Tab title="curl">
    ```bash
    PATH_ID="3090f4ba-b2c4-4a7a-a15e-399e4c0abd23"

    curl -X PATCH "https://api.gcore.com/waap/v1/domains/${WAAP_DOMAIN_ID}/api-paths/${PATH_ID}" \
         -H "Authorization: APIKey ${GCORE_API_KEY}" \
         -H "Content-Type: application/json" \
         -d '{
           "status": "NOT_API",
           "api_groups": []
         }'
    ```

    Returns `204 No Content`.

## Delete an API path

## View scan settings

<p>Returns the current API Discovery configuration: scan type toggles, scan intervals, and the Swagger file URL (if configured).</p>

<Tabs>
  <Tab title="Python SDK">
    ```python
    import os
    import gcore

    client = gcore.Gcore(api_key=os.environ["GCORE_API_KEY"])
    domain_id = int(os.environ["WAAP_DOMAIN_ID"])

    settings = client.waap.domains.api_discovery.settings.get(domain_id)
    print(f"Traffic scan enabled: {settings.traffic_scan_enabled}")
    print(f"Traffic scan interval: {settings.traffic_scan_interval_hours}h")
    print(f"Spec scan enabled: {settings.description_file_scan_enabled}")
    print(f"Spec URL: {settings.description_file_location}")
    ```
  </Tab>
  <Tab title="Go SDK">

## Update scan settings

<p>Updates API Discovery scan configuration. All fields are optional — only the fields included in the request body are changed. The scan interval must be between one and 24 hours.</p>

<Tabs>
  <Tab title="Python SDK">
    ```python
    import os
    import gcore

    client = gcore.Gcore(api_key=os.environ["GCORE_API_KEY"])
    domain_id = int(os.environ["WAAP_DOMAIN_ID"])

    client.waap.domains.api_discovery.settings.update(
        domain_id,
        traffic_scan_enabled=True,
        traffic_scan_interval_hours=12,
    )
    print("Settings updated")
    ```
