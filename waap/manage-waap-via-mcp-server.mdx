Gcore MCP Server enables AI assistants to manage Web Application and API Protection (WAAP) through natural language commands. This guide covers WAAP-specific tools and example conversations.

For general setup instructions, see [Gcore MCP Server overview](/account-settings/integrations/gcore-mcp-server-overview).

## Configure for WAAP

Set the `GCORE_TOOLS` environment variable to expose WAAP tools:

```bash
GCORE_TOOLS="waap.*"
```

This exposes approximately 80 WAAP tools covering domains, policies, rules, and security analytics.

## Available WAAP tools

| Resource | Operations | Description |
|----------|------------|-------------|
| `domains` | list, create, get, update, delete | Protected domains |
| `advanced_rules` | list, create, get, update, delete | Custom security rules |
| `custom_page_sets` | list, create, get, delete | Custom error pages |
| `tags` | list, create, get, delete | Domain tags |
| `analytics` | get | Security analytics |
| `insights` | get | Security insights |
| `statistics` | get | Traffic statistics |
| `ip_info` | lookup | IP reputation lookup |

## Tool reference

### waap.domains.list

Lists all protected domains.

**Parameters:** None required

**Returns:** Array of domains with ID, status, mode, and statistics

### waap.domains.create

Adds a domain to WAAP protection.

**Parameters:**
- `domain` (required): Domain name
- `mode`: Protection mode (monitoring, blocking)

### waap.advanced_rules.create

Creates a custom security rule.

**Parameters:**
- `domain_id` (required): Domain ID
- `type` (required): Rule type
- `action` (required): Action (block, allow, challenge)
- `conditions` (required): Rule conditions

### waap.ip_info.lookup

Looks up IP reputation.

**Parameters:**
- `ip` (required): IP address
