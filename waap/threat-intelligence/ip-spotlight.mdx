IP Spotlight is a threat analytics tool that provides detailed information about a specific IP address.

It operates on the IP-related information we collect from our network to give you insights about the clients that access your domains. This information helps you make better decisions when creating WAF rules and helps with policy configuration to prevent and mitigate attacks.

<Info>
  **Info**

  The IP Spotlight feature is available on the **Enterprise** plan. To enable it for your domain, contact our [sales team](https://gcore.com/waap-contact-sales).
</Info>

## Analyze an IP address

IP Spotlight provides IP details as its source, total number of requests, destinations, whois data, and whether it has been involved in any malicious activity against other domains within our network.

To check an IP address, follow these steps:

1. In the [Gcore Customer Portal](https://accounts.gcore.com/reports/dashboard), navigate to **WAAP** > **IP Spotlight**.

2. In the dropdown at the top-right of the page, select the IP address that you want to inspect.

3. Optionally, use the time filter buttons (**24h**, **7d**, **30d**) to set the time range for the displayed data.

Once you select an IP, the page displays a summary of traffic statistics and detailed threat information for that address.

## Understand the traffic at a glance

The stats bar at the top of the page answers the first question you need when investigating an IP: *how much activity are we dealing with, and how much of it is malicious?*

- **Total requests** — the full volume of traffic from this IP in the selected period.
- **Detected attacks** — how many of those requests were flagged as attack attempts.
- **Prevented attacks** — how many were actively blocked by your WAAP policy.
- **Passed requests** — how many got through, either because they were legitimate or because no rule matched.

A high ratio of detected to prevented attacks may indicate gaps in your WAF configuration that are worth investigating.

## Assess the threat level
The **Threat Summary** is where you get the full picture of what this IP has been doing — on your domains and across the Gcore network.
| Field | What it tells you |
|---|---|
| **IP** | The address under investigation. |
| **Risk Assessment** | The overall threat score: **No risk**, **Low**, **Medium**, **High**, or **Extreme**. Scores are based on internal and external threat intelligence sources, including blocklists and observed behavior. |
| **Tags** | Behavioral labels that describe what kind of activity the IP is known for — for example, `injectionattack`, `multiplerepeatedrequests`, or `headlessbrowser`. These are useful when writing targeted WAF rules. |
| **Attack Types** | The number of distinct attack categories triggered by this IP. A high number suggests a broad, multi-vector threat actor. |
| **Domains** | How many of your domains this IP has targeted. If it's hitting multiple domains, it may be a coordinated attack rather than organic traffic. |
| **Paths** | The variety of URL paths requested. High path counts can indicate scanning or enumeration behavior. |
| **User Agents** | How many different user agents this IP has used. Frequent agent switching is a common evasion technique. |
| **Sessions** | Total number of sessions originating from this IP. |
<Info>
  **Info**

  Risk Assessment scores are not permanent. If an IP stops engaging in malicious activity, its score decreases over time.
</Info>

## Spot patterns over time

The **Traffic Timeline** shows the request volume from this IP over the selected period. Use it to answer questions like:
- Was this a single burst of activity, or is traffic ongoing?
- Does the pattern align with a known attack window?
- Is traffic increasing, decreasing, or cyclical?
A sustained or recurring spike is a strong signal that blocking the IP or creating a rate-limiting rule is warranted.

## Understand what the IP is targeting

The **Top activity** section breaks down the IP's behavior into four categories. Together, they tell you *what* the IP is after and *how* it's going about it.
- **Top Rules** — which of your WAF rules this IP is triggering most. This helps you see if your existing policies are already catching the threat, and which rules may need tuning.
- **Top User Agents** — the clients or tools the IP is using. A list dominated by automation frameworks like `python-requests` or `curl` is a red flag for scripted attacks.
- **Top Domains** — which of your domains are being targeted. If multiple domains appear, the IP may be indiscriminately scanning your infrastructure.
- **Top Paths** — the most requested URL paths. Repeated access to `/login`, `/api/`, or `/admin` paths is a common sign of credential stuffing or vulnerability probing.

### Detailed attack breakdown

The **Top Attacks** table drills into the specific attack types triggered by this IP, including which domains and paths were targeted and the total event count. Use this table to understand the scope of the threat and decide whether to tighten an existing rule or create a new one.

### User agent analysis

The **Top User Agents** table shows every client signature used by this IP, along with the number of requests, attacks, sessions, and affected domains and paths per agent. If you see a legitimate user agent alongside obvious bots, the IP may be rotating agents to evade detection.

### Domain and path exposure

The **Top Domains** and **Top Paths** tables give you a precise view of your exposure: which resources are being probed and how frequently. This information is useful for scoping a targeted block rule or adjusting rate limits on specific endpoints.
