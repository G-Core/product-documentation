## Manage Security Insights  

When an insight is selected, its status can be changed using the **Mark as read**, **Silence**, and **Close** buttons.

<Frame>
  ![Manage security insights](/images/docs/waap/security-insights/security-insights-manage.png)
</Frame>

### Mark insights as read

Insights are not automatically marked as read when viewed. To mark an insight as read, click **Mark as read**. To keep an insight as a reminder for later action or review, leave it unread.

1. Select the insight to mark as unread in the **Security Insights** list.

<Frame>
  ![Select the security insight](/images/docs/waap/security-insights/select-insight.png)
</Frame>

2. Click the **Mark as unread** button.

<Frame>
  ![Mark the security insight as unread](/images/docs/waap/security-insights/mark-as-unread.png)
</Frame>

### Silence insights  

An insight can be silenced if it has been viewed, but no action is taken, or it is not closed. Silencing an insight means pausing all notifications for a specific period (e.g., a week, month, custom period, or indefinitely).  

An insight can be silenced in three ways:

1. For a specific high-risk IP address **allowed by any rule**.
2. For a particular high-risk IP address **allowed by a specific rule**.
3. For **all high-risk IP addresses**.

To silence an insight, follow these steps:

1. Select it in the **Security Insights** view list.

<Frame>
  ![Select the security insight](/images/docs/waap/security-insights/select-insight.png)
</Frame>

2. Click the **Silence** button in its details.

<Frame>
  ![Select Silence](/images/docs/waap/security-insights/selection.png)
</Frame>

3. Select the relevant silence rule and set the notification suppression duration.

<Frame>
  ![Select the silence rule and the duration](/images/docs/waap/security-insights/silence-rule-duration.png)
</Frame>

4. Click the **Save** button. 

<Tip>
  **Tip**

  To stop silencing an insight, see the [Manage Silence Rules](/waap/threat-intelligence/security-insights/manage-silence-rules) guide.
</Tip>

### Close insights 

After an insight is reviewed, it can be closed and removed from the list by following these steps:


1. Select the insight to close.

<Frame>
  ![Select the security insight](/images/docs/waap/security-insights/select-insight.png)
</Frame>

2. Click the **Close** button.

<Frame>
  ![Select Close](/images/docs/waap/security-insights/selection.png)
</Frame>

<Info>
  **Info**

  Closed insights are automatically deleted after 30 days unless reopened.
</Info>

To open a closed insight for further investigation, reassessment, or verification of mitigation effectiveness, follow these steps:

1. In the **Status** dropdown, select **Closed**.

<Frame>
  ![Filter the insights with the Status dropdown](/images/docs/waap/security-insights/select-insights-dropdown.png)
</Frame>

2. Select the insight to reopen.

<Frame>
  ![Select the security insight](/images/docs/waap/security-insights/select-insight.png)
</Frame>

3. Click the **Reopen** button.

<Frame>
  ![Select Reopen](/images/docs/waap/security-insights/reopen-insight.png)
</Frame>

The insight will reappear in the list on the **Security Insights** view.
