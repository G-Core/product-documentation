The WAAP **Events** page provides detailed visibility into traffic and security events processed by WAAP.

This page provides detailed statistics on HTTP requests, displayed as a graph and a table, that can be used to analyze requests, investigate attacks, and review applied protection actions.

<Frame>
  ![Events page in the Customer Portal](/images/docs/waap/analytics/waap-events-blur.png)
</Frame>

<Tip>
  **Tip**

  If you want to view analytics for a different domain, use the dropdown in the top-right corner of the screen and select the needed domain.
</Tip>

### Requests graph

This graph shows information about actions that were applied to incoming requests. The data is displayed in a plot chart format that gives a comprehensive overview of both good and bad requests—potential security threats detected based on a triggered rule.

<Frame>
  ![WAAP Events graph](/images/docs/waap/analytics/waap-events-graph.png)
</Frame>

The information is displayed according to the selected time period, which will update the requests graph and the requests table.

<Frame>
  ![WAAP Events time frame](/images/docs/waap/analytics/waap-events-time-range.png)
</Frame>

### Requests table

This table displays incoming requests for the selected time frame, including good and bad ones (potential threats). Request history is retained for 30 days.

<Frame>
  ![Events table](/images/docs/waap/analytics/waap-events-table-blur.png)
</Frame>

Click any event row to view detailed information about the request.

### Filters

You can apply filters to narrow down the displayed events and focus on specific traffic patterns or security incidents.

<Frame>
  ![Events filters](/images/docs/waap/analytics/waap-events-filters.png)
</Frame>

The Events page supports the following filters:

- **Time range** – Select the period for which events are displayed.
- **Event** – Filter events by the detected security event type.
- **Actions** – Filter requests by the action enforced by WAAP:
  - **Blocked** – Malicious traffic that was denied.
  - **Allowed / Passed** – Legitimate traffic that was permitted.
  - **Monitored** – Suspicious or malicious traffic detected while WAAP is operating in monitoring mode. In this mode, requests are logged but not blocked.
  - **CAPTCHA** – Requests challenged with a CAPTCHA. If the challenge fails, the request is blocked.
  - **JS Challenge** – A background JavaScript validation used to verify legitimate browsers.
- **Countries** – Filter requests by the country of origin.
- **IP Address** – Filter events originating from a specific client IP address.
- **Response Codes** – Filter requests by HTTP response status codes.
- **HTTP Methods** – Filter by request method (for example, GET, POST, PUT).
- **Path** – Filter requests targeting a specific URL path.
- **Request IDs** – Search for specific request identifiers. Multiple request IDs can be entered, separated by commas.
- **Reference IDs** – Group and filter similar attack requests. A reference ID represents requests from the same IP address that triggered the same security rule.
- **Session IDs** – Filter requests belonging to the same client session. A session ID groups a sequence of requests generated during one session.

<Frame>
  ![Events Actions filter](/images/docs/waap/analytics/waap-events-actions.png)
</Frame>

When filters are applied, both the graph and the table update automatically to reflect the selected criteria.

### Export Events

To export Events, click the download icon in the top-right corner of the page.

<Frame>
  ![WAAP Events export](/images/docs/waap/analytics/waap-events-csv.png)
</Frame>

The table data is downloaded in CSV format. If filters are applied, only the filtered dataset is exported.
