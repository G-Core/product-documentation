The **Events** page provides comprehensive information about incoming traffic to your domain. It features detailed statistics about recent requests, potential threats, and frequently triggered rules, allowing you to fine-tune your WAAP settings based on the analyzed data.

<Frame>
  ![Events page in the Customer Portal](/images/docs/waap/analytics/waap-events-blur.png)
</Frame>

<Tip>
  **Tip**

  If you want to view analytics for a different domain, use the dropdown in the top-right corner of the screen and select the needed domain.
</Tip>

This page provides detailed statistics on HTTP requests, displayed as a graph and as a table.

### Requests graph

This graph shows information about actions that were applied to incoming requests. The data is displayed in a plot chart format that gives a comprehensive overview of both good and bad requestsâ€”potential security threats detected based on a triggered rule.

<Frame>
  ![WAAP Events graph](/images/docs/waap/analytics/waap-events-graph.png)
</Frame>

The information is displayed according to the selected time period, which will update the requests graph and the requests table.

<Frame>
  ![WAAP Events time frame](/images/docs/waap/analytics/waap-events-time-range.png)
</Frame>

### Requests table

This table displays incoming requests for the selected time frame, including good and bad ones (potential threats). Request history is retained for 30 days.

<Frame>
  ![Events table](/images/docs/waap/analytics/waap-events-table-blur.png)
</Frame>

Click any event row to view detailed information about the request.

### Filters

You can select multiple filters to get a more granular view of the displayed information.

<Frame>
  ![Events filters](/images/docs/waap/analytics/waap-events-filters.png)
</Frame>

Some filters offer drop-down options, such as **Response Codes**, **HTTP Methods**, **Actions**, and **Countries**.
