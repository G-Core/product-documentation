Gcore WAAP provides multiple ways to control who can access your application. Access control allows you to permit or deny traffic based on IP addresses, geographic location, request metadata, and custom security rules.

The simplest mechanism is the **Firewall**, which lets you allowlist or denylist specific IP addresses or IP ranges. For more advanced scenarios, you can use **CDN security policies** or **Custom Rules** to control traffic based on additional conditions such as country, referrer, or user agent.

## Firewall: Allowlist and denylist IP addresses

The **Firewall** provides a quick way to explicitly allow or block traffic from specific IP addresses or ranges before requests reach your application.

Firewall rules contain a single condition—either **allow** or **block** an IP. They are fast to implement and are useful when you need to immediately permit trusted sources or deny known malicious ones.

All rules with allowed or blocked IPs appear on the **Firewall** page.

<Frame>
![Firewall page in the Customer Portal](/images/docs/waap/ip-security/firewall-page.png)
</Frame>

A rule can appear on this page in one of the following ways:

* You manually added an IP address to the allowlist or denylist on the **Firewall** page.
* You've created a new rule for the IP address on the [Events](/waap/analytics#create-rule-for-ip) page.

<Tip>
**Tip**

If you want to add a rule for verified bots and crawlers, check our [Known Bots](/waap/waap-policies/known-bots) section to see if a similar policy already exists.

Enabling predefined policies gives you immediate protection without further testing and manual adjustments. They are optimized to accurately distinguish between legitimate and harmful automated traffic, reducing the risk of false positives or negatives.
</Tip>

## Add IPs to the allowlist or denylist

You can allow or block traffic for both IPv4 and IPv6 addresses. For each address type, create a separate firewall rule.

1. In the Gcore Customer Portal, navigate to **WAAP** > **Firewall**.

<Frame>
![Firewall page in the Customer Portal](/images/docs/waap/ip-security/firewall-page.png)
</Frame>

2. In the domain dropdown at the top of the page, select the needed domain.

3. Navigate to the required section:

* **Allowed IPs** – Permit specific IPs or IP ranges to access your domain without restrictions or security checks.
* **Blocked IPs** – Deny specific IPs or IP ranges access to your domain.

<Frame>
![Firewall page in the Customer Portal](/images/docs/waap/ip-security/firewall-page.png)
</Frame>

4. Click **Add IP/IP Range**.

To allow or block traffic for an IP range, enter its first and last IP address.

You can specify up to **30 networks** in a single range. Subnet masks are not supported.

5. Enter the IP information and provide a description.

To add an IP range, enter the **first address** and the **last address** in the corresponding fields.

<Info>
**Info**

You can't add a subnetwork to the IP or IP range.
</Info>

6. Click **Save**.

Depending on the selected action, traffic from the IP address will be either allowed or blocked for your domain.

## Manage blocked and allowed IPs

You can view, create, and customize rules for blocking IP addresses and ranges on the **Firewall** page.

If you have many rules configured, you can search by **rule description** to quickly find the one you need.

All blocked and allowed IPs are also visible on the **Custom Rules** page, where you can temporarily disable or enable them if necessary.

<Warning>
**Warning**

If you disable a firewall rule on the **Custom Rules** page, it will no longer be visible on the **Firewall** page. Once re-enabled, the rule will automatically appear on the **Firewall** page again.
</Warning>

### Update an IP or IP range

If the IP address of a trusted client has changed or you've detected new threats within an IP range, you can update or remove the IP entry.

To update or remove an IP or IP range:

1. In the Gcore Customer Portal, navigate to **WAAP** > **Firewall**.

<Frame>
![Firewall page in the Customer Portal](/images/docs/waap/ip-security/firewall-page.png)
</Frame>

2. In the domain dropdown at the top of the page, select the needed domain.

3. Navigate to the relevant section—**Allowed IPs** or **Blocked IPs**.

4. Find the IP you want to manage and click the three-dot icon next to it.

5. Select the required action—**Edit** or **Delete**—and follow the instructions.

<Frame>
![Allowed IPs table with IP management options](/images/docs/waap/ip-security/edit-delete-ip.png)
</Frame>

## Delete multiple rules with group actions

You can delete multiple firewall rules at once.

To do this, select the checkboxes next to the rules you want to remove and choose **Delete** from the **Actions** dropdown.

<Frame>
![Actions dropdown on the Firewall page](/images/docs/waap/ip-security/actions-dropdown.png)
</Frame>

## Additional access control options

In addition to firewall allowlists and denylists, Gcore provides broader access control capabilities.

### CDN security policies

You can control access to your content using CDN-level security policies that filter requests based on:

* **Country**
* **Referrer**
* **IP**
* **User-Agent**

These policies allow you to restrict access based on geographic location, request origin, or client characteristics.

For more information, see [Control access to content with country, referrer, IP, and user-agent policies](https://gcore.com/docs/cdn/cdn-resource-options/security/control-access-to-the-content-with-country-referrer-ip-and-user-agents-policies).

### Custom Rules

For advanced scenarios, you can create **Custom Rules** that define detailed traffic filtering logic.

Custom rules allow you to:

* Combine multiple request attributes
* Apply different actions (block, allow, challenge, etc.)
* Create conditional security policies tailored to your application

Use custom rules when firewall IP rules are not sufficient or when you need more granular traffic control.

See the **Custom Rules** section for more information.
