Gclaw is a Gcore-hosted, fully managed version of [OpenClaw](https://openclaw.ai/) with built-in inference powered by Gcore GPU infrastructure. It allows launching an AI agent instantly without setting up infrastructure, managing tokens, or configuring providers.

## Capabilities

With Gclaw:

- A ready-to-use AI agent launches in seconds.
- Interaction occurs through a chat interface.
- Integrations such as [Telegram](/gclaw/connect-telegram) and [Discord](/gclaw/connect-discord) can be connected.
- Built-in skills for development, productivity, and automation are available.

## Getting started

To launch an instance and begin using Gclaw, follow the [Getting started guide](/gclaw/getting-started).

## Security model

Gclaw runs each instance in an isolated environment. The OpenClaw agent has system-level access to the instance and executes commands based on prompts.

Details about isolation, risks, and responsibilities are in the [security model](/gclaw/security) documentation.

## Integrations and skills

Gclaw includes built-in skills and supports external integrations.

- To view available skills, ask the assistant directly in the chat.
- To configure integrations and API keys, use the [API key configuration](/gclaw/api-keys) guide.

## Support

Gcore provides support for infrastructure and billing issues. For OpenClaw-specific questions, refer to the [OpenClaw documentation](https://docs.openclaw.ai/).

| Issue type | Gcore support |
|------------|---------------|
| Service down or 500 errors | Yes |
| Billing and subscription issues | Yes |
| OpenClaw usage questions | No - refer to OpenClaw docs |
| Feature requests | Logged for consideration |

If the instance is not responding, click **Restart** in the **Settings** section before contacting support.
