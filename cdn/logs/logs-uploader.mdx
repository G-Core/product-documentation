title: "Logs uploader – live"
---

Logs uploader automatically exports CDN resource logs to storage destinations in near real time. Exported logs contain information about user requests sent to cache servers and pre-cache servers (if the [Origin shielding](/cdn/cdn-resource-options/general/enable-and-configure-origin-shielding) feature is enabled).

<Info>
  **Logs uploader** is an add-on feature; [contact Gcore support team] mailto:[support@gcore.com](mailto:support@gcore.com)) to activate it for your account.
</Info>

The **Logs uploader** feature can be enabled and set up in the Gcore Customer Portal on the [Gcore CDN page](https://gcore.com/cdn).

## Configuring Logs uploader

In the **CDN** menu, click **Logs**.

- **Targets** — connect to storage destinations that receive logs.
- **Policies** — set export rules and schedules for log delivery.
- **Configurations** — link policies with targets to control how logs are delivered.

### Targets

A target is the storage destination for logs.

Select the **Targets** tab to view a table of targets:

- Target **ID**\
  Click the column header to sort the table by target IDs.
- **Target** name
  - Click the column header to sort the table by target names.
  - Click a target name to edit configuration or connection details.

<Tip>
  Use the search box above to filter targets by name or ID.
</Tip>

- **Configuration**\
  Click a configuration name to edit the configuration.
- **Status**\
  Connected targets show `Authentication complete` in the **Status** column.
- **More options**\
  Click **⋯** (more options) in the last column to edit or delete the target in that row.

#### Adding targets

To add a new target:

1. Go to **Logs uploader**, select the **Targets** tab, and click **Add target**.
2. Enter the target name.
3. Select a targer destination or a delivery protocol for log export, and configure connection details:

    - Optionally, specify an existing **folder** within the bucket for receiving the logs.

    - Turn on the **Authentication** switch to enable token-based authentication using HTTP headers.

#### Editing targets

To edit a target:

1. Go to **Logs uploader** and select the **Targets** tab.
2. Click target name in the **Target** column, or click **⋯** (more options) in the last column and select **Edit**.

<Info>
  Target authentication status is displayed at the top of the **Edit target** window.
</Info>

4. Edit target name.
5. Edit destination or a delivery protocol for log export.
   - Destination options include Gcore S3-compatible Object Storage, Amazon S3, Alibaba Cloud OSS, and other S3-compatible object storage.
   - Available delivery protocols are FTP, SFTP, and HTTP(S).
6. Configure required connection details:

   For S3-compatible destinations:
   - Endpoint and protocol (for Gcore and other S3-compatible destinations only).
   - Access credentials (key ID and secret access key).
   - Region and bucket name.
   - Optionally, specify an existing folder in storage for receiving the logs.

   For delivery protocols:
   - **FTP**: Hostname and access credentials (login and password). Optionally, the prepend (destination) directory.
   - **SFTP**: Hostname, authentication method (password or private key) and access credentials. Optionally, the prepend (destination) folder.
   - **HTTP(S)**: POST or PUT method, endpoint and protocol. Optionally, custom headers, header/token authentication, and JSON log file compression.

     <Info>
       Compression reduces log file size by up to 99%.
     </Info>

   Click **Save changes** to complete editing and return to the Logs uploader **Targets** tab.

   <Info>
     Correctly configured targets display `Authentication complete` in the **Status** column.
   </Info>

## How near real time log exporting works

Logs are achived and exported to your endpoint every five minutes. If CDN servers are not requested and you didn't select the **Do not send empty logs** checkbox when configuring Logs uploader, an empty log file (± 20 bytes) will be sent to your storage.
