A CDN resource for static assets uses a CNAME-based setup to accelerate delivery of images, scripts, stylesheets, and other static files. The existing DNS provider continues to manage the domain — only one CNAME record needs to be added, and the static file URLs updated.

To accelerate the entire site (both static and dynamic content) without code changes, use the [entire site guide](/cdn/getting-started/create-a-cdn-resource/create-a-cdn-resource-for-the-entire-site) instead.

## Step 1. Start creation

Go to the [CDN](https://portal.gcore.com/cdn/resources/list) page and click **Create CDN resource**.

<Frame>
  ![Create CDN resource button](/images/docs/cdn/getting-started/create-a-cdn-resource/create-a-cdn-resource-for-only-static-files/cdn-resources-create.png)
</Frame>

## Step 2. Configure domain and DNS

The following sections cover entering the custom domain, selecting the DNS option, and optionally enabling WAAP — all done on the same creation form.

### 1\. Enter a custom domain

In the **Add domain** field, enter a subdomain in the format `cdn.<website domain>`. For example, for the site `yourwebsite.com`, enter `cdn.yourwebsite.com`.

This subdomain is used as a CNAME to route static file requests through the CDN. Multiple URL paths can point to it from the website source.

<Warning>
After a CDN resource is created, the custom domain cannot be changed.
</Warning>

<Frame>
  ![Add domain field](/images/docs/cdn/getting-started/create-a-cdn-resource/create-a-cdn-resource-for-only-static-files/cdn-resources-create-1.png)
</Frame>

### 2\. Select the DNS option

Select **Do not delegate** to keep DNS at the current provider. No nameserver change is required — only a single CNAME record is added in [Step 5](#step-5-add-a-cname-record).

### 3\. Enable WAAP (optional)

Enable the **Enable WAAP** toggle to activate Web Application and API Protection alongside CDN acceleration. WAAP provides Layer 7 DDoS protection and web application security; configure it after resource creation using the [WAAP setup guide](/waap/getting-started/configure-waap-for-a-domain).

<Frame>
  ![Enable WAAP toggle](/images/docs/cdn/getting-started/create-a-cdn-resource/create-a-cdn-resource-for-only-static-files/cdn-resources-create-waap.png)
</Frame>

## Step 3. Configure origin

An origin is the source server (website, application, or storage bucket) from which the CDN fetches content. Each origin is associated with an [origin group](/cdn/add-an-origin-group).

Choose one of the following options:

- **Specify content origin** — select this when no origin group exists yet. Follow the instructions in step 2 of the [origin group guide](/cdn/add-an-origin-group#step-2-enter-the-origin-group-name).
- **Select origin group** — select this when an origin group already exists. Choose the required group from the dropdown. This option only appears when existing groups are available.

<Frame>
  ![Configure origin](/images/docs/cdn/getting-started/create-a-cdn-resource/create-a-cdn-resource-for-only-static-files/cdn-resources-create-origin.png)
</Frame>

### Add a description (optional)

Provide any additional notes about this CDN resource. The description is displayed next to the resource CNAME in the CDN resources list and does not affect settings.

## Step 4. Create the resource

Click **Create CDN resource** to create the resource. When WAAP was enabled in Step 2, the button reads **Create and enable protection**.

After creation, the portal displays the generated CNAME value in the format `*.gcdn.co`. Copy this value — it's needed in the next step.

## Step 5. Add a CNAME record

In the DNS provider's control panel, add a [CNAME record](/dns/dns-records#cname-record) for the subdomain entered in Step 2:

| Field | Value |
|---|---|
| **Name / Host** | `cdn.yourwebsite.com` (the subdomain entered in Step 2) |
| **Value / Points to** | The `*.gcdn.co` value shown in the portal, e.g. `cl-1ab23456.gcdn.co` |

{/* TODO: replace with screenshot of the DNS step in the portal showing the generated CNAME value */}
<Frame>
  ![CNAME record setup](/images/docs/cdn/getting-started/create-a-cdn-resource/create-a-cdn-resource-for-only-static-files/dns-cname-setup.png)
</Frame>

When multiple custom domains were entered, create a CNAME record for each one pointing to the same `*.gcdn.co` value.

## Step 6. Integrate the CDN resource with the website

Update the static file URLs to use the CDN subdomain instead of the origin domain.

1. In the website admin panel, replace the origin domain with the CDN subdomain in static file URLs — either via script or manually.

   For example, change `yourwebsite.com/images/image.jpg` to `cdn.yourwebsite.com/images/image.jpg`. Uncached requests continue to reach the origin server, while cached files are served from CDN edge nodes.

   For sites built on a CMS such as WordPress, Joomla, or Drupal, use a plugin to replace domain names in file paths. The [CMS integration guide](/cdn/getting-started/integrate-cdn-with-cms) covers the supported platforms and plugins.

2. Return to the Gcore Customer Portal and click **Confirm**.

<Frame>

CDN resource creation is complete. Click **Open resource settings** to view and configure advanced settings.

If the **... steps left** indicator is not shown, the resource is created and working correctly. If it is shown, see the troubleshooting section below.

## Troubleshoot resource creation

If the **... steps left** indicator appears after creation, something needs attention.

1. Click **Setup guide**. A panel opens showing the status of each creation step.

2. Click **Check DNS setup status**.
