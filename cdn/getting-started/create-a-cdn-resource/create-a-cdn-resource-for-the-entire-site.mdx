A CDN resource for the entire site accelerates both static and dynamic content delivery using Gcore CDN and Managed DNS — without requiring code changes to the website. Gcore takes over DNS management for the domain, routing all traffic through the CDN.

To accelerate only static assets (images, scripts, stylesheets), use the [static assets guide](/cdn/getting-started/create-a-cdn-resource/create-a-cdn-resource-for-only-static-files) instead. That approach uses a simpler CNAME-based setup that keeps DNS at the current provider.

We explain the differences in delivery in the [CDN performance article](https://gcore.com/blog/how-to-speed-up-dynamic-content-delivery-using-cdn).

<Warning>
If either CDN or Managed DNS stops working, the site will also stop being accelerated.
</Warning>

## Create and integrate a CDN resource for the entire site

The following steps cover the complete setup, from entering a domain to changing nameservers at the registrar.

### Step 1. Start creation

Go to [CDN resources](https://portal.gcore.com/cdn/resources/list) and click **Create CDN resource**.

### Step 2. Enter the domain

In the **Add domain** field, enter the domain to accelerate. Accepted formats are:

- Second-level domain (`testdomain.com`)
- Third-level domain (`store.testdomain.com`, `blog.testdomain.com`)
- Higher-level domain (`sales.store.testdomain.com`)

In all cases, the DNS zone created in Gcore Managed DNS is the second-level domain (e.g., `testdomain.com`). The exception is effective top-level domains like `co.uk`, where the zone is the subdomain plus the eTLD: `example.co.uk`.

<Frame>
  ![Add domain field](/images/docs/cdn/getting-started/create-a-cdn-resource/create-a-cdn-resource-for-the-entire-site/add-domain-field.png)

<Info>
**Notes**:

- Domains already registered on another account will be declined.
- When the domain matches an existing DNS zone in the account, that zone is used without duplication.
</Info>

An optional description can also be added. It is displayed next to the resource CNAME in the CDN resources list.

### Step 3. Select DNS delegation

Select the option to delegate DNS to Gcore. This automatically creates a DNS zone for the domain in [Gcore Managed DNS](https://gcore.com/dns).

- When Managed DNS hasn't been used before, the Free plan activates automatically.
- When Managed DNS is already in use, the new zone is added to the existing account.

### Step 4. Enable WAAP (optional)

Enable the **Enable WAAP** toggle to activate Web Application and API Protection alongside CDN acceleration. WAAP provides Layer 7 DDoS protection and web application security; complete the configuration after resource creation using the [WAAP setup guide](/waap/getting-started/configure-waap-for-a-domain).

### Step 5. Configure origin

An origin is the source server from which the CDN fetches content. Choose one of the following options:

- **Specify content origin** — select this when no origin group exists yet. Follow the instructions in step 2 of the [origin group guide](/cdn/add-an-origin-group#step-2-enter-the-origin-group-name).
- **Select origin group** — select this when an origin group already exists. Choose the required group from the dropdown. This option only appears when existing groups are available.

### Step 6. Review DNS records

The system automatically looks up existing DNS records for the domain.

- **When DNS records exist**, they are displayed and can be edited, deleted, or added to as needed.

{/* TODO: replace with screenshot of the DNS records review with existing records shown */}
<Frame>
  ![DNS records with existing entries](/images/docs/cdn/getting-started/create-a-cdn-resource/create-a-cdn-resource-for-the-entire-site/dns-records-existing.png)
</Frame>

- **When no DNS records are found**, a warning indicates that at least an A record should be added. Enter the subdomain (or leave the **Name** field empty for the root) and the origin IP address, then click **Add**.

{/* TODO: replace with screenshot of the DNS records review with no records found and A record entry */}
<Frame>
  ![DNS records with no existing entries](/images/docs/cdn/getting-started/create-a-cdn-resource/create-a-cdn-resource-for-the-entire-site/dns-records-empty.png)
</Frame>

Click **Confirm** to proceed. When WAAP was enabled in Step 4, the button reads **Create and enable protection**.

### Step 7. Change nameservers

A CDN resource and DNS zone are automatically created for the specified domain. To complete the setup, log in to the domain registrar and replace the current nameservers with the Gcore nameservers shown in the portal.

{/* TODO: replace with screenshot of the nameservers step in the portal */}
<Frame>
  ![Change nameservers](/images/docs/cdn/getting-started/create-a-cdn-resource/create-a-cdn-resource-for-the-entire-site/change-nameservers.png)
</Frame>

<Info>
The nameserver values shown in the portal are authoritative. The current values are typically:

```
ns1.gcorelabs.net
ns2.gcdn.services
```
</Info>

Nameserver changes can take up to 24 hours to propagate. Click **Confirm** after updating the registrar settings.

### Step 8. Set up additional options (optional)

Additional features can be activated at this step or configured later in resource settings. Click **Confirm** to complete the setup.
